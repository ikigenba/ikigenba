# Decision 7 — setup provisions the served `www` tree as `<app>:web`, setgid, via one `ensureWWWPerms` helper

## Decision

A service that serves files off disk through the front door (today only `sites`,
selected by `WWWDirsFor` returning a non-empty dir list) gets a `state/www` tree
with `working/`, `public/`, and `private/` subdirs. For nginx to serve those
tiers, the tree must satisfy one **invariant**:

- **owner** `<app>` (the service writes it),
- **group** `web` (so the nginx worker, a `web` member per D6, can traverse and
  read it),
- **tier dirs setgid** (`2750`) so every entry the service later creates under
  them — a new site's `working/<name>/`, its files, the `public|private/<name>`
  symlink — **inherits group `web`** instead of the service's primary group.
  Without setgid, the very next publish drops back to `<app>:<app>` and nginx is
  locked out again.

Nothing needs to be world-readable: nginx always reaches both tiers *through* the
`web` group, so the private tier's bytes stay unreadable to other local users —
the front door's session gate is not the only thing protecting them.

Today `setup.go` expresses www provisioning in **two divergent places**: the
no-fragment (worker) branch chowns `www` to `app:web` (dead — no worker service
has a served tree), while the fragment branch that `sites` actually takes
(`len(opts.WWWDirs) > 0`) chowns it **world-readable `0755 app:app`**. Neither
sets setgid, and the two disagree. This Decision **unifies both onto the `web`
group** and extracts the invariant into a single helper so it is defined once and
reused by deploy (D8) and restore (D9):

```go
// internal/opsctl/www.go (new) — the single definition of the served-tree invariant.
// Idempotent; safe to re-run. Only touches tier dirs that exist.
func (o *Opsctl) ensureWWWPerms(ctx context.Context, app string, l Layout) error {
    if _, err := os.Stat(l.WWWDir()); err != nil {
        return nil // no served tree for this app — nothing to do
    }
    // group-own the whole served subtree by nginx's web group
    if err := o.System.ChownTree(ctx, app, "web", l.WWWDir()); err != nil {
        return fmt.Errorf("ensure www perms: chown %s -> %s:web: %w", l.WWWDir(), app, err)
    }
    // setgid each tier dir so future entries inherit group web
    for _, d := range []string{l.WWWDir(), l.WWWWorkingDir(), l.WWWPublicDir(), l.WWWPrivateDir()} {
        if _, err := os.Stat(d); err != nil {
            continue
        }
        if err := o.System.Chmod(ctx, d, 0o2750); err != nil {
            return fmt.Errorf("ensure www perms: setgid %s: %w", d, err)
        }
    }
    return nil
}
```

Setgid is a new privileged mode change, so the `System` seam gains one method:

```go
// System (seam.go) gains:
//   // Chmod sets path's mode (the box runs `chmod`). Used to set the setgid bit
//   // (02750) on served-tree tier dirs so new entries inherit their group.
//   Chmod(ctx context.Context, path string, mode os.FileMode) error
```

In `Setup`, the fragment branch's www block (`len(opts.WWWDirs) > 0`) creates the
dirs `0750` and then calls `ensureWWWPerms` instead of the old world-readable
`ChownTree(app, app, WWWRoot)`. The now-dead `web`-group www block in the
no-fragment branch (and its `EnsureSystemGroup("web")`, which D6 moves to
init-box) is removed. `setup` assumes the `web` group already exists — D6
guarantees init-box created it before any `setup` runs; if it is somehow absent
the `ChownTree(... "web" ...)` fails loudly, which is the correct "failures are
loud" behavior, not a silent skip.

**Boundary.** D7 changes only how a **served-tree** app's `state/www` is owned and
moded at `setup`, and introduces the shared `ensureWWWPerms` helper + the `Chmod`
seam. It does not create the `web` group or touch the nginx user (D6), does not
change the DEFAULT-app branch (D5 — dashboard gets no www tree), does not change
the unit/user/nginx-fragment steps, and does not change `deploy`/`restore` (D8/D9
reuse the helper). Apps with no served tree (`WWWDirsFor` empty) are unaffected:
`ensureWWWPerms` is only called on the served-tree branch, and self-guards on
`WWWDir` existence anyway.

## Rejected

- **World-readable `0755 app:app`** (the status quo for the fragment branch).
  Exposes the private tier on disk to every local box user and needs no group at
  all — but "the front door can read it" then rests on `other`-readability, which
  can't distinguish public from private. The `web` group serves both tiers to
  nginx while keeping private bytes off `other`. Rejected.
- **Chown the tree but skip setgid.** Fixes the *existing* entries once; the next
  `sites publish` creates `working/<name>/` and the tier symlink with the
  service's primary group `<app>`, re-locking nginx out. Setgid is what makes the
  invariant survive future publishes. Rejected.
- **Leave the two setup branches divergent; only fix the fragment branch inline.**
  Keeps the invariant expressed in two-to-three places (setup, deploy, restore) —
  exactly how it drifted. One helper is the single source of truth. Rejected.
- **Set setgid by passing `0o2750` to `MkdirAll` at creation instead of a
  `Chmod` seam call.** Go's `MkdirAll` mode is masked by umask and the S_ISGID
  bit is not reliably honored across `mkdir`; an explicit post-create `chmod`
  through the seam is both reliable and unit-observable (the fake records it).
  Rejected.

## Verification

- R-AT2M-15HI — **Setup of a served-tree app labels `state/www` `<app>:web` and
  setgids the tier dirs.** After `Setup` of an app whose `WWWDirsFor` is non-empty
  (`sites`), the fake `System` has recorded a `ChownTree(app, "web", WWWDir())`
  **and** a `Chmod(d, 02750)` for each existing tier dir (`WWWDir`,
  `WWWWorkingDir`, `WWWPublicDir`, `WWWPrivateDir`), and has recorded **no**
  `ChownTree(app, app, …)` over the www tree and no `0755` www mode. Falsifiable:
  today's fragment branch records `ChownTree(app, app, WWWRoot())` and no
  setgid. *(unit: temp `OPSCTL_ROOT` + fake `System`)*
- R-AUAI-EX87 — **Setup of a no-served-tree app touches no www/`web` state.**
  After `Setup` of an app whose `WWWDirsFor` is empty (a fragment or worker app
  other than sites), the fake `System` records **no** `ChownTree(_, "web", _)` and
  **no** `Chmod` of a www path, and creates no `state/www` tree — the worker
  symlink / fragment-file behavior (D5/D01) is unchanged. Pins that the web-group
  labelling is scoped to served-tree apps only. *(unit: temp `OPSCTL_ROOT` + fake
  `System`)*
