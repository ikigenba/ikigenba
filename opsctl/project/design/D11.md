# Decision 11 — init-box installs the oauth CLI via its release installer to `/usr/local/bin`

## Decision

`oauth` (`github.com/ikigenba/oauth`) is a standalone OAuth-login CLI that
services (starting with prompts' `auth=sub` capability) drive to mint token
`auth.json` files. It is **box-global substrate** in the same class as D10's
tooling — installed once by provisioning, usable by any service, owned by no
single service — but it is **not** an AL2023 `dnf` package. It ships as a
prebuilt release binary installed by a `curl … | sh` script. This is a new
install *mechanism* alongside D10's `dnf` packages, and future box-global tools
are expected to arrive the same way, so it gets its own Decision and its own seam
primitive rather than being folded into D10's package list.

### The generic `InstallScript` seam primitive

The installer runs a network fetch and executes a shell script as root — an
imperative box op, so it goes through the `System` seam (`seam.go`), where tests
record but never execute it. Because more `curl … | sh` tools are coming, the
seam gets a **generic** primitive, not an oauth-specific method — mirroring how
`InstallPackages` is a thin `dnf` primitive whose *policy* (which packages) lives
in init-box:

```go
// System (seam.go) gains:
//
// InstallScript fetches and runs a curl|sh installer as root (the box runs
// `curl -fsSL <installerURL> | sh`) with env layered over the process
// environment (each entry "KEY=VALUE"). Used for box-global tools distributed
// as a release installer rather than a dnf package. NOT idempotent by the
// seam's own doing — idempotency (or its absence) is the installer's; oauth's
// re-installs the latest release each run.
InstallScript(ctx context.Context, installerURL string, env ...string) error
```

`RealSystem` implements it by piping `curl -fsSL <installerURL>` into `sh` with
the extra env applied; the test fake records `install-script:<installerURL>` plus
the env it was handed, exactly as the existing fake records
`install-packages:<pkgs>`.

### init-box owns the policy

The choice of installer URL and install location lives in `InitBox`
(`internal/opsctl/initbox.go`), not the seam. It is added as a new step
immediately **after** the package install (D10, step 1) and **before** the
cert/nginx branch, so — like the package install — it runs on the `--skip-cert`
path too (oauth needs no nginx and no cert):

```go
// 1b. oauth CLI: box-global login tool (prompts' auth=sub, and future
//     services). Installed to /usr/local/bin so any user can execute it; the
//     installer's `install -m 0755` gives it world-exec mode. BINDIR overrides
//     the installer's ~/.local/bin default. No OAUTH_VERSION => latest release,
//     so re-running init-box refreshes oauth to the newest build.
o.logf("install oauth CLI to /usr/local/bin")
if err := o.System.InstallScript(ctx,
    "https://raw.githubusercontent.com/ikigenba/oauth/main/install.sh",
    "BINDIR=/usr/local/bin"); err != nil {
    return fmt.Errorf("init-box: install oauth: %w", err)
}
```

The installer (see `project/research/oauth-installer-research.md`) honours
`BINDIR=/path` as the exact install directory (overriding its `~/.local/bin`
default) and, absent `OAUTH_VERSION`, downloads the **latest** release
`oauth_linux_amd64.tar.gz`, unpacks it (needs `tar`, provided by D10), and
`install -m 0755`s the binary to `$BINDIR`. The result is
`/usr/local/bin/oauth`, mode 0755, executable by any user — no separate chmod or
per-user install. `install -m 0755` overwrites in place, so a re-run reinstalls
the latest cleanly.

**Fail-loud:** a network failure, a missing release, or a non-zero installer exit
surfaces as a non-nil error from `InstallScript`, and init-box aborts with
`init-box: install oauth: …` rather than continuing as though oauth were present.

## Rejected

- **Fold oauth into D10's `InstallPackages` list.** oauth is not a `dnf` package;
  its install is a network fetch + script exec with a different failure surface,
  a different idempotency story (always-latest, not dnf's already-present no-op),
  and a different real-substrate proof. It is a distinct mechanism and gets a
  distinct Decision and seam op.
- **An oauth-specific seam method (`InstallOAuth`).** Future box-global tools
  install the same `curl … | sh` way; a per-tool seam method multiplies
  near-identical primitives. The generic `InstallScript(url, env…)` carries the
  next tool as one more call from init-box, and keeps the seam a thin executor
  with the *policy* (which URL, which BINDIR) testable in init-box code — exactly
  the `InstallPackages`/package-list split.
- **Pin `OAUTH_VERSION` to a fixed release.** Pinning gives byte-stable
  provisioning, but the operator's decision is that a box should always run the
  **latest** oauth and that re-provisioning refreshes it. So init-box passes no
  `OAUTH_VERSION` and accepts that a later re-provision may install a newer oauth
  than an earlier one — the intended behavior, not drift to guard against.
- **opsctl downloads the release tarball directly (skip the installer script).**
  Avoids executing a remote script, but re-implements os/arch detection, the
  release-URL scheme, and the unpack/install in opsctl, and diverges from the
  upstream-owned install contract. The operator's decision is to use the
  project's published `curl … | sh` path and establish it as the pattern for
  future tools; opsctl consumes the installer rather than reproducing it.
- **Install to a per-user location / not global.** oauth must be usable by any
  service user on the box; `/usr/local/bin` at mode 0755 is the box-global,
  any-user-executable home, installed once by the privileged provisioner (init-box
  runs as root via `sudo`). A per-user `~/.local/bin` install would not be shared.

## Verification

- R-ML75-3NVZ — `InitBox` requests the oauth installer with the correct URL and
  install location: through the fake `System`, the recorded ops contain an
  `install-script:https://raw.githubusercontent.com/ikigenba/oauth/main/install.sh`
  entry carrying env `BINDIR=/usr/local/bin`, and it is present on **both** the
  default and the `--skip-cert` path. Falsifiable: an implementation that omits
  the call, uses a different URL, drops the `BINDIR=/usr/local/bin` env (leaving
  the installer's `~/.local/bin` default), or only installs oauth on the cert path
  records different ops and fails this.
- R-MMF1-HFMO — **Real-substrate (live box `int.ikigenba.com`).** After `opsctl
  init-box` on the box, `command -v oauth` resolves to `/usr/local/bin/oauth`, the
  file is mode `0755` (any non-root user can execute it), and `oauth -V` exits
  cleanly printing a version; re-running `opsctl init-box` reinstalls the latest
  release and `oauth -V` still succeeds. That GitHub actually serves a runnable
  `linux/amd64` oauth binary the installer can fetch, unpack, and place executable
  cannot be falsified by the fake `System` (which records the call without running
  it); it requires the real installer against the real release. The observable
  proof is the runnable binary at the global path, not merely that an install was
  requested.
