# Decision 5 — `opsctl setup` provisions the DEFAULT app without a locations fragment

## Decision

`opsctl setup <app>` provisions one app: the `--system` user, the `/opt/<app>`
tree, the enabled-not-started unit, and the app's nginx artifact (`setup.go`).
For its nginx step it branches on `--fragment`:

- **`--fragment` given** (every path-routed service): render the fragment with
  the literal `__PORT__` and write it to `conf.d/locations/<app>.conf`.
- **no `--fragment`** (the D01 worker/no-route path): create a **symlink**
  `l.FragmentPath()` → `l.ActiveNginxConf()`, i.e. `conf.d/locations/<app>.conf →
  /opt/<app>/etc/current/nginx.conf`, intentionally dangling until the first
  deploy creates `etc/current`.

The **apex/`DEFAULT` app** (dashboard) passes **no `--fragment`** — its nginx
config is the apex vhost, not a `/srv/` location fragment — so today it would
fall into that second branch and get `conf.d/locations/dashboard.conf →
/opt/dashboard/etc/current/nginx.conf`. That is **wrong and box-fatal**. The
DEFAULT app's `etc/current/nginx.conf` is the **full apex vhost** (two `server{}`
blocks plus `include …/conf.d/locations/*.conf`), and the `locations/*.conf`
files are `include`d **inside** the apex `server{}`. So once the first deploy
creates `etc/current`, that symlink resolves to a whole `server{}`-containing
file nested inside a `server{}` → **`nginx -t` fatal**, breaking nginx box-wide
on the deploy's reload. `setup.go` has no DEFAULT-app branch today: it conflates
"no `--fragment` passed" with "worker."

The apex app's nginx config does **not** belong under `locations/` at all. It
lives at `l.ApexBlockPath()` (`conf.d/<app>.conf`), owned by `init-box` (first
render + cert, `initbox.go`) and re-rendered on every deploy by **D4**
(`project/design/D04.md`). So for the DEFAULT app, setup must install **no nginx
artifact whatsoever** — no fragment file and no locations symlink — while still
provisioning everything deploy needs: the user, the versioned `/opt/<app>` tree,
and the unit.

This Decision adds a **DEFAULT-app branch to `Setup`**, selected by a new
`SetupOptions.IsDefault` flag (CLI `--default`). Non-DEFAULT apps — both the
fragment and worker paths — are **unchanged**.

**The discriminator is a flag, not the manifest.** D4 reads `DEFAULT=true` from
the freshly-swapped `etc/current/manifest.env` because deploy runs **after** a
release exists. `setup` runs **before** the first stage/deploy, so `etc/current`
(and any on-box manifest) does **not** exist yet — there is nothing to read.
`init-box` already faces the same "no release yet" timing and already takes the
apex app's identity as a **flag** (`--default-app`, `main.go`). So a
provisioning-time `--default` flag on `setup` is the established pattern, not a
second source of truth competing with D4: the manifest field governs deploy-time
re-rendering; the flag governs provisioning-time layout. They never both read the
same value at the same moment.

```go
// SetupOptions gains:
//   IsDefault bool // this is the apex/DEFAULT app (dashboard): its nginx config
//   // is the apex block at l.ApexBlockPath(), owned by init-box + deploy (D4) —
//   // setup installs NO locations fragment/symlink for it.

func (o *Opsctl) Setup(ctx context.Context, opts SetupOptions) error {
    // ... app/locations-precondition checks unchanged ...

    // A DEFAULT app is the apex vhost, never a /srv/ fragment. Guard the
    // contradiction loudly rather than silently ignoring one input.
    if opts.IsDefault && opts.Fragment != "" {
        return fmt.Errorf("setup: %s: --default (apex app) and --fragment are mutually exclusive "+
            "(the apex app's nginx config is the apex block, not a locations fragment)", opts.App)
    }

    // ... packages, EnsureSystemUser unchanged ...

    // 2. tree
    switch {
    case opts.IsDefault:
        // Routed app whose route is the apex block, not a locations fragment.
        // Same versioned tree a fragment service gets (incl. backups/ for the
        // pre-migration snapshot), no world-readable www tree.
        if err := mkdirAll755(
            l.AppDir(), l.BinDir(), l.EtcDir(), l.LibexecDir(), l.CacheDir(), l.BackupsDir(),
        ); err != nil {
            return fmt.Errorf("setup: create app tree: %w", err)
        }
        if err := os.MkdirAll(l.StateDir(), 0o750); err != nil {
            return fmt.Errorf("setup: create state dir: %w", err)
        }
    case opts.Fragment == "":
        // ... worker branch, unchanged ...
    default:
        // ... fragment branch, unchanged ...
    }

    // 3. unit — unchanged (write + daemon-reload + enable-not-start).

    // 4. nginx artifact
    switch {
    case opts.IsDefault:
        // No locations fragment/symlink: the apex block at l.ApexBlockPath() is
        // owned by init-box (first render + cert) and re-rendered on each deploy
        // (D4). Setup deliberately writes nothing under conf.d/ for the apex app.
        o.logf("default app %s: apex block owned by init-box/deploy; no locations fragment", app)
    case opts.Fragment != "":
        // ... write rendered fragment, nginx -t + reload (or defer), unchanged ...
    default:
        // ... worker symlink → l.ActiveNginxConf(), unchanged ...
    }
    // ...
}
```

**CLI wiring.** `runSetup` (`cmd/opsctl/main.go`) gains a `--default` bool flag,
passed as `SetupOptions.IsDefault`. It mirrors `init-box`'s `--default-app`: the
operator states, at provisioning time, that this app is the apex app. The cutover
command is `opsctl setup dashboard --default` (no `--fragment`, no `--port`
needed for a fragment it will not write).

**Boundary.** D5 changes only what `setup` does for the **DEFAULT** app. It does
**not** touch the fragment branch (rendered fragment + `nginx -t`/reload/defer),
the worker branch (the `conf.d/locations/<app>.conf` → `etc/current/nginx.conf`
symlink), the user/unit steps, or the apex block itself (init-box owns the first
render + cert; D4 owns re-render on deploy). Setup for the DEFAULT app writes
nothing under `conf.d/` — neither `locations/<app>.conf` nor `<app>.conf`.

## Rejected

- **Leave setup as-is; let the worker branch create the locations symlink for
  dashboard.** The status quo, and the box-fatal bug: on the first dashboard
  deploy the symlink resolves to the full apex vhost nested inside the apex
  `server{}`, `nginx -t` fails, and the reload breaks routing box-wide. Rejected —
  this is precisely the defect D5 exists to remove.
- **Detect the DEFAULT app by reading `DEFAULT=true` from the manifest, like
  D4.** Impossible at setup time: setup runs **before** the first stage/deploy, so
  `etc/current/manifest.env` does not exist. There is no on-box manifest to read.
  Rejected on timing.
- **Detect the DEFAULT app by hardcoding the name `"dashboard"` in setup.**
  Couples setup to one app name and to `init-box`'s `--default-app` default. The
  operator already declares the apex app explicitly to `init-box`; setup takes the
  same explicit declaration. Rejected.
- **Derive DEFAULT-ness by comparing the app to the box's configured default-app
  (e.g. inspecting which `conf.d/<name>.conf` init-box wrote).** Reconstructs the
  operator's intent from a filesystem side effect — fragile and implicit. A flag
  states it directly. Rejected (explicit over implicit).
- **Give the DEFAULT app the worker tree (with the `state/www` world-readable
  subtree and `web` group).** Dashboard serves over loopback and shares no files
  with nginx (the apex `auth_request`es dashboard over HTTP), so a `web`-group
  `state/www` tree is inert clutter it never uses, and the worker branch omits the
  `backups/` dir dashboard's pre-migration snapshot needs. The DEFAULT app is a
  routed app whose route is the apex block, so it gets the fragment branch's tree
  (incl. `backups/`) minus any nginx artifact. Rejected.
- **Silently ignore `--fragment` when `--default` is set.** Swallows a
  contradictory operator instruction. The two are mutually exclusive by
  construction (the apex app has no locations fragment); saying so loudly is the
  "failures are loud and specific" promise. Rejected.

## Verification

- R-CIUC-KW66 — **Setup of a DEFAULT app provisions the tree + unit but installs
  no nginx artifact.** After `Setup` with `IsDefault=true` (and no `Fragment`):
  the `/opt/<app>` tree exists (`AppDir`, `BinDir`, `EtcDir`, `LibexecDir`,
  `CacheDir`, `BackupsDir`, `StateDir`), the unit file is written and enabled
  (via the recorded `System.EnableUnit(app+".service", false)`), and **no file or
  symlink exists at `l.FragmentPath()`** (`conf.d/locations/<app>.conf`) **and no
  file exists at `l.ApexBlockPath()`** (`conf.d/<app>.conf`). Falsifiable: today's
  code creates the dangling `l.FragmentPath()` symlink, failing the "no artifact
  at FragmentPath" assertion. *(unit: temp OPSCTL_ROOT + fake `System`)*
- R-CK28-YNWV — **Setup of a non-DEFAULT no-fragment (worker) app still creates
  the locations symlink.** After `Setup` with `IsDefault=false` and an empty
  `Fragment`, `l.FragmentPath()` is a **symlink** whose target is
  `l.ActiveNginxConf()` (`/opt/<app>/etc/current/nginx.conf`) — the D01 worker
  behavior, unchanged. Pins that D5 narrows *only* the DEFAULT case; an
  implementation that skips the symlink for every no-fragment app fails this.
  *(unit: temp OPSCTL_ROOT + fake `System`)*
- R-CLA5-CFNK — **Setup of a fragment app is unchanged.** After `Setup` with a
  non-empty `Fragment` (and `IsDefault=false`), `l.FragmentPath()` is a **regular
  file** whose bytes are the fragment source with `__PORT__` substituted for the
  given `Port` (`renderFragment`), and the fake `System.NginxTest`/`NginxReload`
  are invoked (unless `DeferNginx`). Regression guard on the path-routed services.
  *(unit: temp OPSCTL_ROOT + fake `System`)*
- R-CMI1-Q7E9 — **`--default` with `--fragment` is rejected.** `Setup` with
  `IsDefault=true` **and** a non-empty `Fragment` returns a non-nil error whose
  message names both `--default` and `--fragment` (mutually exclusive), and
  provisions nothing for the nginx step — no file/symlink at `l.FragmentPath()`
  and none at `l.ApexBlockPath()`. Pins the contradiction guard; an implementation
  that silently ignores one input fails this. *(unit: temp OPSCTL_ROOT + fake
  `System`)*
