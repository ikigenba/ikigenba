# Decision 10 — init-box installs the box-baseline command-line tooling (poppler-utils, git, sqlite, tar, curl-minimal)

## Decision

Some suite tooling is **box-global substrate**: present on every provisioned box,
relied on without any per-run or per-service install.

- The content-plane design (`docs/content-plane-design.md`) settled that a
  sandboxed prompts run gets its PDF tooling **from the box** — `pdftotext`,
  `pdftoppm`, `pdfinfo`, which ship on AL2023 in **`poppler-utils`**.
- The repos service provisions repositories by shelling out to **`git`** (`git
  clone` and subsequent git operations); without git on the box, provisioning
  cannot proceed. git is a box-global tool, not a private dependency of one
  service. On AL2023 the `git` package provides `/usr/bin/git`.
- Operators inspect the per-app SQLite databases from the command line with
  **`sqlite3`**, which ships on AL2023 in the **`sqlite`** package
  (`/usr/bin/sqlite3`).
- The oauth release installer (D11) is a `curl … | sh` script that fetches a
  release tarball and unpacks it. It needs **`tar`** to unpack and a **`curl`**
  that supports `-fsSL`. Both are effectively present on AL2023 (curl via
  `curl-minimal` from the base image), but init-box makes the guarantee explicit
  rather than depending on the ambient image: it names **`tar`** and
  **`curl-minimal`** in the baseline. `curl-minimal` (not `curl`) is deliberate —
  it is the package AL2023 already ships, so `dnf install -y curl-minimal` is a
  true no-op, whereas `dnf install -y curl` would attempt to *swap* the base
  `curl-minimal` for the full `curl` package and can fail as a conflict on a
  minimal box.

All are the same class of thing `opsctl init-box` already owns. init-box installs
its substrate packages as one call through the existing `System` seam
(`internal/opsctl/initbox.go`, step 1); this Decision carries the full list in
that same call — no new seam, no new verb, no new option:

```go
// 1. Packages: nginx + certbot (front door), poppler-utils (box-baseline PDF
//    tooling for sandboxed prompts runs), git (repository provisioning), sqlite
//    (the sqlite3 CLI for inspecting per-app databases), and tar + curl-minimal
//    (what the D11 oauth release installer needs to fetch and unpack).
if err := o.System.InstallPackages(ctx, "nginx", "certbot", "poppler-utils", "git", "sqlite", "tar", "curl-minimal"); err != nil {
    return fmt.Errorf("init-box: install packages: %w", err)
}
```

The install remains step 1 of `InitBox`, before the cert/nginx branch, so it runs
on the `--skip-cert` path too. **Idempotency is unchanged**: the `System` seam's
contract (`seam.go`) is `dnf install -y <pkgs…>`, a no-op for already-present
packages, and init-box is documented and tested as safe to re-run — extending the
list preserves that property (`curl-minimal` chosen precisely so the install stays
a no-op).

## Rejected

- **Per-service install via `opsctl setup <svc> --packages …`.** The mechanism
  exists (`SetupOptions.Packages`, an operator flag), but it makes a tool's
  presence depend on the operator remembering a flag on one service's setup
  invocation — a convention, not a provisioning guarantee. These tools belong to
  the **box**, not to one service; the box-global one-time verb is init-box,
  exactly as with nginx/certbot. (`--packages` remains the right home for deps a
  single service privately owns, like scripts' python3.11.)
- **A named package-list constant or a new "system packages" seam.** init-box
  passes its package literals inline at the single call site. The baseline list is
  now seven literals and still reads clearly inline; introducing a constant or a
  dedicated step still restructures more than it adds. If the list grows much
  further, factoring it out remains a future Decision — not triggered here.
- **`git-core` instead of the `git` meta-package.** `/usr/bin/git` is provided by
  `git-core`, but the conventional, unambiguous package name is `git` (it pulls
  `git-core` plus the standard git toolchain); it is the name an operator expects
  and `dnf install -y git` is idempotent all the same.
- **`curl` (the full package) instead of `curl-minimal`.** AL2023 ships
  `curl-minimal` in the base image; `dnf install -y curl` tries to erase it and
  install the full `curl`, which on a minimal box fails as a package conflict
  without `--allowerasing`. `curl-minimal` already provides a `/usr/bin/curl` that
  honours the installer's `-fsSL`, so naming `curl-minimal` keeps the install a
  clean idempotent no-op and never risks the swap. (D11 relies on this curl.)
- **Omitting `tar`/`curl-minimal` and depending on the base image.** They are
  present on AL2023 today, but the D11 installer's success then rests on an
  unstated assumption about the AMI. Naming them makes the dependency explicit and
  the guarantee init-box's, at the cost of two idempotent no-op literals.
- **Installing inside the prompts sandbox at run time.** Slow (per-run dnf),
  requires network and root inside a sandbox that deliberately has neither, and
  contradicts the decision that the box provides the tooling.

## Verification

- R-JQGB-RYA2 — `InitBox` requests the package install of the full box baseline:
  through the fake `System`, the recorded ops contain
  `install-packages:nginx,certbot,poppler-utils,git,sqlite,tar,curl-minimal`
  (including on the `--skip-cert` path). Falsifiable: an implementation missing any
  baseline package (e.g. the previous
  `install-packages:nginx,certbot,poppler-utils,git,sqlite`, or one that names
  `curl` instead of `curl-minimal`) records a different op and fails this.
- R-JRO8-5Q0R — **Real-substrate (live box `int.ikigenba.com`).** After `opsctl
  init-box` on the box, every box-baseline binary resolves and runs: `command -v
  git sqlite3 pdftotext pdftoppm pdfinfo tar curl` succeeds, and `git --version`,
  `sqlite3 --version`, `pdftotext -v`, `tar --version`, and `curl --version` exit
  cleanly; re-running init-box succeeds with the packages already present. That
  AL2023's dnf actually provides these binaries under the names `git`, `sqlite`,
  `poppler-utils`, `tar`, and `curl-minimal` cannot be falsified by the fake
  `System` (which accepts any package name); it requires the real package manager.
  The observable proof is the runnable binaries, not merely that an install was
  requested.
