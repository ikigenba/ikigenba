# Decision 3 — The typed GitHub REST v3 client

## Decision

`internal/gh` exposes a `Client` — a thin, hand-rolled REST v3 client over the
`tokenSource` (D2) — implementing exactly the org-scoped calls the tool surface
(D4) and the loopback twin (D5) need, and nothing more. It is safe for concurrent
use, does no I/O to disk, and reads no environment (constructed from config at the
composition root). Every request sets `Authorization: Bearer <installation
token>` (from D2), `Accept: application/vnd.github+json`, and
`X-GitHub-Api-Version: 2022-11-28`. All calls are scoped to the single configured
org, so callers pass `repo` (and numbers/paths), never an owner.

Illustrative surface (types abbreviated; only the fields the tools surface are
modeled):

```go
type Client struct {
    org   string
    ts    *tokenSource
    http  *http.Client
}

// Repositories
func (c *Client) ReposList(ctx context.Context) ([]Repo, error)
func (c *Client) RepoGet(ctx context.Context, repo string) (Repo, error)

// Pull requests
func (c *Client) PRList(ctx context.Context, repo, state string) ([]PR, error)
func (c *Client) PRGet(ctx context.Context, repo string, number int) (PRDetail, error) // incl. changed files
func (c *Client) PRComment(ctx context.Context, repo string, number int, body string) (Comment, error)
func (c *Client) PRReview(ctx context.Context, repo string, number int, event, body string) (Review, error)
func (c *Client) PRMerge(ctx context.Context, repo string, number int, method string) (MergeResult, error)

// Issues
func (c *Client) IssueList(ctx context.Context, repo, state string) ([]Issue, error)
func (c *Client) IssueGet(ctx context.Context, repo string, number int) (Issue, error)
func (c *Client) IssueCreate(ctx context.Context, repo, title, body string) (Issue, error)
func (c *Client) IssueComment(ctx context.Context, repo string, number int, body string) (Comment, error)
func (c *Client) IssueUpdate(ctx context.Context, repo string, number int, patch IssuePatch) (Issue, error)

// Contents
func (c *Client) FileGet(ctx context.Context, repo, path, ref string) (FileContent, error)
func (c *Client) FilePut(ctx context.Context, repo, path string, in FilePut) (FileCommit, error)

var (
    ErrNotFound = errors.New("github: not found")   // 404
    ErrInvalid  = errors.New("github: unprocessable") // 422 (e.g. bad merge state, validation)
)
```

**Endpoint map** (all under `apiBase`, org = configured `IKIGENBA_GITHUB_ORG`):

| Method | Endpoint |
|---|---|
| `ReposList` | `GET /orgs/{org}/repos` |
| `RepoGet` | `GET /repos/{org}/{repo}` |
| `PRList` | `GET /repos/{org}/{repo}/pulls?state={state}` |
| `PRGet` | `GET /repos/{org}/{repo}/pulls/{n}` + `GET .../pulls/{n}/files` |
| `PRComment` | `POST /repos/{org}/{repo}/issues/{n}/comments` |
| `PRReview` | `POST /repos/{org}/{repo}/pulls/{n}/reviews` (`event`: APPROVE / REQUEST_CHANGES / COMMENT) |
| `PRMerge` | `PUT /repos/{org}/{repo}/pulls/{n}/merge` (`merge_method`) |
| `IssueList` | `GET /repos/{org}/{repo}/issues?state={state}` (drop entries carrying `pull_request`) |
| `IssueGet` | `GET /repos/{org}/{repo}/issues/{n}` |
| `IssueCreate` | `POST /repos/{org}/{repo}/issues` |
| `IssueComment` | `POST /repos/{org}/{repo}/issues/{n}/comments` |
| `IssueUpdate` | `PATCH /repos/{org}/{repo}/issues/{n}` (`state` / `labels` / `assignees`) |
| `FileGet` | `GET /repos/{org}/{repo}/contents/{path}?ref={ref}` (base64-decode `content`) |
| `FilePut` | `PUT /repos/{org}/{repo}/contents/{path}` (`message`, base64 `content`, optional `sha`) |

**Bot-only writes.** No write call (`PRComment`, `PRReview`, `PRMerge`,
`IssueCreate`, `IssueComment`, `IssueUpdate`, `FilePut`) sets an `author`,
`committer`, or any owner-identifying field. `FilePut` omits `author`/`committer`
entirely, so GitHub attributes the commit to the App identity. This is asserted on
the outbound request body (D3 owns the request; the log-side provenance is D4).

**Testability.** Every method is exercised offline against an injected
`http.RoundTripper` stub that asserts the request (method, path, query, body) and
returns a canned GitHub response for decode. The real-provider-accepts-it claim is
not made here — it is D2's `R-DMUT-QF4A` live proof via `health`.

## Rejected

- **GraphQL.** REST v3 matches the per-endpoint, per-verb shape of `gmail`'s client
  and needs no query-composition layer for a fixed, small verb set.
- **`go-github`.** A full SDK for ~14 endpoints drags a dependency the suite
  deliberately avoids; hand-rolled request/response for exactly these calls is
  small and self-contained.
- **Modeling full GitHub response schemas.** Only the fields the tools surface are
  decoded; over-modeling would be dead weight and churn against GitHub's evolving
  payloads.
- **Per-owner author on commits/comments.** Rejected by product: repos are mostly
  public and owner emails must never land in their history (see D4 for where owner
  provenance does go).

## Verification

- `R-DVE4-ETB5` — `ReposList` issues `GET /orgs/{org}/repos` and decodes the repo
  list (name, full name, private flag, default branch). Stubbed.
- `R-DWM0-SL1U` — `RepoGet(repo)` issues `GET /repos/{org}/{repo}` and decodes the
  single repo. Stubbed.
- `R-DXTX-6CSJ` — `PRList(repo, state)` issues `GET /repos/{org}/{repo}/pulls` with
  the `state` query and decodes the PR list. Stubbed.
- `R-DZ1T-K4J8` — `PRGet(repo, n)` fetches the PR **and** its changed files
  (`/pulls/{n}/files`), returning both in `PRDetail`. Stubbed with two canned
  responses.
- `R-E09P-XW9X` — `PRComment(repo, n, body)` POSTs to `/issues/{n}/comments` with a
  JSON body whose `body` is exactly the supplied text and which contains **no**
  owner-identifying field. Stubbed; the request body is asserted to carry no owner
  key.
- `R-E1HM-BO0M` — `PRReview(repo, n, event, body)` POSTs to `/pulls/{n}/reviews`
  with `event` set to the given value (APPROVE / REQUEST_CHANGES / COMMENT).
  Stubbed; the outbound `event` is asserted.
- `R-E2PI-PFRB` — `PRMerge(repo, n, method)` issues `PUT /pulls/{n}/merge` carrying
  the given `merge_method`. Stubbed; the outbound method is asserted.
- `R-E3XF-37I0` — `IssueList(repo, state)` issues `GET /repos/{org}/{repo}/issues`
  and **excludes** entries carrying a `pull_request` field (issues only, not PRs).
  Stubbed with a mixed list; the PR entry is dropped.
- `R-E55B-GZ8P` — `IssueGet(repo, n)` issues `GET /issues/{n}` and decodes the
  issue. Stubbed.
- `R-E6D7-UQZE` — `IssueCreate(repo, title, body)` POSTs to `/issues` with the
  given title and body and no owner field. Stubbed; body asserted.
- `R-E7L4-8IQ3` — `IssueComment(repo, n, body)` POSTs to `/issues/{n}/comments`
  with exactly the supplied body and no owner field. Stubbed.
- `R-EA0X-027H` — `IssueUpdate(repo, n, patch)` issues `PATCH /issues/{n}` carrying
  only the set fields among `state`, `labels`, `assignees`. Stubbed; the outbound
  body reflects exactly the supplied patch.
- `R-EB8T-DTY6` — `FileGet(repo, path, ref)` issues `GET /contents/{path}` with the
  `ref` query and returns the **base64-decoded** file bytes. Stubbed with a
  base64-encoded body; the decoded bytes are asserted.
- `R-ECGP-RLOV` — `FilePut(repo, path, in)` issues `PUT /contents/{path}` with the
  commit `message`, base64-encoded `content`, the optional `sha` when updating, and
  **no** `author`/`committer` field. Stubbed; the request body is asserted to carry
  the message and content and to omit author/committer.
- `R-D0IM-VQ7H` — Response status is mapped to typed errors: `404` → `ErrNotFound`,
  `422` → `ErrInvalid`, other non-2xx → a generic error wrapping the status; no
  non-2xx is silently treated as success. Asserted offline across stubbed statuses.
