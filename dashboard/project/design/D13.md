# Decision 13 — Collector worker: startup sample + 60s ticker on the serve context

**Decision.** The collector is one `appkit.Spec.Workers` function. It samples once
**immediately** at startup (so the page is never blank on boot), then every 60s on
a ticker, until the serve context is cancelled — at which point it returns `nil`
(clean shutdown alongside the HTTP server).

```go
// Config carries the injectable roots/interval so tests drive it fast against
// fixtures; production defaults are the real paths and time.Minute.
type Config struct {
	ManifestRoot string        // service discovery (default "/opt")
	CgroupRoot   string        // default "/sys/fs/cgroup"
	DiskPath     string        // system free-disk fs + /opt/<svc> base (default "/opt")
	MemInfoPath  string        // default "/proc/meminfo"
	Interval     time.Duration // default time.Minute
}

// Collect takes exactly one sample set — system free mem/disk and, per discovered
// service, cgroup memory + /opt/<svc> disk — and Appends them to the store, also
// recording the capacities. A source that is absent records 0; a source that
// errors unexpectedly records 0 AND logs (never aborts the sweep).
func Collect(store *Store, cfg Config, logger *slog.Logger)

// Run samples once, then on every Interval tick, returning nil when ctx is done.
func Run(ctx context.Context, store *Store, cfg Config, logger *slog.Logger) error {
	Collect(store, cfg, logger)          // immediate first sample
	t := time.NewTicker(cfg.Interval)
	defer t.Stop()
	for {
		select {
		case <-ctx.Done():
			return nil
		case <-t.C:
			Collect(store, cfg, logger)
		}
	}
}
```

**Wiring (`cmd/dashboard/main.go`).** Follow the established capture idiom: a
`var rt *appkit.Router` set in the `Handlers` hook, where the shared `*Store` is
constructed and handed to `server.Register`; a `Workers` closure captures both and
runs after Handlers, so `rt.Logger()` is live:

```go
var rt *appkit.Router
store := telemetry.NewStore()
appkit.Main(appkit.Spec{
	App: "dashboard", Mount: "/", Default: true, Port: 3000, MCP: false,
	Migrations: db.FS,
	Handlers: func(r *appkit.Router) error {
		rt = r
		return registerRoutes(rt, store) // store threaded into server.Options.Telemetry
	},
	Workers: []func(context.Context) error{
		func(ctx context.Context) error {
			return telemetry.Run(ctx, store, telemetry.Config{ManifestRoot: manifestRoot}, rt.Logger())
		},
	},
})
```

A worker that *returns* cancels the serve context (appkit's rule), but `Run` only
returns on `ctx.Done()`, so a clean shutdown is the sole exit — the collector never
takes the server down on its own.

**Rejected.**
- **Starting the goroutine inside `Handlers` instead of `Workers`.** A bare
  goroutine wouldn't be bound to the serve context and would leak past shutdown;
  the `Workers` seam exists precisely to tie background work to the serve
  lifecycle.
- **First sample deferred to the first tick.** The product promises the page isn't
  blank on boot; sampling immediately, before the ticker, is the direct way to
  honor it.
- **Aborting the sweep on the first source error.** One unreadable service must not
  blank every other series; each reading is independent, records 0 on failure, and
  the sweep continues.

**Verification.**
- R-FDAM-Q7U8 — `Run` records a sample for every series **immediately** at startup,
  before one interval elapses: with fake sources and a long interval, the store has
  ≥1 sample per series right after `Run` begins (unit test asserts the first sample
  exists without waiting `Interval`).
- R-FEIJ-3ZKX — `Run` samples again on each ticker tick: with a short injected
  interval, after N ticks the store holds N+1 samples per series (unit test).
- R-FFQF-HRBM — `Run` returns `nil` promptly when its context is cancelled (unit
  test cancels ctx and asserts `Run` returns nil without error).
- R-FGYB-VJ2B — `Collect` records `0` for a source whose reader returns an
  unexpected error **and** emits a log line for it, without aborting the other
  series (unit test injects a failing reader + a capturing logger, asserts the
  0 sample, the log, and that sibling series still got their values).
