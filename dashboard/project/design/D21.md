# Decision 21 — Sign-in remembers where you were headed: a validated same-site `return_to` on the web handshake

**Decision.** The web sign-in flow gains an optional **`return_to`** — a
same-site path captured when login begins and replayed when it completes (D22),
so a logged-out human bounced to `/login` from a session-gated page (D20) lands
back on that page after signing in instead of on the apex home. It rides the
**existing handshake row**, exactly as the MCP-origin context already does
(`oauthstate.MCPContext`), so the value is server-side state keyed by the
handshake id (which *is* Google's `state`), never a client-carried token and
never tamperable mid-round-trip.

Three pieces, all in `dashboard/`:

- **Schema.** A new forward-only migration (created with
  `bin/create-migration dashboard add_oauth_state_return_to`) adds a **nullable
  `return_to TEXT`** column to `oauth_state`, mirroring how `005_oauth_state_mcp.sql`
  added the MCP columns. Committed migrations stay frozen; this is a new one.

- **Store.** `oauthstate.Handshake` gains a `ReturnTo string` field, and the
  web-origin mint carries it. Rather than widen `Create`'s signature at every
  caller, `Create(ctx)` keeps minting an empty-`return_to` web handshake and a
  sibling `CreateWeb(ctx, returnTo string)` persists a validated value (the same
  shape as `CreateMCP`). `Consume` reads `return_to` back onto the returned
  `Handshake`. The value is round-tripped verbatim through SQLite.

- **Capture + validation at `GET /login`.** `handleLogin` reads the
  `return_to` query parameter and passes it through a **same-site validator**
  before persisting it; an invalid or absent value persists as **empty string**
  (which D22 treats as "default to `/`"). The validator admits only a clean local
  absolute path and rejects anything that could escape the origin:

  ```go
  // safeReturnTo returns p if it is a same-site absolute path, else "".
  // Reject: empty; not starting with "/"; protocol-relative ("//" or "/\");
  // any URL with a scheme or host; a backslash (browsers fold "\" to "/");
  // any control byte. No open-redirect target ever survives.
  func safeReturnTo(p string) string { … }
  ```

  A valid `return_to` (e.g. `/srv/sites/private/test07/`) is stored on the
  handshake; a hostile one (`//evil.com`, `https://evil.com/x`, `/\evil.com`,
  `https:/evil`, or a bare `evil.com` with no leading slash) yields `""`.

The `return_to` is **web-origin only**. The MCP-origin login path
(`handleAuthorize` → `CreateMCP` → `callbackMCP`) is untouched; it already
resumes via its own captured redirect URI.

**Rejected.**
- **Carrying `return_to` through Google in the OAuth `state`** — `state` is
  already the handshake id; smuggling a client-controlled target through the
  external round-trip is precisely the open-redirect surface the handshake row
  avoids. Server-side on the row is safe and matches the MCP context precedent.
- **A separate `return_to` cookie** — a second cookie with its own lifetime and
  clearing semantics, redundant with the handshake row that already scopes the
  login round-trip.
- **Restricting `return_to` to `/srv/…` only** — once off-site targets are
  forbidden, an apex-relative allowlist buys almost no extra safety and would
  wrongly reject legitimate apex destinations a general contract should serve
  (settled during grilling). Generic same-site is the contract.
- **Preserving multi-parameter query strings in the bounce** — nginx (module-free)
  cannot re-escape a variable, so `@login_bounce` embeds `$request_uri` raw
  (D20); a rare original URL with an `&` loses trailing params but the path — the
  thing that matters — survives. `safeReturnTo` still accepts a single-query path;
  it validates the path shape, not the absence of a query.

**Verification.**
- R-XLRL-ZHZT — the new migration adds a nullable `return_to` column to
  `oauth_state`, and a web handshake minted with a `return_to` **round-trips that
  exact value** through `CreateWeb` → `Consume` against a real temp
  `modernc.org/sqlite` migrated by the appkit runner (the substrate that actually
  holds the column); a plain `Create` handshake round-trips `return_to == ""`.
- R-XO7E-R1H7 — `GET /login?return_to=/srv/sites/private/test07/` persists that
  exact same-site path on the minted handshake (read back from the temp DB / via
  the completed round-trip), proving a valid destination is captured.
- R-XPFB-4T7W — `GET /login` with a hostile `return_to` — each of `//evil.com`,
  `https://evil.com/x`, `/\evil.com`, and a value not starting with `/` — persists
  **empty** `return_to` (never the off-site value), proving `safeReturnTo` rejects
  every open-redirect form so none can be replayed.
