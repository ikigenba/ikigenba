# Decision 26 тАФ Handshakes are provider-bound: `oauth_state` records which IdP a round-trip belongs to, and each callback refuses the other's

**Decision.** With two IdPs, a login handshake must complete at the callback of
the provider it was minted for. The handshake row gains a **`provider`**
column; each provider-specific callback verifies it. This closes the
cross-provider state-confusion hole where a handshake minted for one provider's
flow could be consumed by the other's callback (whose subsequent code exchange
runs against a different IdP than the round-trip that was started).

Three pieces, all in `dashboard/`:

- **Schema.** A new forward-only migration (created with
  `bin/create-migration dashboard add_oauth_state_provider`) adds a
  **`provider TEXT NOT NULL DEFAULT 'google'`** column to `oauth_state`,
  mirroring how `20260711182447_add_oauth_state_return_to.sql` extended the
  table. The default exists only so the migration applies to a table that may
  hold in-flight rows; every new mint writes the value explicitly.

- **Store.** `oauthstate` gains provider constants and threads the value:

  ```go
  const (
      ProviderGoogle = "google"
      ProviderGitHub = "github"
  )
  ```

  `Handshake` gains a `Provider string` field. `CreateWeb` becomes
  `CreateWeb(ctx, provider, returnTo string)` and `CreateMCP` becomes
  `CreateMCP(ctx, provider string, mcp MCPContext)`; both persist the value and
  return it on the row. `Consume` reads `provider` back onto the returned
  `Handshake`. The legacy no-argument `Create` is **deleted** тАФ every caller
  now states its provider (simplicity through discipline: no default-provider
  path survives).

- **Callback verification.** Each callback, immediately after
  `handshakes.Consume` succeeds, requires the handshake's `Provider` to be its
  own (`ProviderGoogle` in `handleCallback`, `ProviderGitHub` in the D28
  callback). A mismatch is treated exactly like a consume failure: warn log,
  `writeFederationReject(r, "", "provider_mismatch")`, `400 bad request`. The
  handshake is already consumed at that point тАФ single-use is preserved; the
  mismatched round-trip simply dies.

**Rejected.**
- **Inferring the provider from the callback path alone (no column).** The
  callback path names the provider the *IdP redirected to*, not the provider
  the *handshake was minted for*; without the column the two can disagree
  silently. The column makes the binding explicit and checkable.
- **Two separate handshake tables/stores per provider.** Same rows, same
  lifecycle, one discriminator column's worth of difference тАФ a second store
  would duplicate the TTL/binding-cookie/consume machinery for nothing.
- **Keeping `Create` as a Google-defaulting convenience.** An implicit default
  provider is exactly the kind of silent assumption this Decision exists to
  remove.

**Verification.**
- R-IBM4-4NJ8 тАФ the new migration adds the `provider` column, and a handshake
  minted with `CreateWeb(ctx, ProviderGitHub, "")` round-trips
  `Provider == "github"` through `Consume` against a real temp
  `modernc.org/sqlite` migrated by the appkit runner, while
  `CreateMCP(ctx, ProviderGoogle, mcp)` round-trips `Provider == "google"` тАФ
  both origins carry the binding.
- R-ICU0-IF9X тАФ `GET /oauth/google/callback` completing a handshake minted with
  provider `github` (state + correct binding cookie) is refused with **400**,
  a `provider_mismatch` federation-reject audit record, and no session cookie тАФ
  and the handshake is consumed (a replay at the GitHub callback also fails).
- R-IF9T-9YRB тАФ `GET /oauth/github/callback` completing a handshake minted with
  provider `google` is refused with **400**, a `provider_mismatch`
  federation-reject audit record, and no session cookie.
