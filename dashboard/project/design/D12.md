# Decision 12 — Metric source readers + service discovery

**Decision.** The four readings and the service list are small, pure-ish functions
in `internal/telemetry`, each taking an **injected path/root** so tests point them
at fixtures and temp trees while production defaults to the real OS paths. All are
Linux/amd64 (the only deploy target); on any non-Linux dev host the OS-coupled
ones simply read absent paths and return 0, which is the agreed "renders empty
locally" behavior.

```go
// System memory: parse MemAvailable + MemTotal (KiB in /proc/meminfo → bytes).
func readMemInfo(r io.Reader) (avail, total int64, err error)

// System disk: free + total bytes of the filesystem backing path (statfs).
// path defaults to "/opt" — where service data grows and where the box runs out.
func readDiskFree(path string) (free, total int64, err error)

// Per-service memory: cgroup-v2 current usage of <svc>.service.
// Reads <cgroupRoot>/system.slice/<svc>.service/memory.current.
// An absent path returns (0, nil) — a normal "unavailable" reading, not an error.
func readCgroupMem(cgroupRoot, svc string) (int64, error)

// Per-service disk: du-equivalent — sum of regular-file sizes under dir.
// An absent dir returns (0, nil).
func dirSize(dir string) (int64, error)

// Service list: the box's MCP services (dashboard excluded — it lacks MCP=true),
// reused verbatim from appkit/inventory so telemetry and the landing agree.
func services(manifestRoot string) ([]string, error) // names, sorted
```

`readCgroupMem` and `dirSize` distinguish **absent** (path does not exist → `0,
nil`, the expected unavailable case) from an **unexpected** error (permission,
malformed content → non-nil error); D13's collector records 0 either way but only
logs the latter. Discovery is `inventory.Read(manifestRoot)` mapped to names, so
the MCP-filter (which already drops the dashboard) is the single source of "which
services exist" across the app.

**Rejected.**
- **A single `/proc`-PID-matching memory backend that also works locally.** The
  operator chose one code path — cgroup only — accepting that local dev shows 0
  rather than maintaining a second `/proc`-RSS backend of near-zero value.
- **`syscall.Statfs` free = `Bfree`.** `Bavail` (blocks available to non-root) is
  the honest "free to a normal process" figure; `Bfree` overcounts by the
  root-reserved margin.
- **Shelling out to `du`/`systemctl`.** Reading `memory.current` and walking the
  tree in-process avoids a subprocess per service per minute and has no PATH/parse
  fragility.
- **A telemetry-specific manifest scan.** `inventory.Read` already globs the
  manifest root and encodes the dashboard-exclusion contract; a parallel scanner
  would be a second thing to keep in sync.

**Verification.**
- R-F4RC-1TND — `readMemInfo` parses `MemAvailable` and `MemTotal` from a
  `/proc/meminfo` fixture and converts KiB→bytes (a fixture with `MemAvailable: N
  kB` yields `N*1024`) (unit test with a fixture reader).
- R-F5Z8-FLE2 — `readDiskFree` returns the **real** `statfs` free and total for an
  existing path: called on a real temp directory it returns a non-zero total and a
  free ≤ total (smoke against the real filesystem — a value only the kernel can
  produce, driven for real, not stubbed).
- R-F774-TD4R — `readCgroupMem` reads `memory.current` for a service from a fixture
  cgroup tree at an injected root and returns its byte value (unit test writes
  `<root>/system.slice/crm.service/memory.current` and asserts the parsed value).
- R-F8F1-74VG — `readCgroupMem` returns `(0, nil)` — not an aborting error — when
  the service's cgroup path is absent under the injected root (unit test).
- R-F9MX-KWM5 — `dirSize` returns the summed size of all regular files under a temp
  directory tree, equal to the total bytes written across nested files (unit test).
- R-FAUT-YOCU — `dirSize` returns `(0, nil)` for a non-existent directory (unit
  test).
- R-FC2Q-CG3J — `services` returns the MCP service names from a temp manifest root
  and **excludes** the dashboard: a manifest root containing a `dashboard`
  manifest (no `MCP=true`) plus MCP services yields the MCP names only, dashboard
  absent (unit test against a staged temp manifest root).
