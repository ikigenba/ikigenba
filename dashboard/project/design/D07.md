# Decision 7 — Login composition: brand title, tagline, sign-in lede, rule, CTA, then a borderless etymology table

**Decision.** The login page (logged-out `GET /`, the `.signin-wall` branch of
`ui/html/index.html`) renders, top to bottom, exactly these elements, each reusing
an existing type style from elsewhere on the page rather than inventing a new one:

1. **`<h1 class="signin-title">Ikigenba</h1>`** — the brand name, in the
   **largest** type style already used on this page: the current `.signin-wall h1`
   treatment (`--text-h1-size` / `--text-h1-lh` / `--text-h1-weight`,
   `font-display`, tight letter-spacing). This is now the page's one `<h1>`.
2. **`<p class="signin-tagline">The place life actually happens.</p>`** — styled
   with the type treatment the old top-of-page wordmark used (`font-display`,
   `--text-h3-size` / `--text-h3-lh` / `--text-h3-weight`), inheriting the muted
   body color from `.signin-wall p` exactly as that wordmark did.
3. **`<p class="signin-lede">Sign in to access your services.</p>`** — styled
   with the small/muted lede treatment the old name-origin lede paragraph used
   (`--text-small-size` / `--text-small-lh`, muted color). The CTA below carries
   the call to action; this line is now supporting copy, not the heading.
4. **`<hr class="signin-rule">`** — a single hairline rule, the same
   `--border-width solid --color-border` treatment the old colophon's
   container border used, now standing alone immediately above the CTA instead of
   above the colophon.
5. **`<a href="/login" class="btn btn-primary btn-lg btn-accent-link">Sign in
   with Google</a>`** — unchanged: same classes, same href, same label.
6. **A borderless, left-aligned three-column etymology table**, block-centered on
   the page:

   | | | |
   |---|---|---|
   | **iki** | 生き | to live |
   | **genba** | 現場 | the place |

   Same content and same per-column text treatment as the retired two-part list
   (bold full-ink romaji, muted kanji, muted gloss) — reflowed from a two-line
   `dt`/`dd` pair per part into one **table row** per part, so the three pieces of
   each part align in three shared columns instead of each part owning its own
   baseline row. The table carries **no visible border or rule of its own** — cell
   padding is the only thing that reads as column structure — and the table block
   is centered on the page while its cell text stays left-aligned.
7. **`<p class="name-origin-say">pronounced <b>EE-kee-GEN-buh</b></p>`** — byte-
   for-byte the same element, class, and CSS as before, last in the block.

This **replaces** the prior composition (a `<h1>` reading "Sign in to access your
services.", a wordmark `<p>` above it, and a name-origin `<aside>` holding a lede
paragraph + a two-item `dt`/`dd` list below the CTA). The name/meaning statement
that used to be one merged sentence ("**ikigenba** — The place life actually
happens.") is now **two separate lines carrying the title and tagline roles**
directly, ahead of the sign-in lede and CTA rather than below them; the two-part
etymology explanation moves from a `dt`/`dd` list to a table but keeps the same
per-part content and per-column text styling.

  The `name-origin` `<aside>` is kept only as a grouping wrapper for the table +
  pronunciation foot (so `name-origin-say`'s existing `margin-top: var(--space-3)`
  keeps that pair visually tighter than the page's normal `--space-6` flex gap,
  exactly as it did against the old `dt`/`dd` list). It carries no border and no
  padding of its own now — the rule that used to live on this wrapper (`border-top`
  above the lede) has **moved** to the standalone `.signin-rule` above the CTA.

- **Styling.** `ui/static/app.css`'s `.signin-wall h1` rule is renamed/rescoped to
  `.signin-wall .signin-title` (same declarations, just no longer the general
  `.signin-wall h1` selector since `<h1>` now carries different content). The old
  inline-styled wordmark `<p>` is gone; its declarations become the named
  `.signin-wall .signin-tagline` rule. The old `.name-origin .name-origin-lede`
  rule's declarations become `.signin-wall .signin-lede`. The old
  `.name-origin`'s `border-top`/`padding-top` become the standalone
  `.signin-wall .signin-rule`. The old `.name-origin-parts`/`dt`/`dd` rules are

  stays semantic-token-only (no hard-coded color/space/radius), matching the

  `.signin-wall .signin-title`/`.signin-tagline`/`.signin-lede` are `(0,2,0)`
  descendant selectors, same specificity trick D7 used before, so each correctly
  out-specifies the general `.signin-wall p`/(none, for `h1`) rule without
  needing `!important`. `.signin-tagline` and `.signin-lede` deliberately do
  **not** set `color`, so both keep inheriting `.signin-wall p`'s muted color —
  exactly how the retired wordmark (which only overrode `font-family`/
  `font-size`/`font-weight` inline) rendered muted before.

This decision governs only the **login** page's composition. It does not touch
the landing/home page (D5), the profile page (D2–D4), routing (D1), the shared
banner (D10, which owns its own separate `.wordmark` in the logged-in banner —
untouched), the `/services` inventory endpoint, the install scripts, or any
schema. It is a pure template + CSS change confined to the `{{else}}` branch of
`index.html` and its associated `app.css` rules.

**Rejected.**
- **Keeping the name+meaning as one merged lede sentence** ("**ikigenba** — The
  place life actually happens.") **above the two-part list** — the new brief asks
  for the brand name and its meaning as two separate, larger lines *ahead of* the
  sign-in prompt and CTA, not as a small merged sentence trailing below them.
- **Leaving the sign-in sentence as the `<h1>`** — the new brief promotes the
  brand name to the page's largest, most prominent line and the CTA button itself
  now carries the call to action, so the sign-in sentence steps back to small
  supporting copy rather than the heading.
- **A `dt`/`dd` definition list for the etymology, reflowed with `display: grid`
  into three visual columns** — a definition list's `dt`/`dd` pairing is
  semantically two roles (term/definition), not three; forcing three columns out
  of it fights the semantics. A `<table>` names three columns natively and is the
  correct element for "align three columns of information."
- **A visible-bordered/ruled table** — the brief is explicit that the table is
  "invisible": no `border`, no zebra striping, no cell background — only padding
  large enough that the columns read as aligned, not run together.
- **Keeping the left `border-left` "spine" rule on the etymology block** — that
  rule existed to show the two parts were subordinate *children* of a lede line
  directly above them. That parent/child framing is gone now that the etymology
  sits well below the title/tagline/lede/CTA block; keeping the spine would be a
  vestigial rule pointing at nothing.
- **Leaving the horizontal rule above the etymology table (its old position)** —
  the brief explicitly relocates the rule to sit just above the "Sign in with
  Google" button.
- **Centering the table's cell text (matching the page's overall centered
  layout)** — the brief is explicit that cell text stays left-aligned even though
  the table itself is centered as a block on the page; this is the one
  intentional divergence from "only reuse existing styles."
- **Renaming the shared banner's `.wordmark` class to match** — the banner
  wordmark (D10, logged-in chrome) is a separate element on a separate page state
  and is untouched by this decision.
