# Decision 8 ‚Äî Eliminate the web-font FOUT (font-display + preload)

**Problem.** The rendered web surface (the logged-out login page, the logged-in
home/landing, and the profile page) flashes: it paints with the fallback system
font, then visibly **swaps** to the web fonts a moment later. Cause:

1. Every `@font-face` block in the embedded `ui/static/tokens.css` declares
   `font-display: swap`, which by design paints fallback first and swaps to the
   web font whenever it finishes loading ‚Äî i.e. *after* first paint.
2. Nothing preloads the fonts, so the fetch only starts after the stylesheet is
   parsed, guaranteeing the swap lands after first paint.

The dashboard is the **apex / `DEFAULT=true` app**, served at the origin root
(`int.ikigenba.com/`), **not** under a `/srv/<svc>/` mount. Its stylesheet links
(`<link rel="stylesheet" href="/static/tokens.css">` in `ui/html/index.html` and
`ui/html/profile.html`) and the `@font-face` `src: url("/static/fonts/X.woff2")`
are **origin-absolute and already correct**: they resolve to the dashboard's own
apex `/static/fonts/‚Ä¶`, which `staticHandler` serves from the embedded
`ui/static/fonts/`. So ‚Äî unlike the path-routed services ‚Äî the dashboard does
**not** need the "self-serve / make `src` mount-relative" change; the `src` is
already pointed at its own fonts. Only the swap behavior and the missing preload
need fixing.

**Decision.** Two coordinated changes; no Go logic, no migration, no new
dependency. The change set is confined to the embedded `ui/static/tokens.css`
(the `@font-face` rules) and the two head-bearing templates,
`ui/html/index.html` and `ui/html/profile.html`.

1. **`font-display: swap` ‚Üí `font-display: optional`** on all four `@font-face`
   blocks (Space Grotesk, IBM Plex Sans, IBM Plex Mono 400, IBM Plex Mono 500).
   `optional` gives the browser a short (~100 ms) block window and then commits
   to whatever is available **with no swap after first paint** ‚Äî this is what
   removes the jar. The design-token *values* (the `--font-*`/`--color-*`/
   `--space-*` variables) and the `src` URLs are unchanged.

2. **Preload the above-the-fold fonts** in the `<head>` of **both** head-bearing
   templates ‚Äî `ui/html/index.html` (serves the logged-out login page and the
   logged-in home/landing) and `ui/html/profile.html` (the session-gated profile
   page). Each `<head>` is independent (there is no shared `<head>` partial), so
   the preload links go in **both**. Add, alongside the existing
   `<link rel="stylesheet" href="/static/tokens.css">`:

   ```html
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="/static/fonts/space-grotesk.woff2">
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="/static/fonts/ibm-plex-sans.woff2">
   ```

   These are the **display** (`Space Grotesk`, headings) and **body**
   (`IBM Plex Sans`) families painted immediately. `crossorigin` is **mandatory**
   ‚Äî fonts are always fetched in CORS mode, so a preload without it double-fetches
   and the preload is wasted. The `href` is origin-absolute, **byte-identical to
   the existing `@font-face` `src` target**, so the preloaded bytes satisfy the
   `@font-face` request. With the font in cache inside the `optional` window,
   first paint uses the real font.

The mono family (`IBM Plex Mono`, used in labels and `code`/`pre`) is
**intentionally not preloaded**: it is minor/below the fold, and `optional`
guarantees no jar regardless ‚Äî at worst its first uncached paint uses the
fallback and the real font is cached for the next view. Preloading it would
compete for bandwidth with the two fonts that matter at first paint.

**Boundary.** D8 owns only the font-loading behavior of the rendered web surface
(`tokens.css` `@font-face` rules + the `<head>` preload links in `index.html`
and `profile.html`). It does not touch the design-token *values*, any other CSS,
any handler/route, or the page composition (D1‚ÄìD7) ‚Äî `staticHandler` (the apex
`GET /static/` route) already serves `static/fonts/*.woff2`. There is **no**
`src`-rewrite change here: the apex `src` is already origin-correct.

**Rejected.**
- **`font-display: block`.** FOIT ‚Äî text is invisible during the block period,
  then pops in. Trades a font swap for a text flash; still a flash. Rejected.
- **`font-display: fallback`.** Still permits a swap inside its (longer) swap
  window on a slow load. `optional` is the only value that forbids a post-paint
  swap outright. Rejected.
- **Preload only, keep `swap`.** If the network is slow enough that the font
  misses first paint, `swap` still swaps. Preload reduces the odds; `optional`
  removes the failure mode. Rejected as insufficient alone.
- **Rewriting the `@font-face` `src` to a relative `url('fonts/‚Ä¶')`.** This is
  the fix the path-routed services need (their origin-absolute `src` wrongly
  resolves to the apex under their mount). The dashboard *is* the apex, so its
  origin-absolute `src` already resolves to its own fonts ‚Äî rewriting it would be
  a no-op churn. Rejected as not applicable here.
- **A single shared `<head>` partial to hold the preload once.** `index.html`
  and `profile.html` each carry their own full `<head>`; introducing a partial is
  a refactor beyond this fix's boundary. Add the preload to both heads instead.
  Rejected.

**Verification** (httptest against the apex handler ‚Äî `testServer(t)` + `do(t,
srv, "GET", ‚Ä¶)` per `server_test.go` ‚Äî and the real `GET /static/` route; the
embedded `tokens.css` and the rendered `<head>` of each page are both reachable
from responses).

- R-P97M-GIJ1 ‚Äî **No `@font-face` declares `font-display: swap`; all use
  `optional`.** Reading the embedded `ui/static/tokens.css` (or `GET
  /static/tokens.css`), every `@font-face` block contains `font-display: optional`
  and the file contains **no** `font-display: swap` occurrence ‚Äî a block left as
  `swap` (the jar-causing value) fails this. *(asset read / served `GET
  /static/tokens.css`)*
- R-PAFI-UA9Q ‚Äî **The rendered `<head>` of every head-bearing page preloads the
  display and body fonts with `crossorigin`.** `GET /` (logged-out login and
  logged-in home) and `GET /profile` (with a live session) each return a `<head>`
  containing a `<link rel="preload" as="font" type="font/woff2" crossorigin ‚Ä¶>`
  for both `space-grotesk.woff2` and `ibm-plex-sans.woff2`, each `href`
  origin-absolute and identical to the `@font-face` `src` target
  (`/static/fonts/‚Ä¶`) ‚Äî a missing `crossorigin`, a missing family, a mismatched
  path, or a head left without the preload fails this. *(httptest; `/profile`
  uses a live session cookie)*
- R-PBNF-820F ‚Äî **The apex actually serves its own fonts at that path.**
  `GET /static/fonts/space-grotesk.woff2` and `GET /static/fonts/ibm-plex-sans.woff2`
  through the real `staticHandler` each return `200` with a woff2 content type
  (`font/woff2` or `application/octet-stream`) and a non-empty body ‚Äî exercising
  the real embedded-FS route the preload/`src` point at, not merely asserting a
  CSS string. A 404 here (font not embedded/served at this path) fails this.
  *(httptest against the real `GET /static/` route)*
