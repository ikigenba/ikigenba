# Decision 15 — Chart rendering: hero line charts + stacked-area charts

**Decision.** Charts are **server-rendered inline SVG**, built by pure functions of
a `telemetry.Snapshot` (in `internal/telemetry`, returning `template.HTML` the
templates embed). No client charting library, no external asset — CSP-clean. Four
charts:

- **Two hero line charts** — free memory, free disk. A single polyline over the
  series, y-axis mapped **`0 → total capacity`** (from the snapshot's
  `TotalMem`/`TotalDisk`), so a stable metric reads as a high, flat line with the
  headroom visible rather than auto-zoomed jitter. Each shows a large **current
  value** (the latest sample, binary-formatted) above the plot. Per-point value on
  hover is a native SVG `<title>` on the vertices — no crosshair JS (it would have
  to re-bind after every 60s `innerHTML` swap; native `<title>` survives the swap
  for free). Stroke uses the Carbon accent.
- **Two stacked-area charts** — accumulated per-service memory, accumulated
  per-service disk. Services are sorted by latest value descending; the **top N =
  7** get their own band, the remainder fold into a single **"Other"** band, so the
  stack height stays the exact accumulated total while the legend stays legible.
  Bands are light-fill areas (`fill-opacity` ~0.85) with a hairline top edge and a
  **2px surface gap** between bands, drawn from a committed **categorical palette**
  (see below). A **legend** names every band.

```go
// Pure builders over a snapshot; each returns SVG as template.HTML.
func heroChart(title string, samples []Sample, total int64) template.HTML   // y: 0→total
func stackedChart(title string, series map[string][]Sample, order []string) template.HTML

// Binary-unit formatter shared by labels/legend/titles.
func humanBytes(n int64) string // 1073741824 → "1.0 GiB"; 1048576 → "1.0 MiB"; 0 → "0 B"

// The band palette: a fixed, ordered, CVD-validated categorical set (8 hues for
// the 7 bands + "Other"). Validated once with the dataviz validator when authored
// (colorblind-safe in light mode); committed as constants — CI asserts the render
// USES it (distinct color per band + legend), not that it re-runs the validator.
var bandPalette = []string{ /* 8 committed hex values */ }
```

The palette deliberately steps **outside** the single-accent Carbon set (the
operator sanctioned more colors for these graphs); everything else — surfaces,
text ink, hairlines, spacing, fonts — stays on the Carbon tokens, light mode only.
A `0`/empty series renders as a flat baseline (the agreed "graphs always draw"
behavior).

**Rejected.**
- **A single 12-line chart, or auto-cycled hues.** Spaghetti, and it violates the
  never-cycle-hues rule; small-count bands + "Other" is the legible form.
- **Auto-scaled hero y-axis.** For a stable metric an auto-zoom turns noise into
  fake drama; `0 → total` is the honest "how much headroom" framing.
- **Unfilled stacked (cumulative) lines.** The operator chose filled bands; fills
  make each service's contribution readable as an area instead of a mental
  subtraction of adjacent lines.
- **A full JS crosshair/tooltip layer.** It would need re-binding after each 60s
  fragment swap; native SVG `<title>` gives per-point value on hover with zero JS.
- **Re-running `validate_palette.js` in CI.** The validator lives in a bundled
  skill, not the repo, so a `go test` shell-out to it isn't reproducible on
  identical repo state; the palette is validated at authoring time and committed,
  and the suite proves the render applies it.

**Verification.**
- R-FO9Q-65IH — `humanBytes` formats bytes in **binary** units:
  `1073741824 → "1.0 GiB"`, `1048576 → "1.0 MiB"`, `0 → "0 B"` (unit test on the
  boundaries).
- R-FPHM-JX96 — the hero chart maps its y-axis `0 → total capacity`: a sample equal
  to `total` plots at the top of the plot area and a `0` sample plots on the
  baseline (unit test asserts the computed y-coordinates for those two values).
- R-FQPI-XOZV — the hero chart renders the **latest** sample as a current-value
  label in binary units (unit test asserts the label text equals
  `humanBytes(lastSample)`).
- R-FRXF-BGQK — the stacked chart folds services beyond the top 7 into one "Other"
  band: given >7 services the rendered band set is the 7 largest plus an "Other"
  band (8 total); given ≤7 there is no "Other" band (unit test at both sides of the
  threshold).
- R-FT5B-P8H9 — the stacked chart is **accumulated**: at each x the top of the
  stack equals the sum of all services' values at that x (unit test on the computed
  band geometry).
- R-FUD8-307Y — the stacked chart renders a **legend** naming every band, including
  "Other" when present (unit test on the rendered SVG/markup).
- R-FVL4-GRYN — each visible band is drawn with a **distinct** color from the
  committed `bandPalette` — no two bands in one chart share a color (unit test).
