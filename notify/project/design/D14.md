# Decision 14 — Delete the chassis shims (`internal/db` wrappers) and true up the doctrine doc

**Decision.** Two cleanups that land the reference shape whole, mirroring crm's
shim deletion:

1. **`internal/db` shrinks to its legitimate content.** The `Open` and
   `Migrate` wrapper functions are deleted — appkit owns DB opening and the
   migration run (`Spec.Migrations: db.FS`). What remains is what genuinely
   must live in a notify package: the embedded migration set (`db.FS`;
   `go:embed` cannot cross modules) and the feed-offset guard tests
   (`migrations_feed_offset_test.go` asserts `002_feed_offset.sql` stays
   byte-identical to `consumer.SchemaSQL`; the load test stays). Test
   harnesses that called the wrappers call appkit's db package directly — no
   test *assertions* change, only harness plumbing.
2. **`notify/AGENTS.md` (and its `CLAUDE.md` symlink — one file, edit
   `AGENTS.md`) states the converted truth**, per the suite doctrine (docs
   state current truth; purge what the change falsified; no archaeology). The
   falsified claims: the consumer loop "run as an appkit `Worker`", the
   `CRM_FEED_URL`-by-name and `NOTIFY_FROM` config story, the
   `Consumes:["crm"]` manifest example, the `internal/server`/`internal/logging`/
   `internal/ids` layout entries (already stale), the embedded-assets landing
   description, and the `internal/mcp` transport description. Replaced by the
   current truth: the `Spec.Consumers` declaration (chassis resolves
   `NOTIFY_<SRC>_FEED_URL`/`NOTIFY_<SRC>_FROM`), `CONSUMES=crm,prompts`, the
   `share/www` web surface through `Spec.WWW`, and `internal/mcp` as the
   `send` tool table over the chassis transport.

**Rejected.**
- **Keeping the wrappers for import brevity** — a per-service shim that can
  drift, for two saved lines per call site; crm's D14 already made this call.
- **Deleting `internal/db` entirely** — the migrations embed must live in a
  notify package and the feed-offset byte-equality guard is genuinely
  app-side; the package shrinks to its legitimate content instead.
- **Leaving `AGENTS.md` for a later pass** — the file would assert wiring that
  no longer exists in the same tree that changed it; docs move with the code.

**Verification.** Structural — no `R-XXXX-XXXX` ids (shim deletion + doc
truth; the behaviors are already pinned by the existing domain/MCP suites and
the D11–D13 ids). Realized by a plan phase whose done bar is:

- `grep -n "func Open\|func Migrate" notify/internal/db/db.go` returns no
  matches, while `grep -n "go:embed migrations" notify/internal/db/db.go`
  still matches and `notify/internal/db/migrations_feed_offset_test.go` still
  exists.
- `grep -n "CRM_FEED_URL\|PROMPTS_FEED_URL\|NOTIFY_FROM\b" notify/AGENTS.md`
  returns no matches, and `grep -c "Consumers" notify/AGENTS.md` is ≥ 1.
- The suite stays green (design Conventions commands) with no assertion
  changes.
