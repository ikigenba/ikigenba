# Decision 8 — Self-serve the landing page's fonts and eliminate the FOUT

**Problem.** The landing page flashes: it paints with the fallback system font,
then visibly **swaps** to the web fonts once they load. Worse, the swap is driven
by the **dashboard's** assets, not notify's own — which also violates the
suite-wide requirement that every service be 100% independent.

The cause is a chain of apex-absolute references:

1. `landing.html` links the stylesheet origin-absolute:
   `<link rel="stylesheet" href="/static/tokens.css">`. A leading-slash URL
   resolves against the site root, so a browser on `/srv/notify/` requests
   `https://<host>/static/tokens.css` — the `/srv/notify/` mount is gone. nginx
   routes apex `/static/` to the **dashboard** (`MOUNT=/`), so the page renders
   with the **dashboard's** tokens.css. notify's own embedded copy (served at
   `/srv/notify/static/tokens.css` by `StaticHandler`, wired at `GET /static/` in
   `cmd/notify/main.go`) is never requested.
2. The `@font-face` `src` in notify's `tokens.css` is likewise origin-absolute
   (`url('/static/fonts/X.woff2')`) and resolves to the apex.
3. Every `@font-face` declares `font-display: swap`, which paints fallback and
   swaps to the web font after load — i.e. after first paint.
4. Nothing preloads the fonts.

Because the stylesheet itself loads from the apex, fixing only notify's embedded
`tokens.css` would be inert — the browser never fetches it. The link must point
at notify's own copy first.

**Decision.** Make the landing page load notify's **own** embedded CSS and fonts,
swap-free. Four coordinated changes; no new dependency, no Go logic beyond the
existing `StaticHandler` (already wired at `GET /static/`). The change set spans
`internal/web/landing.html`, `internal/web/static/tokens.css`, and
`etc/nginx.conf`.

1. **`landing.html`: relativize the stylesheet link.**
   `href="/static/tokens.css"` → `href="static/tokens.css"`. Document-relative,
   it resolves against the mount-root document URL (`/srv/notify/`, canonical
   trailing slash) to `/srv/notify/static/tokens.css` — notify's own copy. (No
   `<base href>` is introduced, and the handler's unused `AssetPath` field is
   **not** threaded into the template — see *Rejected*; the document-relative form
   needs neither.)

2. **`tokens.css`: self-serve the fonts and stop the swap.** On all four
   `@font-face` blocks (Space Grotesk, IBM Plex Sans, and the two IBM Plex Mono
   weights): `font-display: swap` → `font-display: optional` (`optional` forbids a
   post-first-paint swap — this removes the jar), and
   `src: url('/static/fonts/X.woff2')` → `url('fonts/X.woff2')` (relative to the
   now-self-served stylesheet → `/srv/notify/static/fonts/X.woff2`).

3. **`landing.html`: preload the above-the-fold fonts** in `<head>`, beside the
   stylesheet link:

   ```html
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/space-grotesk.woff2">
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/ibm-plex-sans.woff2">
   ```

   The **display** (`Space Grotesk`) and **body** (`IBM Plex Sans`) families,
   painted immediately. `crossorigin` is **mandatory** (fonts fetch in CORS mode;
   without it the preload double-fetches and is wasted). The document-relative
   `href` resolves to the same `/srv/notify/static/fonts/…` URL as the new
   `@font-face` `src`, so the preloaded bytes satisfy the `@font-face` request and
   first paint uses the real font. The mono family is **not** preloaded (minor /
   below the fold; `optional` already prevents its jar).

4. **`etc/nginx.conf`: session-gate notify's own static path.** Today only the
   exact landing root (`= /srv/notify/`) is session-gated; everything else under
   the mount — including `/srv/notify/static/…` — falls to the bearer-gated prefix
   (`location /srv/notify/` → `auth_request /_authn`). A browser carries the
   dashboard **session cookie**, not a bearer token, so it would get **401** on
   the self-served CSS/fonts and the page would be unstyled. Add a session-gated
   static location mirroring the dashboard's own and wiki's precedent:

   ```nginx
   location /srv/notify/static/ {
       auth_request /_session-authn;
       proxy_pass http://127.0.0.1:3201/static/;
       proxy_set_header Host $host;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_http_version 1.1;
   }
   ```

   The prefix `/srv/notify/static/` is replaced by upstream `/static/`, which
   notify's `StaticHandler` serves. The exact `= /srv/notify/` landing location,
   the bearer-gated `/srv/notify/` prefix (MCP, PRM bootstrap), and the
   `@notify_authn_500` rate-limit fallback are unchanged. (notify is a *consumer*
   and serves **no** `/feed`, so there is no feed-denial location to preserve.)

**Supersession.** D8 corrects the apex-borrow that one shipped id pins in its
buggy form, plus one untagged guard:
- **D3's R-ASST-5L2V** (`TestLandingReferencesOnlyLocalStaticAssets`) asserts the
  rendered body contains `/static/tokens.css`. Its intent ("references notify's
  *own* /static/ asset, no cross-service/dashboard URL") is fulfilled — only the
  path form changes to the service-relative truth. The build updates that
  assertion to `href="static/tokens.css"` (and keeps the cross-service/external
  forbidden-URL checks); the id is retained.
- The untagged `TestTokensCSSUsesEmbeddedNotifyFontURLs` asserts `tokens.css`
  contains `/static/fonts/X.woff2` for each of the four fonts. The build updates
  those assertions to the `fonts/X.woff2` form; it carries no `R-` id, so nothing
  is re-pinned.
- **D3's R-ASST-3K9T / R-ASST-7M4W** (StaticHandler serves `/static/tokens.css`
  and `/static/fonts/*.woff2` with correct content types) are **unchanged** and
  remain the real-substrate proof that notify serves its own assets. The untagged
  `TestEmbeddedAssetsDoNotReferenceExternalRuntimeOrigins` also stays valid —
  relativizing introduces no `http://`/`https://`/`dashboard` origin.

**Boundary.** D8 owns the landing page's asset *loading* (the link, the
`@font-face` rules, the preload) and the nginx static gate. It does not change the
design-token values, the landing markup/copy (D6), or the landing handler/route
(D1/D2). It corrects D3's reasoning, which assumed an origin-absolute `/static/…`
is rewritten by nginx back to `/srv/notify/static/…` — nginx strips inbound, it
does not rewrite outbound CSS, so the apex-borrow was real.

**Rejected.**
- **`font-display: block` / `fallback`.** `block` is FOIT (invisible text then
  pop-in); `fallback` still permits a late swap in its window. `optional` is the
  only value that forbids a post-paint swap. Rejected.
- **Keep the apex-absolute link, fix only the dashboard.** Would fix the flash but
  leaves notify dependent on the dashboard serving its CSS — violating the
  independence requirement. Rejected.
- **Thread the handler's `AssetPath` field into the template.** `LandingHandler`
  already computes an `AssetPath` (`/srv/notify/static/tokens.css`), but it is
  **unused** — the template hardcodes the link, and the handler is invoked with
  only `(service, version)`. Wiring `AssetPath` through would re-introduce an
  origin-absolute, mount-pinned link and need a template-data change; the
  document-relative link resolves correctly at the canonical trailing-slash mount
  root with no Go change. The unused field is **not** the chosen mechanism.
  Rejected.
- **Introduce `<base href="{{.Mount}}">`.** Would require threading `Mount` into
  the landing template's data; the document-relative link/preload need no base and
  resolve correctly at the canonical trailing-slash mount root. Rejected.
- **Origin-absolute preload `href` to match the old `src`.** Would re-pin the
  apex-borrow; the whole point is to self-serve. Rejected.

**Verification** (httptest against `LandingHandler` and `StaticHandler`, and a
fragment-grep against `etc/nginx.conf`, mirroring the existing R-NGNX tests).

- R-8JS0-IQDX — **No `@font-face` declares `font-display: swap`; all use
  `optional`.** notify's embedded `tokens.css` contains `font-display: optional`
  in every `@font-face` block and **no** `font-display: swap` occurrence — a block
  left as `swap` (the jar-causing value) fails this. *(served `GET /static/tokens.css`)*
- R-8KZW-WI4M — **Font `src` is service-relative, not origin-absolute.** notify's
  `tokens.css` contains **no** `url('/static/fonts/` occurrence and each
  `@font-face` `src` uses `url('fonts/…woff2')` — so it resolves to notify's own
  `/srv/notify/static/fonts/…`. *(served `GET /static/tokens.css`)*
- R-8M7T-A9VB — **The landing page links its own tokens.css service-relative.**
  `GET /` renders a `<head>` containing `href="static/tokens.css"` and **no**
  origin-absolute `href="/static/tokens.css"` — so the browser fetches notify's
  own copy, not the apex. *(httptest `LandingHandler`)*
- R-8NFP-O1M0 — **The rendered `<head>` preloads the display and body fonts with
  `crossorigin`.** `GET /` returns a `<head>` containing a
  `<link rel="preload" as="font" type="font/woff2" crossorigin …>` for both
  `space-grotesk.woff2` and `ibm-plex-sans.woff2`, each `href` document-relative
  (`static/fonts/…`) and resolving to the same path as the `@font-face` `src`. A
  missing `crossorigin`, a missing family, or an origin-absolute href fails this.
  *(httptest `LandingHandler`)*
- R-8ONM-1TCP — **nginx session-gates notify's own static path.** `etc/nginx.conf`
  contains a `location /srv/notify/static/` block whose body carries
  `auth_request /_session-authn;` and `proxy_pass http://127.0.0.1:3201/static/;`
  — so a browser session can fetch the self-served CSS/fonts. A static path left
  bearer-gated (only the inherited `/srv/notify/` prefix) fails this. *(fragment grep
  against `etc/nginx.conf`)*
