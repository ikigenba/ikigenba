# Decision 12 — Web surface from `share/www` through the chassis (de-embed)

**Problem.** notify's landing page and static assets were embedded in the
binary (`//go:embed` in `notify/internal/web/`), like every service's. The
suite is reverting that: web assets are release artifacts that belong on disk —
visible and diffable in the versioned release tree, editable in dev without a
rebuild, served by one chassis mechanism instead of twelve drifting copies. crm
proved the conversion; notify is the second service converted, bundled with its
consumer-seam pilot (D11). This Decision is the service-side half of appkit's
chassis design (its D5 www-root resolution, D6 `appkit/web`, D7 `Spec.WWW` —
consumed here as a fixed external contract).

**Decision.** notify serves its web surface from the on-disk www root.

1. **The files move.** `notify/internal/web/landing.html` →
   `notify/share/www/landing.html`; `notify/internal/web/static/` →
   `notify/share/www/static/` (same bytes: `tokens.css`, the four woff2
   fonts). `bin/ship` already bundles `notify/share/` into the release
   `share/<version>` tier; on the box the chassis resolves the root to
   `share/current/www`, in dev to `./share/www` or the `NOTIFY_WWW_PATH`
   override.
2. **The Spec opts in.** `cmd/notify/main.go` sets `WWW: true`. The chassis
   loads the site at serve (failing startup loudly if the root is missing),
   auto-mounts `GET /static/`, and exposes the loaded site as `rt.WWW()`.
3. **The landing route renders through the chassis.** The `GET /{$}` mount in
   `Handlers` uses the few-line handler over
   `rt.WWW().Render("landing.html", …)` (D1/D2 carry the exact shape). The
   service-side `GET /static/` mount is **deleted** — the chassis owns it.
4. **`notify/internal/web/` is deleted** — handler, embed shim, template,
   assets, and its tests. The landing/asset tests move to the composition-root
   package (`cmd/notify`), loading the repo-real `share/www` tree via
   `appkit/web` (a relative path from the package dir), so they exercise the
   exact files that ship. The nginx fragment tests (`nginx_test.go`) relocate
   to `cmd/notify` unchanged in their assertions (D10's drift guard keeps its
   registry-derived expectations).
5. **Dev wiring (boundary-crossing).** `bin/start`'s `launch_notify` exports
   `NOTIFY_WWW_PATH="$repo/notify/share/www"` beside its existing exports, so
   a repo-root-launched dev binary finds the source tree. This one line
   crosses the `notify/` boundary and is verified by the live `bin/start`
   smoke, not the Go suite.
6. **Nothing else moves.** The nginx fragment (D4/D8), the landing markup and
   tokens (D6/D7/D8), the MCP surface, and the migrations are untouched. The
   embedded migrations (`internal/db`) stay embedded — they are not web
   assets.

**Rejected.**
- **Keeping the embed** — the suite-wide reversion this work exists to
  complete; embedded assets are invisible on the box and cost rebuilds in dev.
- **`state/www` as the root** — that tree is the sites service's nginx-served
  user content, backed up with `state/`; notify's UI assets are release
  artifacts and live in the shipped `share` tier (appkit D5 records the full
  argument).
- **Keeping a thin `internal/web` package** — with the mechanism in
  `appkit/web` there is nothing left but a handler closure; a package holding
  one closure is ceremony. The composition root is the right home for
  composition.
- **Serving assets from both embed and disk during a transition** — two
  sources of truth; the conversion is atomic within one service.

**Verification** (over the repo-real `notify/share/www` tree; D1's R-LAND-\*,
D3's R-ASST-\*, and D8's R-8\*-\* ids continue to pin the page/asset behaviors
through the new substrate).
- R-4FW1-V9QL — an `appkit/web` Site loaded from `notify/share/www` (the actual
  shipped tree, resolved relative to the test package) renders `landing.html`
  with an injected service and version visible in the output — proving the
  shipped template, not an embedded copy, is the page source.
- R-4H3Y-91HA — the chassis static mount over the same tree serves
  `GET /static/tokens.css` (`200`, `text/css`) and
  `GET /static/fonts/space-grotesk.woff2` (`200`, `font/woff2`) with **no**
  notify-side static handler registered — proving the assets ship on disk and
  the chassis serves them.
