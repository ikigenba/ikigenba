# Decision 25 — Structured MCP adoption (structuredContent, output schemas, closed error codes)

**Problem.** sites's domain tools build their results with the old
`appkitmcp.JSONResult(v)` (a struct marshalled into a single text block) and their
errors with a one-argument `appkitmcp.ErrorResult(string)` carrying a
service-local, open-ended `code` string (`invalid_slug`, `already_exists`,
`path_escapes_working_dir`, `sync_unconfigured`, …) stuffed into a JSON blob. The
suite has since settled a single machine-and-agent verb surface
(`docs/structured-mcp-design.md`): every domain verb returns
`structuredContent` alongside a mirrored text block, declares an `outputSchema`,
and reports failures with a **closed** error vocabulary so a deterministic caller
branches on a code, not on message text. appkit has already cut over — `JSONResult`
is **deleted**, `StructuredResult(v) (map[string]any, error)` and
`ErrorResult(code ErrorCode, msg string)` are the new helpers, and `Tool` carries
an `OutputSchema map[string]any` surfaced by `tools/list` when non-nil. sites does
not currently compile against that appkit; this Decision brings its `internal/mcp`
surface into conformance. It formalizes the JSON sites already emits — **no result
is reshaped**.

This Decision cites `docs/structured-mcp-design.md` (the suite contract — the
result contract, the closed error vocabulary, the prose exceptions) and does not
restate it. sites is **not** an event-plane participant and has **no** hand-copied
loopback guard (its loopback-private surface is nginx's concern; the sole
`X-Owner-Email` reference in `cmd/sites/main.go` is a package doc comment, not a
guard predicate), so the shared-guard clause of the suite contract does not apply
to sites — this Decision is result-shape and error-code only.

**Decision.** Convert `internal/mcp` (`tools.go`, `files.go`, `sync.go`) to the
structured surface:

1. **`JSONResult` → `StructuredResult` on every domain success result.** Each of
   the success sites — `create`, `list`, `delete` (both the found and the
   already-absent idempotent return), `mkdir`, `set_visibility`, `rename`,
   `sync`, `file_write`, `file_edit`, `file_glob`, `file_grep`, `file_list` — returns
   `appkitmcp.StructuredResult(v)`. Every one is the tail return of a handler whose
   signature is already `(map[string]any, error)`, so the swap is a drop-in that
   **honestly propagates** `StructuredResult`'s new error return (`return
   appkitmcp.StructuredResult(v)` — a marshal failure surfaces as a JSON-RPC
   internal error; it is never swallowed). The emitted JSON objects are unchanged;
   `StructuredResult` additionally attaches the machine `structuredContent` and the
   mirrored text block from the same value.

2. **Output schemas per structured domain tool.** Each of the twelve structured
   tools declares a hand-authored `OutputSchema` (a `map[string]any` JSON-schema
   literal, `type`/`properties`/`required`) mirroring the JSON its handler already
   emits verbatim. A small `descOut(name, description string, in, out map[string]any,
   handler …) mcp.Tool` variant sets `OutputSchema` (the existing `desc(...)` stays
   for prose tools, which leave it nil); output-object literals are built with the
   existing `obj(...)`/`descTyp(...)` helpers, house-style with appkit's
   `objectSchema`. The declared shapes (each property `required` unless noted):

   | tool | output object |
   |---|---|
   | `create`, `set_visibility`, `rename` | `{slug:string, name:string, visibility:string, owner_id:string, owner_email:string, url:string, created_at:string, updated_at:string}` (the `renderSite` projection — `slug` is the URL handle, `name` the display label, D15/D20; `visibility` is the three-value enum string; the retired `created_by` is the pair `owner_id`+`owner_email`, `docs/owner-id-design.md`) |
   | `list` | `{sites: array<site-object>}` |
   | `delete` | `{deleted:string}` |
   | `mkdir` | `{created:string, site:string}` |
   | `sync` | `{slug:string, written:integer, deleted:integer}` |
   | `file_write` | `{written:string, site:string, appended:boolean}` |
   | `file_edit` | `{edited:string, site:string, replaced:integer}` |
   | `file_glob` | `{site:string, matches: array<string>}` |
   | `file_grep` | `{site:string, matches: array<{path:string, line:integer, text:string}>}` |
   | `file_list` | `{site:string, files: array<{path:string, size:integer, md5:string}>}` |

   The schema is the machine contract, not a runtime validator — appkit surfaces it
   in `tools/list` and does not check responses against it (suite non-goal); the
   test suite is the conformance check.

3. **Prose exceptions — no output schema.** `guide` (a documentation tool — it
   returns the embedded usage guide as `TextResult`) and `file_read` (a
   **raw-content read** — it returns the requested file's bytes as `TextResult`,
   for a reader, exactly the raw-content-read exception the suite contract names)
   keep `TextResult` and declare **no** `OutputSchema`. They are the only two
   prose exceptions; every other domain tool is structured.

4. **Closed error vocabulary on every failure.** `errResultMsg` becomes a thin
   passthrough `return appkitmcp.ErrorResult(code, msg)` taking an
   `appkitmcp.ErrorCode` and a **plain human message** (the old JSON-blob-as-message
   wrapping is dropped; the human detail stays in `msg`, the machine code is the
   closed `structuredContent.code`). `errResult(err)` maps sentinels to closed
   codes. The full mapping (every current call site → its closed code):

   | failure | old service code | closed `ErrorCode` |
   |---|---|---|
   | `sites.ErrInvalidSlug` / `sites.ErrReservedName` / `sites.ErrInvalidName` | `invalid_slug` / `reserved_name` / (new) | `ErrValidation` |
   | `sites.ErrExists` | `already_exists` | `ErrConflict` |
   | `sites.ErrNotFound` | `not_found` | `ErrNotFound` |
   | unclassified domain error (`errResult` default) | `error` | `ErrInternal` |
   | missing `site` / `source_path` argument | `invalid_site` / `validation` | `ErrValidation` |
   | derived-slug / explicit-slug validation in `sync` | `validation` | `ErrValidation` |
   | confinement rejection (`files.ErrEscapes`, any file tool + `mkdir`) | `path_escapes_working_dir` | `ErrValidation` |
   | dir op failure — `create_site_dir`, `remove_site_dir`, `move_site_dir`, `mkdir` | those strings | `ErrInternal` |
   | file op failure — `walk`, `write`, `read`, `edit`, `glob`, `grep` | those strings | `ErrInternal` |
   | dropbox mirror unconfigured (`sync_unconfigured`) | `sync_unconfigured` | `ErrSourceUnavailable` |
   | dropbox mirror `List` / `Fetch` failure | `list_upstream` / `fetch_upstream` | `ErrSourceUnavailable` |
   | working-tree walk / reconcile failure | `walk_working` / `reconcile` | `ErrInternal` |

   The dropbox mirror is an external source; its unavailability (unwired,
   enumerate failure, or fetch failure) is `source_unavailable`, the one code that
   lets a script distinguish "the upstream is down" from "you asked wrong"
   (`validation`) or "sites broke" (`internal`). `too_large` is unused — sites has
   no size-bounded verb. No `structuredContent.code` outside the closed set
   survives; no legacy fine-grained string remains as a code.

5. **No `JSONResult` reference remains** in `internal/` or `cmd/` (tests included —
   the new tests assert on `StructuredResult` output, they do not mention the old
   token).

**Illustrative signatures.**

```go
// descOut declares a structured tool: same as desc, plus the OutputSchema literal.
func descOut(name, description string, in, out map[string]any,
    handler func(context.Context, json.RawMessage, server.Identity) (map[string]any, error)) appkitmcp.Tool

// errResultMsg passes the closed code + human message straight to the chassis.
func errResultMsg(code appkitmcp.ErrorCode, msg string) map[string]any {
    return appkitmcp.ErrorResult(code, msg)
}

// errResult maps a domain sentinel to its closed code.
func errResult(err error) map[string]any {
    switch {
    case errors.Is(err, sites.ErrInvalidSlug), errors.Is(err, sites.ErrReservedName):
        return errResultMsg(appkitmcp.ErrValidation, err.Error())
    case errors.Is(err, sites.ErrExists):
        return errResultMsg(appkitmcp.ErrConflict, err.Error())
    case errors.Is(err, sites.ErrNotFound):
        return errResultMsg(appkitmcp.ErrNotFound, err.Error())
    default:
        return errResultMsg(appkitmcp.ErrInternal, err.Error())
    }
}
```

**Rejected.**
- **A second, mirrored loopback domain API for machine callers.** The suite
  contract rejects it suite-wide; `structuredContent` on the one MCP surface is the
  machine channel. sites adds no route.
- **Keeping the open-ended service-local codes** (`invalid_slug`,
  `already_exists`, `path_escapes_working_dir`, …) as the machine code. They are
  useful *detail* and survive in the human `msg`, but a machine caller must branch
  on a closed set; an open vocabulary is exactly what the contract removes.
- **Collapsing `guide`/`file_read` into structured results.** Both are prose for a
  reader (a usage guide, raw file bytes); a schema over free text is meaningless and
  the contract names them exceptions.
- **A runtime schema validator in the handler.** The suite makes conformance the
  test suite's job, not the transport's; declaring the schema and testing the
  emitted shape against it is the contract.

**Verification** (driven through the assembled `appkit/mcp` handler with
`tools/call`/`tools/list` over a temp DB, a temp `SITES_ROOT`, and a fake
`sites.MirrorClient` — the same real handler seam the existing `internal/mcp`
tests use; httptest against the real handler is the falsifying substrate).

- R-CW5E-T20N — **A domain success result carries both channels.** A `tools/call
  create` (valid slug) returns a result whose `structuredContent` is the site
  object (`slug`/`name`/`visibility`/`owner_id`/`owner_email`/`url`/`created_at`/`updated_at`)
  **and** whose text content block is the JSON marshalling of that same object —
  not a text-only result. Falsifiable: a result with a text block but no
  `structuredContent`, or a `structuredContent` that differs from the mirrored
  text, fails it (the JSONResult→StructuredResult swap).
- R-CXDB-6TRC — **Every structured domain tool declares an `outputSchema`; the two
  prose tools declare none.** Table-driven over `tools/list`: each structured
  tool (`create`, `list`, `delete`, `mkdir`, `set_visibility`, `sync`,
  `file_write`, `file_edit`, `file_glob`, `file_grep`, `file_list`) has an
  `outputSchema` with `type:"object"`, and `guide` and `file_read` have **no**
  `outputSchema` key. Falsifiable: a missing schema on a structured tool, or a
  present schema on `guide`/`file_read`, fails it. (`rename`'s declaration and
  conformance are pinned by R-0A69-6H6K below.)
- R-CYL7-KLI1 — **Each structured tool's emitted object matches its declared
  schema.** For every structured tool in the table above, a successful `tools/call`
  produces a `structuredContent` whose keys include exactly the properties its
  declared `outputSchema` marks `required` (schema formalizes the real output, not
  an approximation). Falsifiable: a tool whose emitted object omits a required
  schema key, or carries a key absent from its schema, fails it.
- R-0A69-6H6K — **`rename` is a structured tool in full conformance.**
  `tools/list` shows `rename` with an `outputSchema` of `type:"object"` whose
  `required` names the `renderSite` projection (`slug`, `name`, `visibility`,
  `owner_id`, `owner_email`, `url`, `created_at`, `updated_at`); a successful
  `tools/call rename` emits a `structuredContent` matching exactly those keys
  with the mirrored text block; and its failures speak the closed vocabulary —
  an invalid name is `validation`, a missing site is `not_found`, with no
  legacy code string. Falsifiable: a schema-less `rename`, an emitted object
  missing `slug` or `name`, or an out-of-vocabulary error code fails it.
- R-CZT3-YD8Q — **Closed error codes for the domain-sentinel and filesystem
  paths.** Table-driven: invalid/reserved slug and a missing required argument →
  `validation`; an already-existing slug → `conflict`; a missing site → `not_found`;
  an internal filesystem failure (a dir/file op error) → `internal`; and **no**
  legacy fine-grained code string (`invalid_slug`, `already_exists`, `not_found` as
  a service code, `create_site_dir`, `walk`, …) appears as `structuredContent.code`
  — every error code is a member of the closed vocabulary. Falsifiable: any of
  these paths returning a code outside `{validation, not_found, conflict, too_large,
  source_unavailable, internal}`, or the wrong member, fails it. (Confinement and
  mirror mappings are pinned by their own ids below.)
- R-D110-C4ZF — **Confinement rejection is `validation` across all file tools.** A
  `tools/call` to any file tool (and `mkdir`) with a `..`-escaping or outside-root
  path returns `isError` with `structuredContent.code == "validation"` and a
  message identifying the path escape (was `path_escapes_working_dir`). Driven for
  at least `file_read`, `file_write`, and `mkdir`. Falsifiable: a legacy
  `path_escapes_working_dir` code, a code other than `validation`, or a
  non-error result fails it. (Supersedes the *code-value* half of D11's
  R-0JAJ-OIF8, which now pins confinement **detection** only.)
- R-D28W-PWQ4 — **Dropbox mirror unavailability is `source_unavailable`.** With a
  fake `MirrorClient`, `sync` returns `structuredContent.code == "source_unavailable"`
  when the mirror is unconfigured (nil client), when `List` returns an error, and
  when `Fetch` returns an error. Falsifiable: any of these paths returning
  `validation`/`internal` or a legacy `list_upstream`/`fetch_upstream`/
  `sync_unconfigured` code fails it. (The mirror is sites's one external contract;
  the fake client that *errors* is what makes each unavailable-path branch fire.)
- R-D3GT-3OGT — **No `JSONResult` token remains.** `grep -rn 'JSONResult' internal
  cmd --include='*.go'` (tests included — the new tests assert on `StructuredResult`
  and do not mention the old token) returns empty. Falsifiable: any surviving
  `JSONResult` reference fails it. *(source-tree scan)*
