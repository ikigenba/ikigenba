# Decision 20 ŌĆö MCP surface: the visibility enum, explicit at create, and the full transition matrix

**Decision.** The `ikigenba_sites_*` tool table speaks the three-value
visibility vocabulary (D15) end to end. The governing invariant, stated once and
enforced at every entry point: **public and private sites carry a name the
caller chose; unlisted sites carry a generated token name (D27) the caller never
chooses.** A site is created at an explicitly stated visibility, its files are
edited *live* in its served directory, its visibility (and, when the invariant
demands it, its name) is changed with one tool, and it is deleted.

**`create(name?, visibility)`** ŌĆö `visibility` is **required** (one of
`"public"`, `"private"`, `"unlisted"`; anything else ŌĆö including omission ŌĆö is a
`validation` error before any row or directory exists). This is an agent-facing
API: an explicit argument costs nothing and removes the silent default entirely.
`name` is conditional on the invariant:

- `visibility: "public" | "private"` ŌĆö `name` **required**, validated by the
  existing slug grammar and reserved-name guard.
- `visibility: "unlisted"` ŌĆö `name` **forbidden** (a supplied name is a
  `validation` error: the name is the credential; letting a caller pick one
  would let them pick a guessable credential). The handler generates the name
  with the injectable token source (default `sites.NewToken`, D27) and, on the
  astronomically-rare `ErrExists` collision, retries once with a fresh token.

The creator is recorded from the request `Identity` exactly as before (stable
`Identity.OwnerID` ŌåÆ `owner_id`, `Identity.OwnerEmail` ŌåÆ `owner_email`; neither
is a tool argument). Row and directory are born in **one step** at the stated
visibility: `store.Create(ctx, name, id.OwnerID, id.OwnerEmail, v)` +
`os.MkdirAll(SiteDir(v, name))` ŌĆö for unlisted that directory is under the
**public** parent (D16). No intermediate state, no post-create move.

```
create(name?: string, visibility: "public"|"private"|"unlisted")
  ŌåÆ returns the site's JSON projection; "url" is <baseURL><Seg(v)>/<name>/
    (an unlisted site's url therefore ends public/<token>/).
```

**`set_visibility(name, visibility, new_name?)`** ŌĆö the full transition matrix,
with the naming invariant deciding when `new_name` is required, forbidden, or
the name is regenerated. Every transition executes `Store.SetVisibility` (one
UPDATE, D15) and `Layout.Move` (D16) in lockstep and returns the site's
projection with its current `url` ŌĆö a rename is never silent.

| from \ to | `public`/`private` | `unlisted` |
|---|---|---|
| `public`/`private` | name kept; `new_name` **forbidden** (`validation`) | name **regenerated** to a fresh token; `new_name` **forbidden** |
| `unlisted` | `new_name` **required** (slug-validated; missing or invalid ŌåÆ `validation`) ŌĆö the site gets a real, chosen name | name **regenerated** to a fresh token ŌĆö this is **secret rotation**: the old URL stops resolving, the returned site carries the new one |

No transition is forbidden. Setting an already-public site public (or private
private) with no `new_name` is an idempotent no-op that still returns the site.
The unlistedŌåÆunlisted cell is deliberately *not* a no-op: rotating the link is
the point, and it is the revocation mechanism the product promises. Entering
unlisted always regenerates (a site's guessable history must not survive into
its unlisted life), and the token generation reuses `create`'s injectable
source and single-retry-on-collision behavior.

**Tools otherwise unchanged in shape:**

- `delete(name)` ŌĆö removes the row **and** `SiteDir(site.Visibility, name)`.
  Idempotent.
- `sync` ŌĆö still **requires the site to already exist** (content
  reconciliation, never birth); reconciles into the live served directory and
  leaves the site's visibility unchanged.
- The **file tools** (`file_write`, `file_read`, `file_edit`, `file_glob`,
  `file_grep`, `file_list`, `mkdir`) operate on the site's live served
  directory ŌĆö `SiteDir(site.Visibility, name)` resolved from the DB row ŌĆö and
  confine under it via `internal/files.ConfinePath`. An unlisted site is
  addressed by its token name like any other site.

**`renderSite`** projects `name`, `visibility` (the enum string ŌĆö the retired
`public` boolean leaves the projection), `owner_id`, `owner_email`, `url`,
`created_at`, `updated_at`, where `url` is `<baseURL><Seg(v)>/<name>/`. (The
`outputSchema` carrying these fields is D25's.)

**Rejected.**
- **A dedicated `create_unlisted` tool beside a two-state `create`.** Considered
  and initially favored, then rejected with the operator: the settled contract
  is *one* visibility vocabulary, and a separate verb would leave `create`
  speaking a boolean the rest of the surface no longer has. The enum argument
  carries the whole triple.
- **An optional `visibility` defaulting to private.** The old safe-default
  argument served a boolean flag; with three values and an agent-facing caller,
  explicitness wins ŌĆö a required argument is self-documenting in the schema and
  eliminates the name-required-or-forbidden ambiguity a defaulted call would
  reopen.
- **Keeping the old name when a site goes unlisted, or letting the caller pick
  the token.** Either way the "secret" URL contains a guessable name ŌĆö the
  feature's entire value evaporates. Entering unlisted always regenerates.
- **Forbidding unlistedŌåÆpublic (keeping "public" = chosen-name by refusing the
  exit).** Half-baking the matrix to protect the old interface; rejected by the
  operator. The invariant is preserved the honest way: leaving unlisted
  *requires* choosing a name.
- **A separate `rotate` tool.** `set_visibility(name, "unlisted")` on an
  unlisted site *is* rotation; a second verb for the same cell of the matrix is
  vocabulary for free.
- **Taking the owner as a tool argument; `sync` creating sites; a `visibility`
  argument on `sync`.** All previously rejected; unchanged.

**Verification.** (Driven through the assembled `appkit/mcp` handler over a temp
DB, temp `SITES_ROOT`, and a fake mirror client ŌĆö the same seam the existing
`internal/mcp` tests use.)

- R-RDBZ-AE4J ŌĆö the tool table returned by `Tools(...)` contains **no** tool named
  `publish` and **no** tool named `unpublish`.
- R-Z6FF-WUWS ŌĆö calling `create` with a request `Identity` carrying
  `X-Owner-Id: id-alice` and `X-Owner-Email: alice@example.com` persists
  `owner_id == "id-alice"` and `owner_email == "alice@example.com"`, both surfaced
  (as `owner_id` and `owner_email`) on the returned site and on a subsequent
  `list`. Two callers presenting the **same `X-Owner-Email` but different
  `X-Owner-Id`** create sites carrying **distinct `owner_id`** (create reads the
  stable id from `Identity.OwnerID`, not the email) ŌĆö a build still recording the
  email as the owner key, or dropping `owner_id` from the result, fails. (Real
  handler+DB ŌĆö the identity comes from the gate, the persistence and the projection
  are the real substrate.)
- R-H94T-M54D ŌĆö `create` **requires** `visibility`: a call omitting it, and a
  call passing a value outside the three (`"secret"`, `"true"`, `""`), each
  return the `validation` error envelope and create **no** row and **no**
  directory under either tree parent. A build that silently defaults the
  visibility fails.
- R-HACP-ZWV2 ŌĆö `create(name, visibility:"public")` creates the row
  (`visibility == "public"`) **and** the directory at `SiteDir(Public, name)` in
  one step and returns a `url` ending `public/<name>/`;
  `create(name, visibility:"private")` yields the private row, a directory at
  `SiteDir(Private, name)`, and a `url` ending `private/<name>/`; a
  public/private `create` **omitting `name`** returns `validation` and creates
  nothing.
- R-HBKM-DOLR ŌĆö `create(visibility:"unlisted")` with no `name` returns a site
  whose `name` matches `^[a-z2-7]{30}$`, whose `visibility` is `"unlisted"`,
  whose directory exists at `SiteDir(Unlisted, name)` (under the **public**
  parent), and whose `url` ends `public/<name>/`; the same call **with** a
  `name` argument returns `validation` and creates nothing.
- R-HCSI-RGCG ŌĆö with the injectable token source faked to return an
  already-taken token first and a fresh token second,
  `create(visibility:"unlisted")` succeeds with the fresh token (exactly one
  retry) and both sites coexist; the production default of the seam is
  `sites.NewToken`.
- R-HF8B-IZTU ŌĆö `set_visibility(name, "public")` on a private site keeps the
  name, flips the stored visibility, relocates the directory to
  `SiteDir(Public, name)`, and returns a `url` ending `public/<name>/` (and the
  reverse back to private); either call carrying a `new_name` returns
  `validation` and changes nothing; re-asserting the current visibility with no
  `new_name` is an idempotent success returning the unchanged site.
- R-HGG7-WRKJ ŌĆö `set_visibility(name, "unlisted")` from any visibility
  **regenerates the name**: the returned site's `name` is a fresh
  `^[a-z2-7]{30}$` token (ŌēĀ the old name), the old name no longer resolves
  (`get`-style lookups and the old directory are gone), the files now live at
  `SiteDir(Unlisted, token)`, and the `url` ends `public/<token>/`; calling it
  **again** on the now-unlisted site yields a **second, different** token whose
  URL replaces the first (rotation); a `new_name` on any transition into
  unlisted returns `validation` and changes nothing.
- R-HHO4-AJB8 ŌĆö `set_visibility(token, "public", new_name:"launch")` on an
  unlisted site renames the row and directory to `launch` at the public tree
  and returns a `url` ending `public/launch/` (and the private variant lands in
  the private tree); leaving unlisted **without** `new_name`, or with an
  invalid/reserved `new_name`, returns `validation` and changes nothing.
- R-56CN-HE21 ŌĆö calling `sync` with a `source_path`/`slug` naming a site that does
  not exist returns the `not_found` error envelope and creates **no** row and
  **no** directory under either tree parent; calling `sync` for an existing site
  still reconciles its files and leaves its `visibility` unchanged.
- R-RI7K-TH3B ŌĆö a `file_write` to a site resolves under
  `SiteDir(site.Visibility, name)` (the live served directory): after writing
  `index.html` to a public site, the file exists at
  `SiteDir(Public, name)/index.html` and is served by D17's handler ŌĆö there is
  no separate working-tree copy.
- R-RJFH-78U0 ŌĆö `delete(name)` removes both the row (a subsequent `get` yields
  not-found) and the on-disk directory `SiteDir(site.Visibility, name)` (it no
  longer exists).
