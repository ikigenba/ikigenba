# Decision 20 — MCP surface: name + slug split end to end, explicit create, the full transition matrix, `rename`

**Decision.** The `ikigenba_sites_*` tool table speaks the split identity (D15)
end to end. The governing invariant, stated once and enforced at every entry
point: **every site carries an owner-chosen display `name`; its `slug` is the
URL handle — chosen by the caller for public/private sites, a generated token
(D27) the caller never chooses for unlisted sites.** Visibility transitions may
regenerate or replace the slug; **no transition ever touches the name** — the
name is how the owner tells sites apart, whatever their visibility. A site is
created at an explicitly stated visibility, its files are edited *live* in its
served directory, its visibility (and, when the invariant demands it, its slug)
is changed with one tool, its name is changed with `rename`, and it is deleted.

**`create(name, slug?, visibility)`** — `visibility` is **required** (one of
`"public"`, `"private"`, `"unlisted"`; anything else — including omission — is a
`validation` error before any row or directory exists). **`name` is required for
all three visibilities**: it is validated and trimmed by `sites.ValidateName`
(D15); a missing, empty, oversized, or control-character name is a `validation`
error. `slug` is conditional on the invariant:

- `visibility: "public" | "private"` — `slug` **required**, validated by the
  existing slug grammar and reserved-name guard.
- `visibility: "unlisted"` — `slug` **forbidden** (a supplied slug is a
  `validation` error: the slug is the credential; letting a caller pick one
  would let them pick a guessable credential). The handler generates the slug
  with the injectable token source (default `sites.NewToken`, D27) and, on the
  astronomically-rare `ErrExists` collision, retries once with a fresh token.

The creator is recorded from the request `Identity` exactly as before (stable
`Identity.OwnerID` → `owner_id`, `Identity.OwnerEmail` → `owner_email`; neither
is a tool argument). Row and directory are born in **one step** at the stated
visibility: `store.Create(ctx, slug, name, id.OwnerID, id.OwnerEmail, v)` +
`os.MkdirAll(SiteDir(v, slug))` — for unlisted that directory is under the
**public** parent (D16). No intermediate state, no post-create move.

```
create(name: string, slug?: string, visibility: "public"|"private"|"unlisted")
  → returns the site's JSON projection; "url" is <baseURL><Seg(v)>/<slug>/
    (an unlisted site's url therefore ends public/<token>/).
```

**`set_visibility(slug, visibility, new_slug?)`** — the full transition matrix,
with the naming invariant deciding when `new_slug` is required, forbidden, or
the slug is regenerated. Every transition executes `Store.SetVisibility` (one
UPDATE, D15) and `Layout.Move` (D16) in lockstep and returns the site's
projection with its current `url` — a re-slug is never silent. **The display
name is untouched by every cell of the matrix.**

| from \ to | `public`/`private` | `unlisted` |
|---|---|---|
| `public`/`private` | slug kept; `new_slug` **forbidden** (`validation`) | slug **regenerated** to a fresh token; `new_slug` **forbidden** |
| `unlisted` | `new_slug` **required** (slug-validated; missing or invalid → `validation`) — the site gets a real, chosen slug | slug **regenerated** to a fresh token — this is **secret rotation**: the old URL stops resolving, the returned site carries the new one |

No transition is forbidden. Setting an already-public site public (or private
private) with no `new_slug` is an idempotent no-op that still returns the site.
The unlisted→unlisted cell is deliberately *not* a no-op: rotating the link is
the point, and it is the revocation mechanism the product promises — and
because the name survives it, the owner can still tell which site the fresh
link belongs to. Entering unlisted always regenerates (a site's guessable
history must not survive into its unlisted life), and the token generation
reuses `create`'s injectable source and single-retry-on-collision behavior.

**`rename(slug, name)`** — the display-label mutator, new with this Decision.
Validates/trims via `sites.ValidateName` (a bad name is `validation`), calls
`Store.Rename` (D15), and returns the site's projection. It changes **only**
the name: slug, visibility, URL, files, and owner fields are untouched. A
missing site is `not_found`. Its description leads with the searchable
vocabulary ("rename a site's display name/label").

**Tools otherwise unchanged in shape** (every tool addresses a site by its
`slug` — for an unlisted site that is the token, which the caller has from
`create`/`list`):

- `delete(slug)` — removes the row **and** `SiteDir(site.Visibility, slug)`.
  Idempotent.
- `sync` — still **requires the site to already exist** (content
  reconciliation, never birth); already speaks `slug` in its own parameters;
  reconciles into the live served directory and leaves the site's visibility
  (and name) unchanged.
- The **file tools** (`file_write`, `file_read`, `file_edit`, `file_glob`,
  `file_grep`, `file_list`, `mkdir`) keep their `site` parameter, whose value
  is the slug; they operate on `SiteDir(site.Visibility, slug)` resolved from
  the DB row and confine under it via `internal/files.ConfinePath`.

**`renderSite`** projects `slug`, `name`, `visibility` (the enum string),
`owner_id`, `owner_email`, `url`, `created_at`, `updated_at`, where `url` is
`<baseURL><Seg(v)>/<slug>/`. There is no longer a projection field in which
"name" means the URL handle. (The `outputSchema` carrying these fields is
D25's.)

**Rejected.**
- **Letting the name double as the slug for public/private sites.** That is the
  retired model; it made the two interchangeable in callers' heads and made the
  unlisted token *replace* the label. The split is total (settled with the
  operator).
- **Deriving the slug from the name (slugify).** Lossy, edge-case-laden, and
  unnecessary for an agent caller — the agent generating the call can propose a
  slug itself (operator: "the agent creating them can help generate the slug").
- **Addressing tools by name.** Names are non-unique display text;
  disambiguation machinery for no benefit. The slug is the unique handle
  (settled with the operator).
- **Folding rename into `set_visibility`, or a general `update` tool.**
  Renaming is not a visibility transition, and a general update is more
  surface than one mutable field warrants. One small tool.
- **A dedicated `create_unlisted` tool; a defaulted `visibility`; keeping the
  old slug when entering unlisted or letting the caller pick the token;
  forbidding unlisted→public; a separate `rotate` tool; owner as an argument;
  `sync` creating sites.** All previously rejected; unchanged.

**Verification.** (Driven through the assembled `appkit/mcp` handler over a temp
DB, temp `SITES_ROOT`, and a fake mirror client — the same seam the existing
`internal/mcp` tests use. Ids replaced by this rewrite take their tagged tests
with them: a test tagged with a retired id is rewritten under the new id and
the new contract.)

- R-RDBZ-AE4J — the tool table returned by `Tools(...)` contains **no** tool named
  `publish` and **no** tool named `unpublish`.
- R-ZN05-WU3D — `create` **requires** `visibility`: a call omitting it, and a
  call passing a value outside the three (`"secret"`, `"true"`, `""`), each
  return the `validation` error envelope and create **no** row and **no**
  directory under either tree parent. A build that silently defaults the
  visibility fails.
- R-ZO82-ALU2 — `create` **requires a valid `name` at every visibility**: a
  call omitting `name`, passing an empty or whitespace-only name, a >100-rune
  name, or a name containing a control character returns `validation` and
  creates nothing — for public, private, **and** unlisted alike; a valid call
  whose name carries surrounding whitespace (`"  Launch  "`) persists and
  returns the **trimmed** name (`"Launch"`). A build that lets an unlisted
  create omit the name (the retired contract) fails.
- R-ZQNV-25BG — `create(name, slug, visibility:"public")` creates the row
  (`visibility == "public"`, the passed name stored) **and** the directory at
  `SiteDir(Public, slug)` in one step and returns a `url` ending
  `public/<slug>/`; the private variant yields the private row, directory, and
  `url` ending `private/<slug>/`; a public/private `create` **omitting `slug`**
  returns `validation` and creates nothing; an invalid or reserved `slug`
  returns `validation`.
- R-ZRVR-FX25 — `create(name:"Client Preview", visibility:"unlisted")` with no
  `slug` returns a site whose `slug` matches `^[a-z2-7]{30}$`, whose **`name`
  is `"Client Preview"`** (the label survives token generation), whose
  visibility is `"unlisted"`, whose directory exists at
  `SiteDir(Unlisted, slug)` (under the **public** parent), and whose `url` ends
  `public/<slug>/`; the same call **with** a `slug` argument returns
  `validation` and creates nothing.
- R-ZT3N-TOSU — with the injectable token source faked to return an
  already-taken token first and a fresh token second,
  `create(name:…, visibility:"unlisted")` succeeds with the fresh token
  (exactly one retry) and both sites coexist; the production default of the
  seam is `sites.NewToken`.
- R-ZUBK-7GJJ — calling `create` with a request `Identity` carrying
  `X-Owner-Id: id-alice` and `X-Owner-Email: alice@example.com` persists
  `owner_id == "id-alice"` and `owner_email == "alice@example.com"`, both
  surfaced on the returned site and on a subsequent `list`. Two callers
  presenting the **same `X-Owner-Email` but different `X-Owner-Id`** create
  sites carrying **distinct `owner_id`**. A build recording the email as the
  owner key, or dropping `owner_id` from the result, fails.
- R-ZVJG-L8A8 — `set_visibility(slug, "public")` on a private site keeps the
  slug **and the name**, flips the stored visibility, relocates the directory
  to `SiteDir(Public, slug)`, and returns a `url` ending `public/<slug>/` (and
  the reverse back to private); either call carrying a `new_slug` returns
  `validation` and changes nothing; re-asserting the current visibility with no
  `new_slug` is an idempotent success returning the unchanged site.
- R-ZWRC-Z00X — `set_visibility(slug, "unlisted")` from any visibility
  **regenerates the slug and keeps the name**: the returned site's `slug` is a
  fresh `^[a-z2-7]{30}$` token (≠ the old slug) while its `name` is
  byte-identical to before, the old slug no longer resolves (lookups and the
  old directory are gone), the files now live at `SiteDir(Unlisted, token)`,
  and the `url` ends `public/<token>/`; calling it **again** on the
  now-unlisted site yields a **second, different** token whose URL replaces the
  first (rotation) — the name still unchanged; a `new_slug` on any transition
  into unlisted returns `validation` and changes nothing.
- R-ZXZ9-CRRM — `set_visibility(token, "public", new_slug:"launch")` on an
  unlisted site renames the row and directory to `launch` at the public tree,
  keeps the **name** byte-identical, and returns a `url` ending
  `public/launch/` (and the private variant lands in the private tree); leaving
  unlisted **without** `new_slug`, or with an invalid/reserved `new_slug`,
  returns `validation` and changes nothing.
- R-ZZ75-QJIB — `rename(slug, name)` returns the site with the new **trimmed**
  name and everything else unchanged (slug, visibility, `url`, `owner_id`,
  `owner_email`, `created_at` all byte-identical to before; the site's files
  still served); an invalid name (empty, whitespace-only, >100 runes, control
  character) returns `validation` and changes nothing; a missing site returns
  `not_found`. It works on an unlisted site addressed by its token.
- R-00F2-4B90 — `delete(slug)` removes both the row (a subsequent lookup yields
  not-found) and the on-disk directory `SiteDir(site.Visibility, slug)` (it no
  longer exists).
- R-56CN-HE21 — calling `sync` with a `source_path`/`slug` naming a site that does
  not exist returns the `not_found` error envelope and creates **no** row and
  **no** directory under either tree parent; calling `sync` for an existing site
  still reconciles its files and leaves its `visibility` unchanged.
- R-RI7K-TH3B — a `file_write` to a site resolves under
  `SiteDir(site.Visibility, slug)` (the live served directory): after writing
  `index.html` to a public site, the file exists at
  `SiteDir(Public, slug)/index.html` and is served by D17's handler — there is
  no separate working-tree copy.
