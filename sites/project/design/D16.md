# Decision 16 — Filesystem layout: files live at the served path; unlisted shares the public tree

**Decision.** A site's files live **directly** at the location they are served
from — there is no working tree and no symlink indirection. The filesystem has
exactly **two** parents, matching the two nginx locations (D18): visibility maps
onto a tree segment, and **unlisted maps to the public segment** — an unlisted
site is served through the very same ungated public tier; its protection is its
unguessable name (D27), not a third tree. `internal/sites/layout.go`:

```go
const DefaultRoot = "/opt/sites/state/www" // SITES_ROOT default (unchanged)

const (
    PublicSeg  = "public"
    PrivateSeg = "private"
)

// Seg maps a visibility to its on-disk / URL tree segment:
//   Public   → "public"
//   Unlisted → "public"   (served ungated; the token name is the credential)
//   Private  → "private"
func Seg(v Visibility) string

// SiteDir is a site's on-disk directory: <root>/<Seg(v)>/<slug>.
func (l Layout) SiteDir(v Visibility, slug string) string

// SiteBase is <root>/<Seg(v)> — the parent of that segment's site dirs.
func (l Layout) SiteBase(v Visibility) string
```

There is deliberately **no** `unlisted/` directory and no third nginx location:
the whole feature stays inside the existing routing (the operator's MCP-only
scope), and the URL of an unlisted site visibly carries the `public/` segment —
accepted, since the tier *is* unauthenticated and the secrecy lives entirely in
the name.

**Visibility changes move (and may rename) the directory in lockstep with the
row.** The unlisted transitions (D20) rename a site as they move it, so `Move`
takes the full before/after coordinates:

```go
// Move relocates a site's directory from <root>/<Seg(from)>/<slug> to
// <root>/<Seg(to)>/<newSlug>. Called by the MCP set_visibility path in lockstep
// with Store.SetVisibility (D15). A no-op when source and destination paths are
// equal. Tolerates a missing source dir (a site with no files yet). Refuses an
// existing destination (never merges two sites' files).
func (l Layout) Move(slug string, from Visibility, newSlug string, to Visibility) error
```

The database row (D15) is authoritative; `SiteDir`/`Move` keep the filesystem a
faithful mirror. The MCP tools are the only writer (D20), so the two never
drift. Note `Move` between `Public` and `Unlisted` with an unchanged slug would
be a same-path no-op — in practice that pair never occurs with an unchanged
slug, because every transition into unlisted renames to a fresh token (D20).

**Why this is safe without a read-time symlink guard.** The served tree holds
real files only; the only confinement concern is ordinary path traversal within
a site dir, which the static server confines per request via
`internal/files.ConfinePath` (D17) and the file tools confine on write
(D10/D11). There is no symlink layer to defend.

**Rejected.**
- **A third `unlisted/` tree and nginx location.** It buys nothing — the tier
  would be configured identically to `public/` (ungated, proxied) — and costs a
  routing change, a fragment edit, and a URL scheme that breaks if it ever
  changes again. Rejected with the operator: the feature is MCP-only.
- **Hiding the `public/` segment from unlisted URLs (URL rewriting).** Cosmetic
  only; the segment is honest (the tier is unauthenticated) and rewriting it
  would smuggle routing logic into a feature that deliberately has none.
- **Keeping a working tree / symlink served-tree.** Rejected previously with the
  publish lifecycle; unchanged.

**Verification.**
- R-H5H4-GTWA — the path formula covers all three visibilities:
  `SiteDir(Public, "blog")` equals `<root>/public/blog`,
  `SiteDir(Private, "blog")` equals `<root>/private/blog`, and
  `SiteDir(Unlisted, "tok…")` equals `<root>/public/tok…` (the **public**
  segment — an implementation inventing an `unlisted/` parent fails);
  `Seg` returns `"public"`, `"private"`, `"public"` for the three values
  respectively.
- R-H6P0-ULMZ — `Move` relocates and renames in one call: after
  `Move("blog", Private, "tok", Unlisted)` on a site whose files were at
  `<root>/private/blog`, the files exist at `<root>/public/tok` and
  `<root>/private/blog` no longer exists; a `Move` whose source and destination
  paths are equal is a no-op that leaves the files in place; a `Move` with a
  missing source dir succeeds (site with no files yet); a `Move` onto an
  existing destination directory errors and leaves both directories untouched.
- R-QYP6-P587 — the working-tree/served-symlink machinery is gone: a scoped grep
  over `internal/` (excluding `project/`) for `symlinkTarget`, `os.Symlink`,
  `WorkingDir`, `ServedDir`, `ServedTierBase`, and `ServedBase` returns no matches,
  and `internal/sites/publish.go` does not exist.
