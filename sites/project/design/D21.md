# Decision 21 ‚Äî MCP self-discovery: routing `instructions` + a `guide` tool (replacing `describe`)

**Decision.** sites adopts the suite's MCP self-discovery convention
(`docs/mcp-discovery-convention.md`, piloted on crm) so a connecting agent can
find sites and drive it from the connection alone. Two channels change; no domain
verb's *behavior* changes.

**Tier 0 ‚Äî the `initialize` `instructions`.** The `Instructions` const in
`internal/mcp/mcp.go` is rewritten to name sites's domain in the vocabulary a
user actually types (the routing hooks that let tool search match sites without
an external alias skill), state the verb flow, and point at the `guide` tool ‚Äî
and it no longer mentions `describe`. Illustrative:

```go
const Instructions = "Host small static websites on this box ‚Äî web pages, " +
    "landing pages, marketing or docs sites, hand-written HTML/CSS/JS ‚Äî each " +
    "served at its own URL: public (anyone), private (logged-in dashboard " +
    "users), or unlisted (anyone who has the long secret link). Create a site " +
    "at a stated visibility, write its files, and change its visibility any " +
    "time; edits are live immediately, no publish step. Call guide for the " +
    "site model and worked examples."
```

**Tier 2 ‚Äî a `guide` tool replaces `describe`.** The lightweight `describe` tool
and its `toolDescribe()` JSON blob are **deleted**. In their place a single
read-only `guide` tool returns an embedded usage document as text, exactly as the
crm pilot does:

```go
//go:embed guide.md
var guideDoc string

// guide: flat, read-only, input-free; returns the embedded document verbatim.
desc(tool("guide"), "Return the sites usage guide ‚Äî the site model, name/slug \
and confinement rules, and worked basic/advanced examples (create a public page \
in one call, import from Dropbox). Read once before your first create.",
    obj(map[string]any{}),  // no properties, no required
    func(ctx, args, _ ) (map[string]any, error) { return appkitmcp.TextResult(guideDoc), nil })
```

`guide` takes no inputs, reads nothing, mutates nothing, and does not error. The
document is embedded (versioned and shipped with the binary), so the guide an
agent reads always matches the running service. The surface is **16 tools**
(D13): `guide` replaced `describe` 1:1, and the name/slug split added `rename`
(D20); `create`/`sync` keep their names (their schemas evolve under D20).

**`guide.md` contents** (one screen; the reference bulk that must not live in
Tier 0/1):

1. **Model** ‚Äî a site = a display **name** (free-form label, how you tell sites
   apart) + a **slug** (its URL address) + visibility
   (`public`/`private`/`unlisted`) + creator; files served live at the site's
   URL; **`create` is the only way a site comes into being** ‚Äî every other tool
   (`file_*`, `mkdir`, `sync`, `set_visibility`, `rename`, `delete`) requires
   the site to already exist and returns `not_found` otherwise. `visibility` is
   always stated explicitly ‚Äî there is no default. **Every site gets a name you
   choose**, whatever its visibility. Public/private sites also get a slug you
   choose; an **unlisted** site's slug is generated for you: a long random
   string nobody can guess, so its URL is the credential ‚Äî share the link with
   exactly who should see it. Tools address a site by its slug; `rename`
   changes only its display name.
2. **Slug and name rules** ‚Äî slug: 1‚Äì63 chars, lowercase alphanumeric + hyphen,
   must start alphanumeric; reserved names rejected; you never supply a slug
   for an unlisted site. Name: free-form text up to 100 characters, not blank ‚Äî
   spaces and capitals are fine.
3. **Confinement** ‚Äî file paths are relative to the site root; absolute paths and
   `..` escapes are rejected (`path_escapes_working_dir`).
4. **Basic ‚Äî a public page in two calls** ‚Äî
   `create(name:"Launch", slug:"launch", visibility:"public")` ‚Üí
   `file_write(site:"launch", file_path:"index.html", content:"‚Ä¶")` ‚Üí visit the
   returned `url`.
5. **Basic ‚Äî a private site** ‚Äî `create(name:"Runbook", slug:"runbook",
   visibility:"private")` ‚Üí write files ‚Üí served only to logged-in users;
   promote later with `set_visibility(slug:"runbook", visibility:"public")`.
6. **Basic ‚Äî a secret link (unlisted)** ‚Äî `create(name:"Client Preview",
   visibility:"unlisted")` (a name, but no slug) ‚Üí write files ‚Üí hand out the
   returned long random `url`; anyone with the link can view it, nobody without
   it can find it. The site keeps its name through everything ‚Äî your listing
   always shows "Client Preview", never the token. **Rotate a leaked link**
   with `set_visibility(slug:"<token>", visibility:"unlisted")` ‚Äî the site gets
   a fresh URL, the old one goes dead, and the name is untouched. Leaving
   unlisted requires choosing a real slug:
   `set_visibility(slug:"<token>", visibility:"public", new_slug:"launch")`.
7. **Advanced ‚Äî import from Dropbox** ‚Äî
   `create(name:"Marketing", slug:"marketing", visibility:"private")` first
   (`sync` will not create it), then `sync(source_path:"/sites/marketing")`,
   then `set_visibility(slug:"marketing", visibility:"public")`.
8. **Error self-correction** ‚Äî `not_found` ‚Üí "call `create` first";
   `already_exists`; `invalid_slug`; `reserved_name`; `validation` on a missing
   visibility, a missing or blank name, a slug supplied for unlisted, or a
   missing `new_slug` when leaving unlisted.

**Tier 1 ‚Äî keyword-forward the two descriptions that change.** Under D20 the
`create` and `sync` descriptions are rewritten anyway; they lead with the domain
vocabulary tool search indexes (`create` ‚Üí "static website/site"; `sync` ‚Üí
"import ‚Ä¶ from a Dropbox-mirrored folder"). `sync`'s wording deliberately avoids
"publish"/"deploy" ‚Äî reintroducing that vocabulary would re-imply the deleted
publish step.

**The guide is referenced in exactly two places** ‚Äî the Tier 0 `Instructions`
and the `guide` tool's own description ‚Äî and **nowhere else**. It is deliberately
not repeated in every tool description (that would re-inflate Tier 1 for no gain);
an agent that ignores the pointer and calls a write tool cold self-corrects from
the `not_found` envelope.

**Rejected.**
- **Keeping `describe` alongside a new `guide`.** Two orientation tools is the
  redundancy the convention collapses; the crm pilot has only `guide`. `describe`
  is removed, not kept.
- **A `guide` MCP *resource* instead of a tool.** Most clients do not surface
  resources into the agent's context; the tool is the channel with the hard
  delivery guarantee (`tools/list` in every client).
- **Repeating the guide pointer in every tool description.** Re-inflates Tier 1
  on every connection; the always-loaded Tier 0 pointer already reaches the agent
  before it first needs the guide.
- **A parameterized/topic-scoped guide.** Premature: sites's guide fits one
  screen. Start flat and input-free; scope only if it outgrows a screen.

**Verification** (through the assembled `appkit/mcp` handler over a temp DB, temp
`SITES_ROOT`, and a fake mirror client ‚Äî the same seam D13/D20 use; plus a
content assertion over the repo-real embedded `guide.md`).
- R-57KJ-V5SQ ‚Äî `tools/list` through the assembled handler contains a tool named
  `guide` and **no** tool named `describe`; `guide`'s `InputSchema` is an object
  schema that declares **no** `required` fields; and `tools/call guide` with
  omitted arguments returns a **single non-empty text content block** (not an
  error envelope) ‚Äî proving `guide` is present, flat, read-only, and input-free.
- R-58SG-8XJF ‚Äî the embedded `guideDoc` text contains all four load-bearing
  anchors: the create-first rule (a site must exist / `create` is the only birth
  path), a public example that invokes `create` with `visibility` stated, the
  Dropbox flow that shows `create` **before** `sync`, and the confinement token
  `path_escapes_working_dir` ‚Äî proving the reference bulk actually lives in Tier 2.
- R-01MY-I2ZP ‚Äî the embedded `guideDoc` text contains the unlisted anchors: the
  word `unlisted`, an unlisted create example that passes a **`name`** but
  **no** `slug`, a statement that the URL is the credential/secret link, a
  rotation example (`set_visibility` back to `unlisted`), and a statement that
  the site's name survives (the listing shows the name, not the token) ‚Äî
  proving an agent can discover the secret-link feature, its revocation, and
  the name-keeps rule from the guide alone.
- R-5A0C-MPA4 ‚Äî the `Instructions` string contains the routing word `website` and
  the pointer word `guide`, and does **not** contain `describe` ‚Äî proving Tier 0
  carries user vocabulary and points at the guide, with the old tool name gone.
- R-5B89-0H0T ‚Äî the guide is referenced in exactly two channels: `Instructions`
  contains `guide` and the `guide` tool's own `Description` is non-empty; and for
  **every other tool** in the table, its `Description` does not contain the
  substring `guide` ‚Äî proving Tier 1 was not re-inflated with the pointer.
- R-5CG5-E8RI ‚Äî the `sync` tool's `Description` contains neither `publish` nor
  `deploy` (the deleted-vocabulary guard) while staying findable (it contains
  `Dropbox`) ‚Äî proving the reworded description does not resurrect the publish
  step.
