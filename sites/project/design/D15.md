# Decision 15 — Data model: slug + display name, the three-value `visibility` enum, owner id/email

**Decision.** The `sites` row records *what exists, where it is addressed, what
it is called, at which visibility, owned by whom*. Two distinct fields carry the
identity that one overloaded column used to carry: the **`slug`** is the URL
handle (the PK, the address), and the **`name`** is the human display label. For
public/private sites the owner chose the slug; for unlisted sites the slug is
the generated 30-char token (D27) — but **every** site, unlisted included,
carries an owner-chosen `name` that identifies it in any listing. Visibility is
a **TEXT enum of exactly three values** — `public`, `private`, `unlisted` —
enforced by SQLite:

| column | type | meaning |
|---|---|---|
| `slug` | TEXT PK | the URL handle (grammar `^[a-z0-9][a-z0-9-]{0,62}$`, reserved-name guard). Owner-chosen for public/private sites; the generated 30-char token (D27) for unlisted sites — the token satisfies the same grammar, so one column and one grammar serve all three visibilities |
| `name` | TEXT NOT NULL | the human display label — free-form owner-chosen text (validated by `ValidateName` below), **not** slug-constrained, **not** unique. Never changed by any visibility transition; changed only by an explicit rename |
| `visibility` | TEXT NOT NULL CHECK (`visibility IN ('public','private','unlisted')`) | the site's visibility, exactly one of the three |
| `owner_id` | TEXT NOT NULL | stable owner key (the nginx-forwarded `X-Owner-Id`, appkit `Identity.OwnerID`); captured at create. Sites does not owner-scope any query (the `slug` is the global handle), so no logic reads it — it is stored so the suite keys on the stable id and any future owner grouping has it |
| `owner_email` | TEXT NOT NULL | owner email that created the site — a **write-once snapshot** captured from the request headers at row creation, never updated afterward, never read for logic. Display only |
| `source_path` | TEXT | dropbox-sync provenance (nullable; **unchanged**) |
| `created_at` | TEXT NOT NULL | UTC creation timestamp |
| `updated_at` | TEXT NOT NULL | UTC last-update timestamp |

The former single `name`-as-slug column is **retired**: it was doing two jobs,
and for unlisted sites the generated token *replaced* the label entirely, so an
owner looking at a listing could not tell what an unlisted site was. The slug
and the name are not the same thing; the schema now says so.

**The name validator** (`internal/sites`). The display name is free-form but
not lawless:

```go
// ErrInvalidName joins ErrInvalidSlug etc. as a domain sentinel.
var ErrInvalidName = errors.New("invalid site name")

// ValidateName trims leading/trailing Unicode whitespace and returns the
// trimmed name. It returns ErrInvalidName when the trimmed value is empty,
// longer than 100 runes, or contains a control character (Unicode Cc —
// newlines and tabs included). "Q3 Launch Page" is valid; "" and "\n" are not.
func ValidateName(raw string) (string, error)
```

The **trimmed** value is what gets stored. Names are deliberately **not
unique** — the slug is the identity; two sites named `test` are the owner's own
mess, not an error.

**The Go visibility type** (`internal/sites`) is unchanged:

```go
type Visibility string

const (
    Public   Visibility = "public"
    Private  Visibility = "private"
    Unlisted Visibility = "unlisted"
)

func ParseVisibility(s string) (Visibility, error) // outside the three → ErrInvalidVisibility
```

**Schema change is one new immutable migration that rebuilds the table** (per
Conventions; created with `bin/create-migration sites <name>`, never hand-named).
All previously committed migrations stay frozen. There is **no production data
in sites** (operator ruling), so the rebuild **does not carry rows across** —
it drops and recreates:

```sql
DROP TABLE sites;
CREATE TABLE sites (
    slug        TEXT    PRIMARY KEY,
    name        TEXT    NOT NULL,
    visibility  TEXT    NOT NULL CHECK (visibility IN ('public','private','unlisted')),
    owner_id    TEXT    NOT NULL,
    owner_email TEXT    NOT NULL,
    source_path TEXT,
    created_at  TEXT    NOT NULL,
    updated_at  TEXT    NOT NULL
) STRICT;
CREATE INDEX idx_sites_owner ON sites(owner_id);
```

`STRICT` is preserved; the `CHECK` makes SQLite itself the enum's enforcer.

**Store contract** (`internal/sites/store.go`). The `Site` struct carries both
fields; `SetVisibility` keeps its optional-rename shape (now renaming the
**slug**), and a new `Rename` is the display-label mutator:

```go
type Site struct {
    Slug       string     // URL handle (PK)
    Name       string     // human display label; survives every visibility transition
    Visibility Visibility
    OwnerID    string     // stable owner key, captured at create
    OwnerEmail string     // write-once display snapshot
    SourcePath string     // dropbox-sync provenance (unchanged)
    CreatedAt  time.Time
    UpdatedAt  time.Time
}

// Create inserts a fresh row at the caller-stated visibility, stored verbatim.
// slug and name are stored exactly as passed (the MCP layer validates and
// trims; internal callers are trusted). ownerID/ownerEmail are captured from
// the request Identity. Returns ErrExists on a duplicate slug.
func (s *Store) Create(ctx context.Context, slug, name, ownerID, ownerEmail string, v Visibility) (Site, error)

// SetVisibility updates a site's visibility and, when newSlug is non-empty,
// renames the row's slug in the same single UPDATE (the unlisted transitions
// re-slug; public↔private passes ""). The display name is never touched. Bumps
// updated_at; never touches name/owner_id/owner_email/created_at. Returns
// ErrNotFound for a missing slug and ErrExists when newSlug collides.
func (s *Store) SetVisibility(ctx context.Context, slug string, v Visibility, newSlug string) error

// Rename updates only the display name (already validated/trimmed by the
// caller) and bumps updated_at. Slug, visibility, owner fields, and created_at
// are untouched. Returns ErrNotFound for a missing slug.
func (s *Store) Rename(ctx context.Context, slug, name string) error
```

`Get`/`List`/`Delete`/`SetSourcePath` keep their shapes, keyed by `slug`, their
column lists carrying `slug` + `name`. `List` stays ordered by `slug`. The
on-disk directory move that accompanies a visibility change is D16's
`Layout.Move`, invoked in lockstep by the MCP handler (D20).

**Rejected.**
- **Keeping name-as-slug and adding a label only to unlisted sites.** A
  nullable label used by one visibility is a wart: a site transitioning
  public→unlisted→public would gain then lose the field, and the confusion
  (which field identifies the site?) survives. Settled with the operator: the
  split applies to all three visibilities.
- **Slug-grammar names, or unique names.** A label constrained to slug grammar
  recreates the name/slug interchangeability this Decision removes; uniqueness
  on a display label adds failure modes for no gain. Settled with the operator:
  free-form, non-unique.
- **Deriving the slug from the name (slugification).** A lossy transformation
  with edge cases (emoji, non-ASCII, collisions); the caller is an agent for
  whom stating the slug explicitly costs nothing (D20 requires it).
- **Carrying existing rows across in the rebuild.** There is no production data
  in sites (operator ruling); a backfill that invents labels for token-named
  rows buys nothing. The rebuild drops and recreates.
- **A separate `rename_slug` store method beside `SetVisibility`.** Every slug
  rename in the contract happens *because of* a visibility transition (D20);
  two calls would open a gap where slug and visibility disagree. One UPDATE,
  one moment. (The display-name `Rename` is different: it is *not* coupled to
  any transition, so it is its own method.)

**Verification.** (Proven against a real migrated SQLite DB, as before.)
- R-Z9L9-PCXQ — `Store.Create` persists slug, name, and visibility **verbatim
  for each of the three visibility values**: three creates with `Public`,
  `Private`, and `Unlisted`, each with a distinct free-form name (one
  containing spaces and an uppercase letter), read back (via `Get` and `List`)
  with exactly the passed slug, the passed name byte-for-byte, and the passed
  visibility. An implementation that collapses name into slug, slug-normalizes
  the name, or defaults any value, fails.
- R-ZAT6-34OF — `Store.SetVisibility(ctx, slug, v, "")` updates the visibility
  and keeps the slug; `SetVisibility(ctx, slug, v, "newslug")` updates **both
  slug and visibility** so a subsequent `Get` of the old slug is `ErrNotFound`
  and of the new slug returns the new visibility; **in both cases the `name` is
  byte-identical to what it was before the call**; `updated_at` advances; a
  missing slug yields `ErrNotFound`; a `newSlug` colliding with an existing row
  yields `ErrExists` and leaves both rows unchanged. An implementation that
  rewrites the display name on a transition fails.
- R-ZC12-GWF4 — `Store.Rename(ctx, slug, name)` changes **only** the display
  name: a subsequent `Get` returns the new name with the slug, visibility,
  `owner_id`, `owner_email`, and `created_at` unchanged, and `updated_at`
  advanced; a missing slug yields `ErrNotFound`.
- R-ZD8Y-UO5T — `Store.Create` persists `owner_id == ownerID` and
  `owner_email == ownerEmail` **exactly as passed** (the email is a snapshot,
  not derived, defaulted, or used as the key); two `Create`s with the **same
  `ownerEmail` but different `ownerID`** persist **distinct `owner_id`** values.
  A build that stored the email into `owner_id`, or derived one column from the
  other, fails. (Distinct, unrelated id and email values.)
- R-ZEGV-8FWI — after the full migration set runs, `pragma table_info(sites)`
  reports a `slug` TEXT primary-key column, a `name` TEXT NOT NULL column, and
  a `visibility` TEXT column — and **no** `public` column (nor
  `created_by`/`tier`/`published`/`published_at`); a direct INSERT with
  `visibility = 'bogus'` is **rejected by SQLite itself** (the CHECK errors),
  and a direct INSERT with a NULL `name` is rejected — proven against the real
  SQLite the runner uses.
- R-ZFOR-M7N7 — `ValidateName` returns the **trimmed** name for valid input
  (`"  Q3 Launch Page "` → `"Q3 Launch Page"`, nil error) and returns
  `ErrInvalidName` for: the empty string, a whitespace-only string, a string
  whose trimmed form exceeds 100 runes (a 101-rune name of multi-byte runes —
  a byte-counting implementation fails), and a string containing a control
  character (`"a\nb"`). A validator that accepts empty, counts bytes, or
  passes control characters fails.
