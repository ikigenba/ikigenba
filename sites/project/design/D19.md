# Decision 19 — The landing page lists the sites that exist, by name

**Decision.** The landing route grows from a static version card into the
service's owner-facing index: it lists the sites the store holds, alongside the
version. The route stays `GET /{$}` at the bare mount root, session-gated at nginx
(D4), rendered in-process from the DB (always accurate, no generated file on
disk). **The row's identity is the site's display name** (D15): the name is what
lets the owner tell sites apart in a large list — an unlisted site shows its
owner-chosen name, never a bare 30-char token, as its visible label. The slug
still travels in the data island (search matches it, D22) but is not a visible
column.

The `GET /{$}` handler queries `store.List(ctx)` and passes a view model to the
existing `share/www/landing.html` template through `rt.WWW().Render`:

```go
type landingView struct {
    Service string
    Version string
    Sites   []siteRow
}

type siteRow struct {
    Slug          string // URL handle; island data only, not a visible column
    Name          string // display label; the visible, linked row identity
    URL           string // absolute front-door URL; identical to the MCP `url` field
    Visibility    string // the enum string "public" / "private" / "unlisted" (D15), rendered verbatim
    CreatedBy     string // the creator email, sourced from the site's owner_email snapshot (D15)
    CreatedAt     string // pre-formatted UTC for human display
    CreatedAtSort string // RFC3339 UTC — machine-sortable timestamp for the client (D22)
}
```

The landing page is a browser display surface, not an MCP result, so it shows the
creator **email only** (the site's `owner_email` snapshot). The stable `owner_id`
is not exposed in the page.

`landing.html` renders one row per `siteRow` — the **name** (linked, below), the
visibility label (printed verbatim: `public`, `private`, or `unlisted`),
creator, created-at — and renders cleanly (no rows / an explicit empty state)
when `Sites` is empty. The name is free-form user text; the Go template's
contextual auto-escaping is what makes rendering it safe, and the client-side
row rebuild (D22) must build cells via `textContent`, never `innerHTML`. The
client controller's row rebuild prints the same `visibility` string verbatim in
its label cell. The layout and styling are D6.

**The JSON data island (the client's data source, D22).** Alongside the visible
table, `landing.html` emits the same rows **once more** as a JSON array in an
inline `<script type="application/json" id="sites-data">…</script>`. Each element
is `{slug, name, url, visibility, createdBy, createdAt, createdAtSort}` — the
visible table and the island are rendered from the **one** `Sites` slice, so
they never disagree. `createdAtSort` (RFC3339 UTC) is the machine-sortable
timestamp D22 sorts on; `createdAt` stays the human display string. Each island
element's `url` is the **same server-computed absolute front-door URL** as the
row's anchor `href` (one URL per site, no second computation). An empty `Sites`
slice renders an empty island (`[]`). The island is what D22's client controller
parses; the server itself does no filtering, sorting, or pagination.

**The name is a link to the site.** Each row's name renders as
`<a href="{{.URL}}">{{.Name}}</a>`. `URL` is the site's **absolute front-door
URL**, built by the handler the same way the MCP layer builds its `url` field
(`internal/mcp/tools.go`'s `siteURL`): `baseURL + sites.Seg(v) + "/" + slug +
"/"`, where `Seg` maps the visibility to its tree segment (unlisted → `public`,
D16) and `baseURL` carries its trailing slash. There is **one** notion of a
site's URL: the landing link, the tool-reported `url`, and the per-row
**copy-URL button** (D6 markup / D22 behaviour) all carry the same string — the
copy control reuses this canonical URL, it does not compute a second one.
Absolute (not relative) so it matches the tool `url` and never encodes a second
scheme; the private link works for the viewer because the landing page is
already behind the same `/_session-authn` gate.

`baseURL` is computed once at the composition root and shared by the landing
handler and the MCP handler (unchanged).

The session-gate (D4) and the name+version heading (D6) are unchanged.

There is **no** `state/www/index.html` file: the index is the live-rendered
landing page, not a generated artifact.

**Rejected.**
- **Showing the slug as its own visible column beside the name.** For unlisted
  sites the slug is a 30-char token that wrecks the layout — the exact defect
  this Decision removes; for public/private sites the name usually resembles
  the slug anyway. The URL is one click away on the copy button, and the slug
  stays searchable via the island. Settled with the operator: name replaces
  slug in the table.
- **A generated `state/www/index.html` file; serving the index publicly;
  walking the filesystem; a relative link.** All previously rejected;
  unchanged.

**Verification.** (Rendered over the repo-real `share/www` tree via
`appkit/web.Load` with fixed inputs; handler ids drive the `GET /{$}` handler
against a real store, in-process.)
- R-ZI4K-DR4L — rendering `landing.html` with a `Sites` slice of three sites
  (one public, one private, one unlisted — the unlisted row carrying a token
  slug and a human name, each row a distinct creator and created-at) produces
  output containing, for each site, its **name** as the row's visible text, its
  visibility label rendered **verbatim** (`public`, `private`, and `unlisted`
  each appear on their row), its creator, and its created-at — and the unlisted
  row's 30-char token slug does **not** appear as the anchor's link text (the
  name does). An implementation that shows the token as the row identity, or
  collapses unlisted into the public label, fails.
- R-RC42-WMDU — rendering `landing.html` with an empty `Sites` slice returns
  without error and still contains the service version (the page degrades to the
  heading with no site rows).
- R-ZJCG-RIVA — driving the `GET /{$}` handler (constructed with a fixed
  `baseURL`) against a store seeded with a **public** site (slug `x-site`, name
  `X Site`) and a **private** site (slug `y-site`, name `Y Site`) renders
  `X Site` inside an anchor whose `href` is `<baseURL>public/x-site/` and
  `Y Site` inside an anchor whose `href` is `<baseURL>private/y-site/` — the
  anchor text is the **name**, the `href` is built from the **slug**. A wrong
  implementation that links the slug text, renders the name as plain text,
  drops the trailing slash, uses the wrong visibility segment, or points both
  rows at one href fails. (Handler + real store substrate, in-process.)
- R-ZKKD-5ALZ — rendering `landing.html` (over the repo-real `share/www`) with
  a `Sites` slice of three sites (one per visibility) emits a `<script
  type="application/json" id="sites-data">` whose parsed JSON is an array of
  three objects, each carrying **`slug`**, **`name`**, `url`, a `visibility`
  string equal to that row's enum value, `createdBy`, `createdAt`, and a
  `createdAtSort` RFC3339 timestamp; rendering with an **empty** slice emits an
  island parsing to `[]`. An implementation that omits `name` or `slug` from
  the island, or emits malformed JSON, fails.
- R-ZLS9-J2CO — driving the `GET /{$}` handler (fixed `baseURL`) against a
  store seeded with an **unlisted** site (token slug, name `Client Preview`)
  renders `Client Preview` inside an anchor whose `href` is
  `<baseURL>public/<token>/` — the **public** segment (unlisted serves from the
  ungated public tier, D16) — byte-identical to the island element's `url` and
  to the MCP `url` for the same site. An implementation that invents an
  `unlisted/` URL segment, gates the URL as private, or links the token as the
  visible text fails.
- R-IEWI-3MXP — driving the `GET /{$}` handler (fixed `baseURL`) against a
  store seeded with a **public** and a **private** site, each island element's
  `url` is **byte-identical** to that same row's visible anchor `href` — the
  island and the table report the one canonical per-site URL. An implementation
  that recomputes the island URL differently, drops the trailing slash, or
  desyncs it from the anchor fails. (Handler + real store substrate,
  in-process.)
