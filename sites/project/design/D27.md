# Decision 27 — The unlisted-name token generator

**Decision.** An unlisted site's name *is* its credential, so it is generated,
never chosen. `internal/sites` gains one function:

```go
// NewToken returns a freshly generated unlisted-site name: exactly 30
// characters drawn uniformly from the lowercase base32 alphabet "abcdefghij
// klmnopqrstuvwxyz234567" using crypto/rand. 30 × log2(32) = 150 bits of
// entropy. The result always satisfies the site slug grammar
// ^[a-z0-9][a-z0-9-]{0,62}$, so the ordinary name column, validation, and URL
// scheme serve it unchanged. Panics only if crypto/rand fails (fail loudly —
// a box whose CSPRNG is broken must not mint credentials).
func NewToken() string
```

Properties the choice pins down:

- **Alphabet `a-z2-7` (lowercase base32).** Every character is legal in a slug,
  the token can never collide with a hyphenated human name's *shape* by
  accident (no hyphens, and 30 unbroken random characters are visually
  unmistakable as generated), and it excludes `0`/`1` lookalike ambiguity when
  a human reads a link aloud.
- **Length 30 → 150 bits.** Far beyond guessability (a scanner testing a
  billion names a second for a billion years touches ~2⁸⁵ of the 2¹⁵⁰ space),
  comfortably under the 63-char slug cap, short enough to paste anywhere.
- **`crypto/rand`, never `math/rand`.** The token is a credential; a seedable
  PRNG would make every "random" URL derivable from the seed.
- **Uniqueness is the store's job, not the generator's.** `NewToken` is
  stateless; the `sites.name` primary key rejects the astronomically unlikely
  collision and the create path retries once on `ErrExists` (D20).

The MCP layer consumes the generator through an injectable seam (a function
field on the tool-handler struct defaulting to `sites.NewToken`) so the
collision-retry path is testable with a deterministic fake (D20's
R-HCSI-RGCG); production always uses the real generator.

**Rejected.**
- **Hex / base64url / mixed-case alphabets.** Uppercase and `_`/`-`-adjacent
  characters either violate the slug grammar or degrade read-aloud clarity; hex
  needs 38 chars for the same entropy for no benefit.
- **A shorter token (16–20 chars).** Still unguessable in practice, but the
  cost of 10 more characters is zero and 150 bits removes the conversation
  entirely.
- **UUIDs.** 122 bits, hyphenated shape (visually similar to a plausible slug
  scheme), and a format that invites parsing; a flat random string invites
  none.
- **Checking uniqueness inside the generator (a store round-trip).** A stateless
  generator plus the primary key is simpler and race-free; a pre-check reads as
  a guarantee it cannot actually give.

**Verification.**
- R-H7WX-8DDO — `NewToken()` returns a string that matches `^[a-z2-7]{30}$` and
  passes the site slug validation used by `create`; two successive calls return
  **different** strings; over 200 calls every character position takes more than
  one distinct value (a constant or low-entropy stub fails; the real
  crypto/rand source trivially passes).
