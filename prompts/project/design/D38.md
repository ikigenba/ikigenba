# Decision 38 — OpenAI subscription authentication (`auth: "sub"`)

## Decision

prompts can authenticate OpenAI inference with a **ChatGPT subscription credential** instead of the metered `OPENAI_API_KEY`. The choice is per-config: the `auth` key (D2) selects `"key"` (the default, empty ⇒ key) or `"sub"`; validation (D3) gates the vocabulary. This decision owns the credential file contract, its path resolution, the store lifecycle, and the factory wiring.

**The credential file.** The file is a **raw OAuth token-endpoint JSON response**, exactly as the `oauth` CLI (`github.com/ikigenba/oauth`, installed to `/usr/local/bin` on every box by opsctl init-box) writes to stdout. It is **operator-provisioned** — the login flow is manual and outside this spec; the design assumes the file exists at the resolved path when `auth:"sub"` is used. agentkit's `openai/subscription` package owns the format: `subscription.Load(path)` parses it, derives the ChatGPT account id from the JWT claim, and the resulting `*subscription.Store` auto-refreshes near expiry and **atomically rewrites the file** (the refresh-token lineage rotates). Two consequences are contractual:

- The file must be **writable by the service process** and must survive deploys — on the box that is `/opt/prompts/state/auth.json` (mode 0600, `prompts:prompts`; `state/` is the durable, service-owned tree deploys never touch).
- The file is a **single lineage**: exactly one process owns and refreshes it. Locally, each developer logs in separately to their own file; a lineage is never copied between machines.

**Path resolution.** `internal/provider` owns it:

```go
// ResolveAuthPath returns PROMPTS_OPENAI_AUTH_PATH when set, else
// filepath.Join(filepath.Dir(getenv("PROMPTS_DB_PATH")), "auth.json") —
// the state-dir default (/opt/prompts/state/auth.json in production).
func ResolveAuthPath(getenv func(string) string) string
```

Production sets no env var (the state-dir default lands on the box path above). Local dev pins a durable per-user file in `prompts/.envrc`:

```sh
export PROMPTS_OPENAI_AUTH_PATH="$HOME/.secrets/openai-auth.json"
```

(`~/.secrets` file contents are secret; the service reads and rewrites the file, tooling never prints it.)

**The store: one lazy singleton per process.** `internal/provider` owns the handle:

```go
type SubAuth struct { path string; mu sync.Mutex; store *subscription.Store; err error }

func NewSubAuth(path string) *SubAuth
func (a *SubAuth) Available() bool                          // file exists at path (validation probe, D3)
func (a *SubAuth) Store() (*subscription.Store, error)      // lazy subscription.Load, exactly once, cached
```

`Store()` calls `subscription.Load(a.path)` on **first** use and caches the result (success or failure is re-attempted only while nil — a failed load is retried on the next call so an operator can drop the file in without a restart; a loaded store is permanent). The singleton matters: `subscription.Store` serializes refreshes internally, so every sub-authenticated call in the process — runner sessions and `/complete` alike — must share one Store or concurrent refreshes race the rotating refresh token. Service startup does **not** touch the file; a box that never uses `auth:"sub"` never needs it (lazy, fail-loud-at-use).

**Factory wiring.** `internal/provider` gains a builder constructor preserving the existing seam type (D4's `func(prompt.Config, func(string) string) (agentkit.Provider, error)`):

```go
func NewBuilder(sub *SubAuth) func(prompt.Config, func(string) string) (agentkit.Provider, error)
```

The returned function dispatches on `cfg.Auth`: `"sub"` (already validated: provider `openai`, no `base_url`) obtains `sub.Store()` and returns `openai.New(openai.Subscription(store))`; a store error fails the build **naming the configured path**. Every other value takes today's key path (`Build`, unchanged). The composition root constructs `subAuth := provider.NewSubAuth(provider.ResolveAuthPath(os.Getenv))` once and injects `provider.NewBuilder(subAuth)` into both the runner and the inference executor, and `subAuth.Available` into validation (D3).

**Identity, cost, recording.** A subscription-authenticated provider identifies as `openai.subscription` (agentkit's provider name) versus `openai` for key auth; requests go to agentkit's pinned chatgpt.com base URL. Costs agentkit resolves for subscription turns are **notional API-rate equivalents**, not subscription spend — `calls`/`usage` rows record them as-is, and the stored config JSON already carries `auth`, so no schema change (D28 untouched). Embeddings are **excluded**: `openai.Subscription` is a Responses-surface credential; `BuildEmbedder` keeps requiring `OPENAI_API_KEY` regardless of any `auth` setting.

**Live-substrate stance.** The real external contracts — the token refresh against `auth.openai.com` (rotation, atomic rewrite, account claim) and chatgpt.com accepting a subscription-authenticated conversation — are proven by agentkit's own `openai/subscription` and provider integration tests, the same boundary D29 drew for key-authenticated providers. prompts' ids stay at its own seams, pinned by the `openai.subscription` provider-name discriminator.

## Rejected

**Eager `subscription.Load` at service start** — couples every start to an optional credential and breaks boxes that never use sub; lazy-at-first-use with create-time file-presence validation keeps failures early where it matters (authoring) and loud where it can't be earlier (run).

**A `*subscription.Store` per build call** — races the rotating refresh-token lineage across concurrent calls; the store's internal serialization only protects callers sharing one instance.

**Delivering the credential via SSM/env like other secrets** — the file is mutable state (rotated on refresh), not a static secret; a workstation-pushed copy goes stale after the first on-box refresh. The state tree is the correct home.

**Changing the factory seam signature to carry the store** — ripples through `Runner.buildProvider` and the inference executor's field type for no gain; a closure constructor (`NewBuilder`) keeps the seam byte-identical.

**Honoring `base_url` with `auth:"sub"`** — the subscription credential pins its own base URL inside agentkit; letting a config override silently fight it violates fail-loud. Validation rejects the combination (D3).

## Verification

- R-T319-YQNF — `NewBuilder`'s function, given a config with `auth:"sub"`, provider `openai`, and a valid token-response file at the `SubAuth` path, returns a provider whose name is `openai.subscription`; the same config with `auth` empty or `"key"` (and the key env var set) returns a provider whose name is `openai`.
- R-T496-CIE4 — after one successful `auth:"sub"` build, deleting the credential file and building again with the same `SubAuth` still succeeds (the store was loaded exactly once and is cached; no per-call re-read).
- R-T6OZ-41VI — an `auth:"sub"` build whose `SubAuth` path is missing or contains invalid JSON returns an error whose message contains the configured path.
- R-T7WV-HTM7 — `ResolveAuthPath` returns `PROMPTS_OPENAI_AUTH_PATH` verbatim when that env var is set, and `filepath.Join(filepath.Dir($PROMPTS_DB_PATH), "auth.json")` when it is not.
