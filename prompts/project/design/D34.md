# Decision 34 — The `ui/` namespace: one session-gated prefix for the human browse surface

## Decision

prompts' human web surface grows from a single name+version card into a **browse
UI** (the pages themselves are D35). This Decision owns the **URL space and the
gate**: every human page lives under one path prefix, **`/ui/`**, and that whole
prefix is session-gated by **one** nginx location — so adding a page never
touches the nginx fragment again, and the bearer surface (`/mcp`, PRM, `/health`)
can never be shadowed by accident.

**The three moves:**

1. **One nginx location gates the whole namespace** (`prompts/etc/nginx.conf`).
   A new **prefix** location beside the existing exact-match root, mirroring the
   session tier the root already uses (D10): `auth_request /_session-authn`, the
   `error_page 401 = @login_bounce;` opt-in (D20), all four owner headers
   captured and forwarded (the D10 four-header contract, same `$prompts_owner*`
   variables), no `X-Client-Id` (the session tier has no client identity):

   ```nginx
   # Browse UI — every human page, gated by the dashboard browser SESSION.
   # One prefix location: adding a UI page never edits this fragment again.
   location /srv/prompts/ui/ {
       auth_request /_session-authn;
       error_page 401 = @login_bounce;
       auth_request_set $prompts_owner         $upstream_http_x_owner_email;
       auth_request_set $prompts_owner_id      $upstream_http_x_owner_id;
       auth_request_set $prompts_owner_name    $upstream_http_x_owner_name;
       auth_request_set $prompts_owner_picture $upstream_http_x_owner_picture;
       proxy_set_header X-Owner-Email   $prompts_owner;
       proxy_set_header X-Owner-Id      $prompts_owner_id;
       proxy_set_header X-Owner-Name    $prompts_owner_name;
       proxy_set_header X-Owner-Picture $prompts_owner_picture;
       proxy_pass http://127.0.0.1:3002/ui/;   # strips /srv/prompts, keeps /ui/
       proxy_set_header Host $host;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_http_version 1.1;
   }
   ```

   A longest-prefix match beats the bearer prefix `location /srv/prompts/`, so
   `/srv/prompts/ui/...` takes the session tier while `/mcp`, the PRM doc, the
   `/feed` 404 guard, and `/static/` keep their existing locations untouched.
   The loopback port stays the literal `3002` like every other line of the
   fragment.

2. **The mount root redirects into the namespace.** The `GET /{$}` handler in
   `registerRoutes` no longer renders a page; it issues a redirect with the
   **relative** target `ui/` (`http.Redirect(..., "ui/", http.StatusSeeOther)`).
   Relative, so it composes with the nginx prefix strip: a browser at
   `/srv/prompts/` lands on `/srv/prompts/ui/`; an httptest request at `/` lands
   on `/ui/`. The root's nginx location (session gate, four headers, login
   bounce) is unchanged — the redirect happens *behind* the gate, so an
   unauthenticated browser still bounces to login at the root, and an
   authenticated one arrives at the prompts tab. The root route keeps its D10
   properties: exact-match `{$}` (shadows nothing) and ungated in-process.

3. **The in-process `/ui/` routes are ungated, like every human route.** The
   page handlers (D35) mount as plain `rt.HandleFunc("GET /ui/...", ...)` — no
   `RequireIdentity` (that wrapper is the bearer gate), no identity-header
   reads. nginx is the sole trust boundary; the pages are deliberately **not
   owner-scoped** (any logged-in dashboard user browses all rows — the same
   whole-box posture as the `calls`/`usage` MCP tools, D32).

**Asset paths from a nested namespace.** UI pages live one and two path segments
deep (`/ui/runs`, `/ui/runs/{id}`), so the document-relative asset links the old
root-level landing page used (`static/tokens.css`, D13) would resolve *inside*
`/ui/` and miss. UI templates therefore reference the shared assets by the
**mount-literal absolute path** — `/srv/prompts/static/tokens.css`,
`/srv/prompts/static/fonts/…` — the same "the mount is a literal" stance the
nginx fragment itself takes (D14 Rejected records that registry does not own
mounts). The existing session-gated `location /srv/prompts/static/` (D13) serves
them unchanged. D13's font/asset ids re-anchor to this path form.

## Rejected

- **A session-gated location per page path** (`/srv/prompts/runs`,
  `/srv/prompts/prompts/`, …) — cleaner top-level URLs, but every future page
  is another fragment edit and another chance to shadow the bearer surface; the
  fragment's job is gating tiers, not routing pages.
- **Serving the prompts tab at the bare root instead of redirecting** — leaves
  the rest of the UI under `/ui/` with the default tab outside it: two homes,
  asymmetric links, and the root page would need its own asset-path special
  case. One namespace, one redirect.
- **Client-side redirect (meta refresh / JS)** — an HTTP redirect is one
  round-trip, works without JS, and keeps the root handler a pure function.
- **Passing the mount into templates (`<base href>`/`Mount` field)** — D13
  already rejected threading a mount value; the mount is a committed literal in
  this service's own fragment, so writing the same literal in its own templates
  adds no coupling that does not already exist.

## Verification

- R-ZW7P-88WL — `GET /` (exact root, httptest against the composition-root
  handler) returns **303 See Other** with `Location: ui/` (the relative form,
  not `/ui/` and not an absolute URL) — so the redirect lands on
  `/srv/prompts/ui/` through nginx and on `/ui/` in tests.
- R-ZXFL-M0NA — `etc/nginx.conf` contains a `location /srv/prompts/ui/` block
  whose body carries `auth_request /_session-authn;`,
  `error_page 401 = @login_bounce;`, all four owner-header
  `auth_request_set`/`proxy_set_header` pairs (`X-Owner-Email`, `X-Owner-Id`,
  `X-Owner-Name`, `X-Owner-Picture` from the `$prompts_owner*` variables), **no**
  `X-Client-Id` forwarding, and `proxy_pass http://127.0.0.1:3002/ui/;`. A block
  missing the session gate, the bounce, any owner header, or proxying to a
  different upstream path fails. *(string assertions over `etc/nginx.conf` in
  `cmd/prompts/web_test.go`, like R-7NY0-UIO6/R-7P5X-8AEV)*
- R-ZYNH-ZSDZ — a `/ui/` page request (httptest) with **no** identity header
  and no bearer still returns its page — the UI routes are mounted without
  `RequireIdentity`, trusting nginx as the sole gate.
