# Decision 1 — Module dependency

## Decision

`prompts/go.mod` requires the published agentkit at the release that carries the model catalog and the OpenRouter provider:

```
require github.com/ikigenba/agentkit v0.6.0
```

There is no local `replace` for agentkit. The `appkit`, `eventplane`, and `registry` replace directives are unchanged.

The repo-root `go.work` resolves prompts' agentkit dependency from the module cache during local development, exactly as production (`GOWORK=off`) does. The import-guard tests continue to assert that no retired local-fork import path appears.

The v0.6.0 surface prompts consumes, beyond what v0.2.x already provided (research §2):

- the advisory `catalog` package (`Lookup`, `Resolve`, `ListByProvider`, `Entry`, `ReasoningSpec`) — research §5;
- typed-credential provider constructors (`New(cred Credential, opts ...Option)` on every provider sub-package);
- the `github.com/ikigenba/agentkit/openrouter` provider;
- consumer-owned cost resolution via `Conversation.Pricing`.

## Rejected

**Pin an intermediate release (v0.4.x/v0.5.x)** — rejected: v0.6.0 is the first release whose catalog carries the OpenRouter-routed vendors (Grok, DeepSeek, Kimi) and the corrected `glm-5.1` reasoning spec; the whole point of this change is that inventory.

**Vendor a model list inside prompts** — rejected: the catalog is agentkit's authoritative, versioned statement of the supported provider/model/effort combinations; a private copy would drift.

## Verification

Pure structural decision — enables D2–D4, D7, D9. No behavior ids of its own; correctness is implied when the suite compiles against v0.6.0 and the behavioral ids of the dependent decisions pass.
