# Decision 1 — Module dependency

## Decision

`prompts/go.mod` replaces its `require agentkit v0.0.0` + `replace agentkit => ../agentkit` pair with a direct dependency on the published package:

```
require github.com/ikigenba/agentkit v0.1.0
```

The local `replace` for `agentkit` is removed entirely. The `appkit` and `eventplane` replace directives are unchanged.

The `go.work` file at the repo root already lists `./agentkit` (module path `agentkit`). It does **not** contain a `replace github.com/ikigenba/agentkit => ./agentkit` mapping, so the workspace correctly resolves prompts' new dependency from the module cache (`~/.local/go/pkg/mod/github.com/ikigenba/agentkit@v0.1.0/`) during local development, not from the local tree.

All imports of `agentkit/...` sub-packages in the `prompts` module are replaced with imports of `github.com/ikigenba/agentkit` and its provider sub-packages (`github.com/ikigenba/agentkit/anthropic`, `.../openai`, `.../google`, `.../zai`). No other service module is touched.

## Rejected

**Keep the local agentkit and the published agentkit side by side** — the local module would be renamed (e.g., `agentkitv0`) while prompts imports both. Rejected: two agentkit packages in the same binary adds confusion and is unnecessary once the migration is complete.

**Use a `go.work` replace directive** — add `replace github.com/ikigenba/agentkit => ./agentkit` to `go.work` so local dev and production use the same tree. Rejected: the whole point of this migration is to consume the published package; local dev should exercise it too.

## Verification

Pure structural decision — enables all subsequent decisions. No behavior ids of its own; correctness is implied when the suite compiles and the tests in D2–D9 pass.
