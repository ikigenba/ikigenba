# Decision 3 — Validation

## Decision

`validateConfig` in `prompts/internal/prompt/service.go` is the create/update gate. It validates the stored config against the **agentkit model catalog** (`github.com/ikigenba/agentkit/catalog`) — the authoritative table of supported provider/model/effort combinations — and returns the (possibly provider-filled) config on success:

```go
func validateConfig(c Config, getenv func(string) string) (Config, error)
```

Callers pass `os.Getenv`; tests inject a fake environment. On success the returned Config is what gets stored (it differs from the input only when `provider` was derived).

**Provider env var map** (contractual — must match exactly):

| provider   | env var              |
|------------|----------------------|
| anthropic  | `ANTHROPIC_API_KEY`  |
| openai     | `OPENAI_API_KEY`     |
| google     | `GEMINI_API_KEY`     |
| zai        | `ZAI_API_KEY`        |
| openrouter | `OPENROUTER_API_KEY` |

**Validation steps** (in order — fail fast, each failure a `*ValidationError`):

1. **Model in catalog.** `catalog.Lookup(c.Model)`; a miss rejects, naming the unknown model. Chat use requires a non-nil `entry.Pricing` (an embedding-only entry is not a runnable prompt model and rejects the same way).
2. **Provider resolution.** If `c.Provider` is empty, set it to `entry.Provider` (the catalog default) — the derived value is stored. If non-empty, it must appear in the env var map (unknown strings reject) **and** the catalog must route the model through it: `catalog.Resolve(c.Provider, c.Model)` must return `ok`. A provider the catalog cannot route that model through rejects, naming both.
3. **Reasoning control vs the model's `ReasoningSpec`.** The first non-zero reasoning control (precedence per D2: `effort` → `thinking_level` → `thinking_budget` → `thinking:false`) is mapped to its `agentkit.ReasoningValue` form (`Level`, `Level`, `Budget`, `DisableReasoning`) and checked with `entry.Reasoning.Accepts(v)`. Not accepted rejects, naming the model's accepted vocabulary (its levels, or its budget range and sentinels, or that reasoning cannot be disabled). A model with a nil `Reasoning` spec accepts no reasoning control — any set control rejects. No control set is always valid (model default).
4. **API key present.** The resolved provider's env var must be non-empty via `getenv`.

The old blanket `provider: "anthropic"` default (`applyProviderDefault`) is removed — the default is per-model, from the catalog. `providerRegistry` and every per-provider `Pricing` lookup are removed (the v0.6.0 providers have no `Pricing` method; the catalog is the truth).

**Import changes**: `service.go` drops the provider sub-package imports (`agentkit/anthropic`, `.../openai`, `.../google`, `.../zai`) and imports `github.com/ikigenba/agentkit` + `github.com/ikigenba/agentkit/catalog` instead.

`Import` (Dropbox adoption) keeps seeding `importDefaultModel = "claude-sonnet-4-6"` — a catalog entry — and now stores its derived provider the same way as create.

## Rejected

**Advisory validation (agentkit's own stance)** — the catalog upstream never blocks a model from being sent; prompts deliberately does. A hosted service's stored configs should fail at authoring time, not mid-run (operator decision).

**Synthesizing OpenRouter wire slugs for unrouted models** — rejected: prompts would be guessing slugs the catalog does not vouch for; an explicit provider is honored exactly when the catalog carries the route. Widening coverage is an agentkit catalog change.

**Validate reasoning at run time only** — rejected for the same fail-fast reason as models; the catalog's `ReasoningSpec.Accepts` makes the create-time check one call.

## Verification

- R-JVR2-WAUP — `validateConfig` with an unknown provider string (e.g. `"badprovider"`) returns a non-nil `*ValidationError`.
- R-JWYZ-A2LE — `validateConfig` with a model name absent from the agentkit catalog returns a non-nil `*ValidationError`.
- R-1ONM-PPDU — `validateConfig` with an empty provider and a catalog model returns a Config whose `Provider` equals that model's catalog default provider (e.g. `"grok-4.5"` → `"openrouter"`, `"gpt-5.5"` → `"openai"`).
- R-1PVJ-3H4J — `validateConfig` with an explicit provider the catalog does not route the model through (e.g. `provider: "google"`, `model: "claude-sonnet-4-6"`) returns a non-nil `*ValidationError`.
- R-1R3F-H8V8 — `validateConfig` with an enum-reasoning model and an `effort`/`thinking_level` value outside the model's catalog `Levels` (e.g. `effort: "ultra"` on `claude-sonnet-4-6`) returns a non-nil `*ValidationError`.
- R-1SBB-V0LX — `validateConfig` with a range-reasoning model and a `thinking_budget` outside the model's catalog range and sentinels (e.g. `1_000_000` on `gemini-2.5-pro`, whose max is 32768) returns a non-nil `*ValidationError`.
- R-1TJ8-8SCM — `validateConfig` with `thinking: false` on a model whose catalog spec has `CanDisable: false` (e.g. `claude-fable-5`) returns a non-nil `*ValidationError`.
- R-1UR4-MK3B — `validateConfig` with a reasoning control the model's catalog spec accepts (a listed level, an in-range budget, or a sentinel value) returns a nil error.
- R-1VZ1-0BU0 — `validateConfig` with an openrouter-resolved model and `OPENROUTER_API_KEY` empty in the injected environment returns a non-nil `*ValidationError` naming `OPENROUTER_API_KEY`.
- R-JY6V-NUC3 — `validateConfig` with a valid provider and catalog model, but the provider's API key env var empty, returns a non-nil `*ValidationError`.
- R-JZES-1M2S — `validateConfig` with a valid provider, catalog model, and the API key env var set returns a nil error.
