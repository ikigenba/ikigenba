# Decision 7 — Runner

## Decision

`runner.go` drives each run as a single `agentkit.Conversation.Send` + stream drain.

**`execute` shape** (key steps, after the pinned-input read):

```go
// 1. Build the provider (D4)
provider, err := r.buildProvider(cfg, os.Getenv)

// 2. Resolve the wire model and pricing from the catalog. The stored model is
//    the catalog name (D2); the provider must be sent the route's wire form
//    (e.g. openrouter carries "deepseek/deepseek-v4-flash"). Validation (D3)
//    already guaranteed resolvability at create/update; a pinned config that
//    no longer resolves (catalog moved under it) fails the run loudly.
routeProvider, wireModel, entry, ok := catalog.Resolve(cfg.Provider, cfg.Model)
// !ok → terminal run error naming provider+model

// 3. Build the Conversation
builtins := tools.All(r.sandbox.Root(run.ID), r.sourcePortAllowed) // D5/D21 — eager, always surfaced
suiteGroups := r.discover(ctx, run.OwnerEmail, run.PromptID) // D6 — deferred groups

conv := &agentkit.Conversation{
    Provider:          provider,
    Model:             wireModel,     // the route's wire form, not the catalog name
    Pricing:           entry.Pricing, // consumer-owned cost resolution (v0.6.0)
    System:            buildSystem(sysPrompt),
    Log:               logFile,       // LogRecord JSONL written here
    Gen:               buildGenSettings(cfg),
    Retry:             buildRetryPolicy(cfg),
    MaxToolIterations: cfg.ToolLoopLimit, // 0 → agentkit default (1000)
    Tools:             builtins,          // sandbox tools only
    DeferredTools:     suiteGroups,       // suite tools, loaded on demand (D19)
}

// 4. Send the single user turn
userMsg := buildUserMessage(userPrompt, eventBytes)
stream := conv.Send(ctx, userMsg)

// 5. Drain — logging is a side-effect of the Log writer; we drain for
//    the error and usage
for range stream.Events() {}

runErr := stream.Err()
usage := stream.Usage()
cost := stream.Cost()
conv.Close() // emits the summary LogRecord

// 6. Serialize usage for the run row
usageJSON := serializeUsage(usage, cost)
```

**Cost resolution** is agentkit's v0.6.0 chain: a provider-reported cost wins; otherwise `Conversation.Pricing` (set here from the catalog entry) prices the usage; the zero-cost `WarnCostUnknown` path is unreachable for prompts because every storable model is catalog-backed with rates.

**`buildSystem`**: concatenates the framing prompt with the optional system prompt, separated by `"\n\n"`.

**`buildUserMessage`**: for a manual run returns `userPrompt` directly. For an event-triggered run appends the event preamble + pretty-printed JSON as a second paragraph.

**`buildGenSettings`**: maps Config fields to `agentkit.GenSettings`. Reasoning precedence (effort → thinking_level → thinking_budget → thinking, first non-zero wins). Duration strings parsed via `time.ParseDuration`.

**`buildRetryPolicy`**: maps `MaxAttempts`, `BaseDelay`, `MaxDelay`, `MaxElapsed`, `IgnoreRetryAfter` from Config to `agentkit.RetryPolicy`. Zero values leave each field at the agentkit default.

**`serializeUsage`**: marshals `agentkit.Usage` and `agentkit.Cost` into the `usage_json` column format.

**Run output format**: `logFile` receives the agentkit's native `LogRecord` JSONL (`turn_start`, `message`, `tool_use`, `tool_result`, `usage`, `turn_end`, `summary`).

**Test seam**: `Runner.buildProvider` (injectable, D4) and `Runner.discover` (injectable, D6 — returning `[]agentkit.DeferredToolGroup`) are the two seams. Tests inject a fake `buildProvider` returning a stub `agentkit.Provider` (the v0.6.0 interface: `RoundTrip` + `Name` only) whose `RoundTrip` records the incoming `*agentkit.Request` (so tests can assert the wire model) and returns a pre-canned `*agentkit.RoundTrip` (no events, `FinishStop`, no error, and a chosen `Usage`) so the conversation completes in one turn.

## Rejected

**Send the catalog name as `Conversation.Model`** — works only when the route's wire form equals the catalog name (native providers); openrouter routes need vendor-namespaced slugs. Resolving once at spawn covers both uniformly.

**Re-validate the full config at spawn** — validation is the create/update gate (D3); the runner only re-resolves the route it needs, and fails loudly if the pinned config no longer resolves.

**Multi-turn conversation** — prompts runs a single user turn per run (one `Send` call). Multi-turn is out of scope.

## Verification

- R-K5I9-YGS9 — a run seeded with `provider: "fake"` uses the injected fake provider's `RoundTrip` (verified by injecting a provider that records calls); runs with different provider strings use different injected providers.
- R-1X6X-E3KP — a run pinned to a catalog model whose route carries a distinct wire form (e.g. `provider: "openrouter"`, `model: "deepseek-v4-flash"`) reaches the injected provider with `Request.Model` equal to the catalog route's wire slug (`"deepseek/deepseek-v4-flash"`), not the stored catalog name.
- R-1ZMQ-5N23 — after a run whose fake provider reports non-zero token usage and no provider-side cost, the run row's `usage_json` carries a cost greater than zero, computed from the model's catalog rates via `Conversation.Pricing`.
- R-K6Q6-C8IY — after a run completes, `output.jsonl` contains at least one line parseable as a `LogRecord` with `type: "turn_start"` and one with `type: "summary"`.
- R-K7Y2-Q09N — after a successful run, the run row's `usage_json` is a non-empty JSON object (not `""`).
- R-K95Z-3S0C — a run started with a TTL of 50ms against a fake provider that blocks until context cancellation is marked `failed` with `error: "run TTL exceeded"`.
