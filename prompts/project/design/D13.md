# Decision 13 — Self-serve the pages' fonts and eliminate the FOUT

**Problem (as originally found).** The v1 landing page flashed: it painted with
the fallback system font, then visibly swapped to the web fonts once they
loaded — and the swap was driven by the **dashboard's** assets, not prompts'
own, because origin-absolute `/static/…` references resolve at the apex (nginx
routes apex `/static/` to the dashboard; nginx strips inbound prefixes but does
not rewrite outbound HTML/CSS). That violated the suite requirement that every
service be 100% independent, and `font-display: swap` plus no preload made the
swap visible.

**Decision.** prompts' human pages load prompts' **own** on-disk CSS and
fonts, swap-free. Four coordinated rules, applied to every UI page template
(D35) and the shared `share/www/static` assets; no Go logic beyond the chassis
static mount (`Spec.WWW`, serving `GET /static/`):

1. **Pages link the stylesheet by the mount-literal absolute path.** Every UI
   page's `<head>` links `href="/srv/prompts/static/tokens.css"` — prompts'
   own copy through its own session-gated static location — never the bare
   origin-absolute `/static/tokens.css` (the apex borrow). The mount-literal
   form (D34) is depth-independent, so it works from `/ui/`, `/ui/runs`, and
   `/ui/runs/{id}` alike, where the old document-relative
   `static/tokens.css` of the root-level landing card would resolve inside
   `/ui/` and miss.

2. **`tokens.css` self-serves the fonts and never swaps.** On all four
   `@font-face` blocks: `font-display: optional` (forbids a post-first-paint
   swap), and `src: url('fonts/X.woff2')` — relative to the stylesheet, so it
   resolves to `/srv/prompts/static/fonts/…`, prompts' own copy.

3. **Pages preload the above-the-fold fonts** in `<head>`, beside the
   stylesheet link:

   ```html
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="/srv/prompts/static/fonts/space-grotesk.woff2">
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="/srv/prompts/static/fonts/ibm-plex-sans.woff2">
   ```

   The **display** (`Space Grotesk`) and **body** (`IBM Plex Sans`) families,
   painted immediately. `crossorigin` is **mandatory** (fonts fetch in CORS
   mode; without it the preload double-fetches and is wasted). The preload
   `href` resolves to the same URL as the `@font-face` `src`, so the preloaded
   bytes satisfy the font request and first paint uses the real font. The mono
   family is **not** preloaded (minor; `optional` already prevents its jar).

4. **nginx session-gates prompts' own static path** (`etc/nginx.conf`) — the
   `location /srv/prompts/static/` block with `auth_request /_session-authn`
   proxying to the chassis static mount, so a browser session can fetch the
   self-served CSS/fonts:

   ```nginx
   location /srv/prompts/static/ {
       auth_request /_session-authn;
       proxy_pass http://127.0.0.1:3002/static/;
       proxy_set_header Host $host;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_http_version 1.1;
   }
   ```

**Boundary.** D13 owns the pages' asset *loading* (the stylesheet link, the
`@font-face` rules, the preloads) and the nginx static gate. It does not own
the pages' content (D35), the Home link (D12), the routes (D34), or the token
values.

**Rejected.**
- **`font-display: block` / `fallback`.** `block` is FOIT (invisible text then
  pop-in); `fallback` still permits a late swap in its window. `optional` is
  the only value that forbids a post-paint swap.
- **Origin-absolute `/static/…` references.** Resolve at the apex — the
  dashboard's assets — recreating the dependency this Decision removes.
- **Document-relative references (`static/tokens.css`).** Correct at the
  mount root only; broken one segment deep. Superseded by the mount-literal
  form when the surface moved under `/ui/` (D34).
- **Introduce `<base href>` / thread a `Mount` template field.** The mount is
  a committed literal in this service's own nginx fragment; writing the same
  literal in its own templates adds no new coupling (D34 Rejected).

**Verification** (httptest against the chassis-rendered UI pages and the
chassis static mount, both over the repo-real `prompts/share/www` tree, and a
fragment-grep against `etc/nginx.conf`).

- R-DFKP-IVZU — **No `@font-face` declares `font-display: swap`; all use
  `optional`.** The served `GET /static/tokens.css` contains
  `font-display: optional` in every `@font-face` block and **no**
  `font-display: swap` occurrence.
- R-DGSL-WNQJ — **Font `src` is stylesheet-relative, not origin-absolute.**
  The served `tokens.css` contains **no** `url('/static/fonts/` occurrence and
  each `@font-face` `src` uses `url('fonts/…woff2')` — resolving to prompts'
  own `/srv/prompts/static/fonts/…`.
- R-DI0I-AFH8 — **Every UI page links prompts' own tokens.css.** Each rendered
  UI page's `<head>` contains `href="/srv/prompts/static/tokens.css"` and
  **no** bare origin-absolute `href="/static/tokens.css"` — so the browser
  fetches prompts' own copy from any page depth, never the apex.
- R-DJ8E-O77X — **Every UI page's rendered `<head>` preloads the display and
  body fonts with `crossorigin`.** A
  `<link rel="preload" as="font" type="font/woff2" crossorigin …>` for both
  `space-grotesk.woff2` and `ibm-plex-sans.woff2`, each `href` beginning
  `/srv/prompts/static/fonts/` and resolving to the same path as the
  `@font-face` `src`. A missing `crossorigin`, a missing family, or a bare
  apex-absolute href fails this.
- R-DKGB-1YYM — **nginx session-gates prompts' own static path.**
  `etc/nginx.conf` contains a `location /srv/prompts/static/` block whose body
  carries `auth_request /_session-authn;` and
  `proxy_pass http://127.0.0.1:3002/static/;`. A static path left bearer-gated
  fails this. *(fragment grep against `etc/nginx.conf`)*
