# Decision 2 — Config struct

## Decision

`prompt.Config` in `prompts/internal/prompt/model.go` carries the full agentkit generation and retry surface. `model` is required; `provider` is **optional** — when omitted at create/update it is derived from the model's catalog entry (D3) and stored explicitly, so a stored config always carries a concrete provider. The 11 optional keys map directly to fields on `agentkit.Conversation`:

```go
type Config struct {
    // Required: the catalog model name. Optional: the provider; when empty it
    // is filled from the catalog entry's default provider before storage.
    Provider string `json:"provider,omitempty"`
    Model    string `json:"model"`

    // Generation controls (agentkit.GenSettings)
    Temperature    *float64 `json:"temperature,omitempty"`
    TopP           *float64 `json:"top_p,omitempty"`
    MaxTokens      int      `json:"max_tokens,omitempty"`
    Effort         string   `json:"effort,omitempty"`           // maps to Gen.Reasoning via agentkit.Level
    ThinkingBudget *int     `json:"thinking_budget,omitempty"` // maps to Gen.Reasoning via agentkit.Budget
    ThinkingLevel  string   `json:"thinking_level,omitempty"`  // maps to Gen.Reasoning via agentkit.Level
    Thinking       *bool    `json:"thinking,omitempty"`        // false → agentkit.DisableReasoning()

    // Retry policy (agentkit.RetryPolicy)
    MaxAttempts      int    `json:"max_attempts,omitempty"`
    BaseDelay        string `json:"base_delay,omitempty"`        // Go duration string, e.g. "500ms"
    MaxDelay         string `json:"max_delay,omitempty"`
    MaxElapsed       string `json:"max_elapsed,omitempty"`
    IgnoreRetryAfter bool   `json:"ignore_retry_after,omitempty"`

    // Loop and provider tuning
    ToolLoopLimit int    `json:"tool_loop_limit,omitempty"` // maps to Conversation.MaxToolIterations
    BaseURL       string `json:"base_url,omitempty"`        // any provider: alternate API root
}
```

**Storage**: Config is stored verbatim as `config_json` (TEXT) in the prompts table, marshalled as a JSON object. No migration of the JSON column structure is required — new keys are simply absent in old rows and default to zero values. Full-replacement update semantics: every `update` call replaces the entire Config; keys not supplied revert to zero/omitempty.

**Model storage**: the `model` field stores the **catalog model name** — the globally unique key of an `agentkit/catalog` entry (e.g. `"claude-sonnet-4-6"`, `"gpt-5.5"`, `"grok-4.5"`, `"kimi-k3"`, `"glm-5.2"`). It is never a vendor-namespaced wire slug (`deepseek/deepseek-v4-flash`); the runner resolves the wire form at spawn (D7). Free-form model strings outside the catalog are rejected at validation (D3).

**Reasoning precedence**: `effort`, `thinking_budget`, `thinking_level`, and `thinking` are mutually exclusive reasoning controls — only the first non-zero one is applied at run time (in that order), matching agentkit's single `Gen.Reasoning` field. The chosen control must be native to the model's catalog `ReasoningSpec` (validated in D3).

**`base_url`** applies to every provider: each provider sub-package (anthropic, openai, google, zai, openrouter) exposes `WithBaseURL`, and the factory (D4) applies the field uniformly.

## Rejected

**Keep `provider` required** — rejected: catalog model names are globally unique and every entry names its default provider, so requiring the field only creates a redundant chance to get it wrong. It stays *settable* to override routing (e.g. deliberately reaching a model through OpenRouter when the catalog routes it there).

**Store the wire model slug** — rejected: routing knowledge (which wire name a provider uses) is the catalog's, and it can change per provider; storing the stable catalog name keeps configs valid across routing changes.

**Embed duration fields as `time.Duration`** — rejected: `time.Duration` marshals as a raw number; Go duration strings ("500ms", "2m") are readable in stored JSON.

## Verification

- R-JTBA-4RDB — a Config with all 11 optional keys set marshals to JSON and unmarshals back with all values preserved exactly.
- R-JUJ6-IJ40 — updating a prompt with a Config that omits a previously-set optional key results in a stored config where that key is absent (full-replacement, not merge).
