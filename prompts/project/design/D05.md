# Decision 5 — Built-in sandbox tools

## Decision

The run's built-in sandbox tools live in `prompts/internal/tools/` and implement
the published `agentkit.Tool` interface. The toolset is **thirteen** tools: the
six file/shell tools, `Fetch` — the content-plane acceptor tool (D21) — and the
six file-share tools (D26 owns their contract).

**Package shape**:

```go
// prompts/internal/tools/tools.go
package tools

import "github.com/ikigenba/agentkit"

// ShareConfig locates the account file share's loopback filesystem API for the
// six file-share tools (D26). BaseURL is the share service's loopback base
// (registry-defaulted at the composition root, the existing DROPBOX_BASE_URL
// seam); ClientID is the caller identity sent as X-Client-Id on every mutating
// call ("prompts:" + the run's prompt id, the suite.go convention).
type ShareConfig struct {
    BaseURL  string
    ClientID string
}

// All returns the thirteen built-in tools confined to sandboxRoot.
// sandboxRoot is threaded in at call time (not package-level global) so tests
// supply a temp directory and the runner supplies the per-run sandbox path.
// sourcePortAllowed is the Fetch tool's loopback-port confinement seam (D21).
// share locates the file share's loopback API for the File* tools (D26);
// tests point BaseURL at an httptest server.
func All(sandboxRoot string, sourcePortAllowed func(port int) bool, share ShareConfig) []agentkit.Tool
```

Each tool is constructed with `agentkit.NewTool[In](name, description, fn)` over
a typed input struct (the JSON Schema is derived from the struct's
`jsonschema` tags), where `fn` is a closure over `sandboxRoot` (and, for
`Fetch`, `sourcePortAllowed`; for the `File*` tools, `share`) returning
`(string, error)` per `agentkit.Tool.Call`.

**Tool names and descriptions** (names are capitalized in the tool table; the
framing prompt refers to them in lowercase prose):

| tool  | description |
|-------|-------------|
| Bash  | Run a foreground shell command from the sandbox root. |
| Read  | Read a file inside the sandbox. |
| Write | Write a file inside the sandbox. |
| Edit  | Edit a file inside the sandbox by replacing text. |
| Glob  | Find files inside the sandbox using a glob pattern. |
| Grep  | Search files inside the sandbox for a regular expression. |
| Fetch | Fetch a suite content URL into the sandbox (D21 owns its full contract). |
| FileList | List entries in the account's file share (D26). |
| FileGet | Copy a file from the file share into the sandbox (D26). |
| FilePut | Copy a sandbox file into the file share (D26). |
| FileDelete | Delete a file or folder in the file share (D26). |
| FileMove | Move or rename an entry in the file share (D26). |
| FileMkdir | Create a folder in the file share (D26). |

The sandbox confinement logic (path validation against `sandboxRoot` — join,
Clean, resolve the longest existing ancestor's symlinks, verify containment
with `filepath.Rel`) is shared by every sandbox-path-taking tool — `Fetch`'s
`dest_path` and the `File*` tools' sandbox-side paths (`FileGet.dest_path`,
`FilePut.source_path`) included.

**No package-level state**: `tools.All(...)` is called once per run (in the
runner's `execute`), producing a fresh slice of tool values. The tool
implementations are stateless.

## Rejected

**Keep the local agentkit module for its tools only** — renaming the local module to something like `promptstools` and keeping a replace directive. Rejected: unnecessary coupling to a dead-end module; the tool logic is simple enough to copy.

**Hand-written JSON Schema via `agentkit.RawTool`** — the typed
`agentkit.NewTool[In]` constructors with `jsonschema`-tagged input structs are
what the package uses; a second hand-written schema per tool would be a copy
that can drift from the struct that actually decodes the arguments.

**An `All` variant without the confinement/share seams** — multiple
constructors for one toolset invite the wrong variant being wired by mistake;
the seams are required arguments, and a caller that wants "nothing allowed"
passes a `func(int) bool { return false }` (and an empty `ShareConfig` is a
composition-root error, not a silent degrade).

## Verification

- R-F5X1-XH6C — `tools.All(sandboxRoot, allowed, share)` returns exactly
  **13** tools named `Bash`, `Read`, `Write`, `Edit`, `Glob`, `Grep`, `Fetch`,
  `FileList`, `FileGet`, `FilePut`, `FileDelete`, `FileMove`, `FileMkdir`;
  each satisfies the `agentkit.Tool` interface.
- R-K1UK-T5K6 — each file/shell tool's `Call` rejects a path that escapes `sandboxRoot` (path traversal attempt returns an error without touching the filesystem outside the sandbox).
