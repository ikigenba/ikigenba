# Decision 4 — Provider factory

## Decision

A shared package `internal/provider` owns the single factory that constructs the correct `agentkit.Provider` from a pinned `Config` and a `getenv` accessor — consumed by both the runner (agent sessions, D7) and the one-shot inference paths (`/complete`, D29):

```go
package provider

func Build(cfg prompt.Config, getenv func(string) string) (agentkit.Provider, error)
```

The function switches on `cfg.Provider` and:

1. Reads the appropriate API key via `getenv` (same env var map as D3, five providers).
2. Calls the matching provider sub-package's constructor with its **typed credential** — the v0.6.0 constructors take a `Credential`, not a string:
   - `anthropic.New(anthropic.APIKey(key), opts...)`
   - `openai.New(openai.APIKey(key), opts...)`
   - `google.New(google.APIKey(key), opts...)`
   - `zai.New(zai.APIKey(key), opts...)`
   - `openrouter.New(openrouter.APIKey(key), opts...)`
3. Passes `WithBaseURL(cfg.BaseURL)` for **any** provider when `cfg.BaseURL` is non-empty.
4. Returns the constructed provider, or an error if the key is empty or the provider string is unrecognised (should not happen since validateConfig gated create/update, but fail loudly).

The package also owns the embedding-provider factory consumed by `/embed` (D30):

```go
func BuildEmbedder(providerName string, getenv func(string) string) (agentkit.EmbeddingProvider, error)
```

covering the providers with embedder constructors (`openai.NewEmbedder`, `google.NewEmbedder`) and failing loudly for any other name.

The injectable seams are unchanged in shape: `Runner.buildProvider` (D7) and the inference executor's factory field (D29) are both of type `func(prompt.Config, func(string) string) (agentkit.Provider, error)`, defaulting to `provider.Build`.

## Rejected

**Keeping `buildProvider` package-level in `runner`** — the original home; with `/complete` and `/embed` needing the identical five-way switch, a second copy or a cross-import of `runner` from `inference` would both be wrong shapes. One leaf package, two consumers.

**A generic credential abstraction over the five sub-packages** — each provider defines its own closed `Credential` type; a prompts-side unifying interface would add a layer over a five-case switch that is already trivial.

**Subscription credentials** — v0.5.0+ supports subscription auth for openai; prompts uses API keys only (the box's `.envrc` contract). Not wired.

## Verification

Pure structural decision — no behavior of its own. Its correctness is exercised by D7's runner tests and D29/D30's handler tests, which inject fake factories or exercise `provider.Build`'s error paths through their own ids, and by the compile against the v0.6.0 constructors.
