# Decision 6 ‚Äî Suite discovery

## Decision

`prompts/internal/suite/suite.go` returns the suite's peer tools as **deferred tool groups** ‚Äî `[]agentkit.DeferredToolGroup`, one group per reachable peer ‚Äî instead of a flat `[]agentkit.Tool`. The heavy per-tool descriptions and schemas still get fetched eagerly at spawn (they are needed the moment the in-run agent loads a tool), but they are no longer all surfaced to the model up front: agentkit's deferred-tools feature (research ¬ß2) shows the model a compact catalog and loads tools on demand. The best-effort, never-crash contract is unchanged: an unreachable or garbled peer is logged and skipped; `Discover` never returns an error and always returns a non-nil slice (possibly empty); self (`prompts`) is excluded so a run cannot spawn a run.

**New signature**:

```go
func Discover(ctx context.Context, manifestRoot, owner, promptID string) []agentkit.DeferredToolGroup
```

**Group shape**: for each non-self peer whose `tools/list` succeeds, one group:

- `Name` ‚Äî the service's manifest name (`svc.Name`), the catalog section heading.
- `Blurb` ‚Äî the peer's MCP `initialize` `instructions` string (research ¬ß3), fetched concurrently alongside `tools/list` with the same identity headers and per-call timeout. Best-effort within best-effort: an `initialize` failure or an absent `instructions` field yields `Blurb: ""` and the group is still returned ‚Äî a peer with tools but no blurb degrades to bare names in the catalog, never to exclusion.
- `Tools` ‚Äî the same `agentkit.RawTool` wrapping as before, unchanged: qualified name `ikigenba_<svc>_<verb>` (the existing `qualify` logic), discovered description and schema, `Call` dispatching `tools/call` to the peer with transport errors and MCP `isError` both surfaced as non-terminal Go errors (agentkit feeds them back as `isError` tool results; `terminalToolError` deliberately unused). The within-service duplicate guard is unchanged.

**`mcpclient` gains `Initialize`**: one new method on the existing client, same JSON-RPC plumbing as `ListTools`/`CallTool`:

```go
// Initialize calls JSON-RPC initialize and returns the server-level
// instructions string ("" when the server publishes none).
func (c *Client) Initialize(ctx context.Context) (string, error)
```

It sends the standard `initialize` params (protocol version, client info) and reads only `result.instructions`; capabilities negotiation is not consumed (the suite protocol is fixed).

**Eager pre-flight preserved**: discovery still happens once, at spawn, inside the runner's injectable `discover` closure ‚Äî not lazily at `Send`. `agentkit.MCPServer` remains unused for the same reason as before: its resolution fails the whole `Send` when a peer is down, violating best-effort. Deferral is agentkit's `DeferredTools` field consuming this function's output (D19), not agentkit's MCP path.

## Rejected

**Use `agentkit.MCPServer`** ‚Äî unchanged rejection: the published agentkit's MCP resolution propagates a down peer as a `Send` error; a down peer must not fail the run. (agentkit's own spec additionally records `MCPServers` deferral as out of scope.)

**Keep returning a flat `[]agentkit.Tool` and let the runner group it** ‚Äî the grouping facts (service name, blurb) are discovery facts, known here and only here; flattening them away and re-deriving them in the runner splits one concern across two packages.

**Skip a peer whose `initialize` fails** ‚Äî conflates "no blurb" with "no service"; the tools are already listed and callable, and a catalog section with bare names is strictly better than a missing service.

**Fetch blurbs from each service's `describe` tool** ‚Äî `describe` is a per-service domain verb that not all peers expose; `initialize.instructions` is the MCP-standard server-level description every peer already publishes (research ¬ß3).

## Verification

- R-K32H-6XAV ‚Äî when `Discover` is called with an inventory that includes a peer whose `tools/list` call returns an error, the returned groups contain no group for that peer and `Discover` does not return an error.
- R-K4AD-KP1K ‚Äî when `Discover` is called with a reachable peer that lists one tool, the returned groups contain a group for that peer whose `Tools` holds exactly one `agentkit.Tool` with the service-qualified name (`ikigenba_<svc>_<verb>`) and the discovered schema; calling that tool's `Call` method dispatches to the peer.
- R-9JNO-RZM2 ‚Äî `mcpclient.Initialize` against a server whose `initialize` result carries an `instructions` string returns that string; against a result with no `instructions` field it returns `""` without error.
- R-9KVL-5RCR ‚Äî a peer whose `tools/list` succeeds but whose `initialize` fails (or returns no instructions) still yields its group, with `Blurb == ""` ‚Äî the group is not dropped and `Discover` does not return an error.
- R-9M3H-JJ3G ‚Äî with two reachable non-self peers and a `prompts` self entry in the inventory, `Discover` returns exactly two groups, each `Name` equal to its peer's manifest service name and each `Blurb` equal to that peer's published instructions; no group exists for `prompts`.
