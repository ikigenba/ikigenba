# Decision 12 — A top-left Home link to the dashboard landing page

**Decision.** Every prompts human page carries one constant navigation
affordance: a **top-left Home link** that returns the viewer to the
**dashboard landing page**. It is a plain anchor to the apex root —
`<a class="home" href="/">Home</a>` — pinned to the page's top-left corner and
styled with the Carbon label tokens. Originally introduced on the v1 landing
card, it now lives in the shared UI chrome (D35) so it appears on every page
under `/ui/`.

**Why `href="/"`.** prompts' pages are served under `/srv/prompts/`; nginx is
the front door and the apex root `/` is the **dashboard** (the `DEFAULT` app
that owns the install/landing surface). A root-absolute link to `/` therefore
resolves — through nginx — to the dashboard landing page. No `/srv/...` prefix
and no dynamic value: the target is the bare apex root, the same for every
viewer.

**Styling.** A `.home` rule beside the other label-style selectors: absolutely
positioned to the top-left of the centered column (`main` carries
`position: relative`), mono label scale, muted ink brightening to full-strength
text on hover/focus — no new token and no new asset.

**Link text is `Home`, deliberately not `Dashboard`.** The page tests guard
against external/cross-service asset URLs by forbidding telltale substrings
(`http://`, `https://`, `dashboard`) in the rendered body. Labeling the
control `Home` keeps that guard green while still conveying the destination.

**Rejected.**
- **Linking to `/srv/dashboard/` or a computed dashboard URL** — the dashboard
  is the apex `DEFAULT`; its landing page is `/`. A `/srv/...` path would be
  wrong.
- **A back-button styled as a large CTA / header bar** — out of proportion; a
  quiet corner link matches the minimal Carbon aesthetic.
- **Labeling it `Dashboard` / `← Dashboard`** — trips the body's `dashboard`
  asset guard; `Home` conveys the same intent without weakening the test.

**Verification.**
- R-HOME-2T4X — every rendered UI page body contains a home link to the
  dashboard apex: an anchor whose `href` is exactly `/` (the body contains
  `href="/"`), distinct from the page's static asset references.
