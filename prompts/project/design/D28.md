# Decision 28 — The `calls` table: one durable row per inference unit

## Decision

prompts becomes the suite's accountable inference owner. Every unit of inference it executes — an agent-session run, a one-shot completion (D29), an embedding call (D30) — writes exactly one durable row into a new `calls` table, the single suite-wide record of who ran what inference, on which model, at what cost. A new `internal/calls` package owns the table.

**Grain.** One row per *inference unit*, not per provider round-trip: a completion call is one row, an embedding call is one row, and an agent-session run is one row carrying the run's aggregate usage and cost. (agentkit exposes only per-`Send` aggregate usage to its consumer — research §7 — so per-round-trip session rows are not implementable without an agentkit change; the run's `output.jsonl` retains per-turn detail while it lives.)

**Class vocabulary** (exactly these strings, stored and exposed verbatim — the product's contractual constants):

```
session   completion   embedding
```

**Origin grammar.** `origin` states who caused the call, in one of three forms:

- `user:<email>` — an owner-initiated run via MCP
- `trigger:<source>` — an event-fired run (`<source>` is the event's producer, e.g. `trigger:cron`)
- `service:<name>` — a sibling daemon calling `/complete` or `/embed` (e.g. `service:wiki`)

Validated by regex `^(user|trigger|service):[a-z0-9._@-]+$` (case-insensitive local parts allowed for emails: the check lowercases before matching).

**Name grammar.** `name` is the stable workload label used for grouping and spend reporting. For `completion`/`embedding` rows it must match `^[a-z0-9][a-z0-9-]*\.[a-z0-9][a-z0-9._-]*$` (`<service>.<stage>`, e.g. `wiki.compile`, `wiki.embed-page`). For `session` rows it is the prompt's name, falling back to the prompt id when the prompt is unnamed.

**Schema** — one new timestamped migration, created with `bin/create-migration prompts create-calls`:

```sql
CREATE TABLE calls (
  id            TEXT PRIMARY KEY,          -- ULID (internal/ids)
  class         TEXT NOT NULL CHECK (class IN ('session','completion','embedding')),
  origin        TEXT NOT NULL,
  name          TEXT NOT NULL,
  group_id      TEXT NOT NULL DEFAULT '',  -- run_id for sessions; caller-supplied job id otherwise
  attempt       INTEGER NOT NULL DEFAULT 1,
  owner_email   TEXT NOT NULL DEFAULT '',  -- '' when no user identity applies
  provider      TEXT NOT NULL,
  model         TEXT NOT NULL,             -- catalog name, not wire slug
  input_tokens  INTEGER NOT NULL DEFAULT 0,
  output_tokens INTEGER NOT NULL DEFAULT 0,
  total_tokens  INTEGER NOT NULL DEFAULT 0,
  usage_json    TEXT NOT NULL DEFAULT '',  -- full agentkit Usage, marshaled
  cost_usd      REAL NOT NULL DEFAULT 0,
  error         TEXT NOT NULL DEFAULT '',
  request_body  TEXT,                      -- NULL = never stored or pruned
  response_body TEXT,
  started_at    TEXT NOT NULL,             -- RFC3339 UTC
  ended_at      TEXT NOT NULL
);
CREATE INDEX calls_name_started ON calls (name, started_at);
CREATE INDEX calls_group ON calls (group_id);
CREATE INDEX calls_started ON calls (started_at);
```

`input_tokens` = Usage.InputUncached + CacheReadInput + CacheWriteInput; `output_tokens` = Usage.Output + ReasoningOutput; `total_tokens` = Usage.Total. The numeric columns exist so `usage` aggregations (D32) are plain SQL; `usage_json` preserves full fidelity (cache buckets, reasoning split).

**Store API** (`internal/calls`, opened over the shared appkit DB handle):

```go
type Store struct { db *sql.DB; now func() time.Time }

func (s *Store) Insert(ctx context.Context, row Row) error
func (s *Store) InsertTx(ctx context.Context, tx *sql.Tx, row Row) error // for D33's same-tx write
func (s *Store) PruneBodies(ctx context.Context, olderThan time.Time) (int64, error)
func (s *Store) List(ctx context.Context, f Filter) ([]Row, error)   // consumed by D32
func (s *Store) Get(ctx context.Context, id string) (Row, error)      // detail incl. bodies
func (s *Store) Aggregate(ctx context.Context, g GroupBy, f Filter) ([]Bucket, error)
```

`Filter` carries class, origin, name, group_id, errors-only, since/until, limit (default 50, max 500), offset. `GroupBy` ∈ {name, origin, model, day}; `Bucket` carries the key plus calls count, summed input/output/total tokens, summed cost, error count. `Insert` failure propagates to the caller — a call that cannot be recorded fails (accountability is a hard requirement; fail loudly).

**Body retention.** Metrics rows are never deleted. `PruneBodies` sets `request_body`/`response_body` to NULL on rows with `ended_at` older than the cutoff. Config: `PROMPTS_CALLS_BODY_RETENTION_DAYS`, default `30`, surfaced in `ManifestExtras` beside the outbox retention keys. The composition root runs the sweep at boot and on a 24h ticker.

## Rejected

**One row per provider round-trip for sessions** — agentkit's consumer-visible usage is aggregate per `Send` (one `usage` LogRecord per turn loop, `stream.Usage()` totals); per-round-trip rows would require forking the provider seam or parsing internals. The accounting questions (spend by workload/origin/model/time) are answered at unit grain.

**Separate tables per class** — three near-identical schemas, three sweep paths, and every "all inference" query becomes a UNION. One table with a `class` column is the point of the exercise.

**Deleting whole rows on retention** — the table is the permanent spend record; deleting rows silently falsifies historical totals. Only bodies age out.

**Naming it the inference "ledger"** — `ledger` is a sibling service in this suite; the term is reserved. The table, package, and tools say `calls`.

## Verification

- R-5J1W-8BCM — a Row inserted with every field populated reads back via `Get` with identical class, origin, name, group_id, attempt, owner_email, provider, model, token counts, usage_json, cost_usd, error, bodies, and timestamps.
- R-5K9S-M33B — after `PruneBodies` with a cutoff newer than a row's `ended_at`, that row's `Get` returns NULL bodies while every metric field (tokens, cost, origin, name, timestamps) is unchanged.
- R-5LHO-ZUU0 — `PruneBodies` leaves the bodies of a row with `ended_at` newer than the cutoff intact.
- R-5MPL-DMKP — `List` honors each filter dimension (class, origin, name, group_id, errors-only, since/until) and pagination: a query matching 3 of 5 seeded rows returns exactly those 3, and limit/offset walk them deterministically.
- R-5NXH-REBE — `Aggregate` grouped by name over a seeded mix of rows returns per-name buckets whose call counts, token sums, and cost sums equal the seeded totals, and a since/until window excludes rows outside it.
