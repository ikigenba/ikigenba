# Decision 30 — `POST /embed`: the synchronous embedding endpoint

## Decision

The third workload class: embeddings, served from the same `internal/inference` package and the same loopback plumbing tier as `/complete` (D29). Sibling services (wiki's page/query vectors) get their embeddings through prompts, so the embedding provider key also leaves every other service.

**Request** (`POST /embed`, JSON):

```json
{
  "origin":     "service:wiki",           // required, D28 origin grammar
  "name":       "wiki.embed-page",        // required, D28 name grammar
  "group_id":   "job-01H…",               // optional
  "attempt":    1,                         // optional, default 1
  "model":      "text-embedding-3-small", // required, catalog embedding entry
  "provider":   "openai",                 // optional, catalog default when omitted
  "dimensions": 512,                       // optional; 0/omitted = model native
  "role":       "document",               // required: "document" | "query"
  "inputs":     ["…", "…"]                // required, non-empty, no blank strings
}
```

**Validation**: origin/name grammar (D28); `catalog.Lookup(model)` must return an entry with non-nil `Embedding`; the resolved provider must be one with an embedder constructor (`openai`, `google` — research §7); `dimensions`, when non-zero, must lie within the entry's `[MinDimension, MaxDimension]`; `role` ∈ {`document`, `query`} (mapped to `agentkit.InputDocument` / `agentkit.InputQuery`); `inputs` non-empty with no empty strings; body ≤ 10 MiB. Failure → `400` with `{"error": "…"}`, no row.

**Execution**: build the embedding provider from the same env-keyed credentials (`openai.NewEmbedder(openai.APIKey(key))` / `google.NewEmbedder(...)`), acquire the per-provider call gate (D31), then one batch call:

```go
emb := &agentkit.Embedder{
    Provider:   embProvider,
    Model:      wireModel,
    Pricing:    &entry.Embedding.Pricing,
    Dimensions: req.Dimensions,
}
res, err := emb.Embed(ctx, req.Inputs, role)
```

**Recording**: one `calls` row, class `embedding`. `request_body` = the marshaled inputs; `response_body` = NULL always (vectors are bulk numeric payload with no audit value — the caller owns the vectors; the row owns the spend). Usage maps to `input_tokens`/`total_tokens`; `output_tokens` stays 0. Insert failure → `500`, as in D29.

**Response**: `200` with

```json
{ "call_id": "01J…", "vectors": [[0.1, …], …], "usage": { … }, "cost_usd": 0.0002 }
```

vectors in input order, one per input. Provider failure → `502`, row records the error.

**Test seam**: `agentkit.EmbeddingProvider` is an interface; tests inject a fake returning fixed vectors and usage through the real handler.

## Rejected

**Folding embeddings into `/complete`** — different request/response shape (batch inputs and vectors vs messages and text), different catalog surface (`EmbeddingInfo` vs `Pricing`), different provider constructor. Two small endpoints beat one polymorphic envelope.

**Storing vectors in `response_body`** — hundreds of floats per input, useless for inspection or audit, and the retention sweep would exist mostly to delete them. The spend metrics are the record.

**Per-input rows** — one batch call is one billed unit; splitting it fabricates rows that never happened.

## Verification

- R-604H-L3QC — a valid request through the real handler with an injected fake embedding provider returns `200` with one vector per input in input order, usage and cost from the fake's reported usage priced by the catalog entry, and exactly one `calls` row of class `embedding` with non-NULL `request_body` and NULL `response_body`.
- R-61CD-YVH1 — a request naming a catalog model with no embedding entry (e.g. a chat model) returns `400` naming the problem; no row.
- R-62KA-CN7Q — a `dimensions` value outside the model's catalog `[min, max]` range returns `400` naming the accepted range; no row.
- R-63S6-QEYF — a request resolving to a provider with no embedder (e.g. `anthropic`) returns `400`; no row.
- R-6503-46P4 — an empty `inputs` array, or one containing an empty string, returns `400`; no row.
- R-667Z-HYFT — when the injected embedding provider fails, the endpoint returns `502` and the `calls` row records the error.
