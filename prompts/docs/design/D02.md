# Decision 2 — Config struct

## Decision

`prompt.Config` in `prompts/internal/prompt/model.go` is expanded to carry the full agentkit generation and retry surface. `provider` and `model` remain required (validated at create/update). The 11 optional keys map directly to fields on `agentkit.Conversation`:

```go
type Config struct {
    // Required
    Provider string `json:"provider"`
    Model    string `json:"model"`

    // Generation controls (agentkit.GenSettings)
    Temperature    *float64 `json:"temperature,omitempty"`
    TopP           *float64 `json:"top_p,omitempty"`
    MaxTokens      int      `json:"max_tokens,omitempty"`
    Effort         string   `json:"effort,omitempty"`           // maps to Gen.Reasoning via agentkit.Level
    ThinkingBudget *int     `json:"thinking_budget,omitempty"` // maps to Gen.Reasoning via agentkit.Budget
    ThinkingLevel  string   `json:"thinking_level,omitempty"`  // maps to Gen.Reasoning via agentkit.Level
    Thinking       *bool    `json:"thinking,omitempty"`        // false → agentkit.DisableReasoning()

    // Retry policy (agentkit.RetryPolicy)
    MaxAttempts      int    `json:"max_attempts,omitempty"`
    BaseDelay        string `json:"base_delay,omitempty"`        // Go duration string, e.g. "500ms"
    MaxDelay         string `json:"max_delay,omitempty"`
    MaxElapsed       string `json:"max_elapsed,omitempty"`
    IgnoreRetryAfter bool   `json:"ignore_retry_after,omitempty"`

    // Loop and provider tuning
    ToolLoopLimit int    `json:"tool_loop_limit,omitempty"` // maps to Conversation.MaxToolIterations
    BaseURL       string `json:"base_url,omitempty"`        // Zai only: alternate API root
}
```

**Storage**: Config is stored verbatim as `config_json` (TEXT) in the prompts table, marshalled as a JSON object. No migration of the JSON column structure is required — new keys are simply absent in old rows and default to zero values, which the agentkit treats as "use provider default". Full-replacement update semantics: every `update` call replaces the entire Config; keys not supplied revert to zero/omitempty (absent from JSON, treated as default by the agentkit).

**Model storage**: the `model` field stores the canonical bare API ID as returned by the provider's registry (e.g. `"claude-sonnet-4-6"`, `"gpt-5.5"`, `"gemini-2.5-flash"`, `"glm-5.2"`). No short aliases are accepted or stored; the validation layer (D3) rejects unrecognised strings immediately.

**Reasoning precedence**: `effort`, `thinking_budget`, `thinking_level`, and `thinking` are mutually exclusive reasoning controls — only the first non-zero one is applied at run time (in that order), matching agentkit's single `Gen.Reasoning` field.

## Rejected

**Keep the existing narrow Config** (`provider`, `model`, `effort`, `max_tokens`, `temperature`) — rejected: this is exactly what we are replacing; the product requires the full 11-key surface.

**Embed duration fields as `time.Duration`** — rejected: `time.Duration` is an int64 and would marshal as a raw number, not a human-readable string. Go duration strings ("500ms", "2m") are idiomatic and readable in stored JSON.

**Support short aliases** — rejected (product decision): full bare IDs only; aliases were removed to keep stored config unambiguous.

## Verification

- R-JTBA-4RDB — a Config with all 11 optional keys set marshals to JSON and unmarshals back with all values preserved exactly.
- R-JUJ6-IJ40 — updating a prompt with a Config that omits a previously-set optional key results in a stored config where that key is absent (full-replacement, not merge).
