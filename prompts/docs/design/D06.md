# Decision 6 — Suite discovery

## Decision

`prompts/internal/suite/suite.go` is adapted to return `[]agentkit.Tool` instead of `agent.ToolSource`. The best-effort, never-crash contract is unchanged: an unreachable or garbled peer is logged and skipped; `Discover` never returns an error and always returns a non-nil slice (possibly empty).

**New signature**:

```go
func Discover(ctx context.Context, manifestRoot, owner, promptID string) []agentkit.Tool
```

The eager pre-flight pattern is preserved: tool listing happens at spawn time (inside `discover` closure call in `runner.execute`), not lazily at `Send` time. This is why we do not use `agentkit.MCPServer` — the published agentkit's `resolveMCPTools` fails the whole `Send` call if any listed server is unreachable, breaking the best-effort contract.

**Tool wrapping**: each discovered tool becomes an `agentkit.RawTool`:

```go
agentkit.RawTool(
    qualifiedName,  // "ikigenba_<svc>_<verb>" — same qualify() logic as before
    discoveredDescription,
    discoveredSchema,
    func(ctx context.Context, input json.RawMessage) (string, error) {
        text, isError, err := client.CallTool(ctx, bareVerb, input)
        if err != nil {
            return "", errors.New("suite: tool " + qualifiedName + " failed: " + err.Error())
        }
        if isError {
            return "", errors.New(text)
        }
        return text, nil
    },
)
```

Both transport errors and `isError` MCP responses become non-terminal Go errors (the agentkit's `runTool` wraps them as `isError` tool result blocks, keeping the conversation alive). Only a `terminalToolError` would abort the run — we deliberately do not use it here.

**`source` struct is removed**: the old `source` struct with `Descriptors`, `Owns`, and `Dispatch` methods is deleted. The three maps (`tools`, `owner`, `verb`) collapse into a single `[]agentkit.Tool` slice built during discovery.

**Runner field update**: `Runner.discover` changes type from `func(...) agent.ToolSource` to `func(...) []agentkit.Tool`. In `execute`, the result is appended directly to `Conversation.Tools`.

## Rejected

**Use `agentkit.MCPServer`** — the published agentkit's `resolveMCPTools` retries but ultimately returns an error that propagates to `Send()` if a peer is unreachable. This violates the best-effort contract; a down peer must not fail the run.

**Keep the `agent.ToolSource` interface with an adapter** — an adapter wrapping `[]agentkit.Tool` in a fake `ToolSource` would only be needed if we kept the old local `agent.Run` signature. Since D7 replaces that entirely, the adapter has no purpose.

## Verification

- R-K32H-6XAV — when `Discover` is called with an inventory that includes a peer whose `tools/list` call returns an error, the returned tool slice contains no tools from that peer and `Discover` does not return an error.
- R-K4AD-KP1K — when `Discover` is called with a reachable peer that lists one tool, the returned slice contains exactly one `agentkit.Tool` with the service-qualified name and the discovered schema; calling that tool's `Call` method dispatches to the peer.
