# Decision 4 тАФ Provider factory

## Decision

`runner.go` gains a package-level function `buildProvider` that constructs the correct `agentkit.Provider` from a pinned `Config` and a `getenv` accessor:

```go
func buildProvider(cfg prompt.Config, getenv func(string) string) (agentkit.Provider, error)
```

The function switches on `cfg.Provider` and:

1. Reads the appropriate API key via `getenv` (same env var map as D3).
2. Calls the matching provider sub-package's `New(apiKey, opts...)`.
3. For `"zai"`, passes `zai.WithBaseURL(cfg.BaseURL)` when `cfg.BaseURL` is non-empty.
4. Returns the constructed provider, or an error if the key is empty or the provider string is unrecognised (should not happen since validateConfig gated create/update, but fail loudly).

The old `clientFactory` field on `Runner` and `defaultClientFactory` function are removed. The `newClient` injectable seam is replaced by an injectable `buildProvider` field of type `func(prompt.Config, func(string) string) (agentkit.Provider, error)` for testing.

```go
type Runner struct {
    store         *prompt.Store
    sandbox       *sandbox.Manager
    ttl           time.Duration
    buildProvider func(prompt.Config, func(string) string) (agentkit.Provider, error)
    discover      func(ctx context.Context, owner, promptID string) []agentkit.Tool
    // ... cancels, userCancelled unchanged
}
```

The `discover` field return type changes from `agent.ToolSource` to `[]agentkit.Tool` (D6).

## Rejected

**Pass only the API key to a factory** тАФ the old `clientFactory(apiKey, model string) (provider.Client, error)` passed model at factory time. The published agentkit takes model on the `Conversation`, not at provider construction. The new factory takes the full `Config` because `BaseURL` (for Zai) also lives there.

**Construct the provider inside `execute`** тАФ inline construction with a switch statement in `execute` would work but makes the factory un-injectable for tests. A named function with an injectable seam preserves the existing test pattern.

## Verification

Pure structural decision тАФ no behavior of its own. Its correctness is exercised by D7's runner tests, which inject a fake `buildProvider` that returns a stub `agentkit.Provider`.
