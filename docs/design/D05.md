# Decision 5 — Built-in tools

## Decision

The six built-in tools (bash, read, write, edit, glob, grep) currently live in the local `agentkit/tools/` sub-packages and are advertised via `tools.All() []Descriptor`. That package is going away (D1). The tool implementations are moved into a new package `prompts/internal/tools/` which owns exactly the same six tools but implements the published `agentkit.Tool` interface.

**Package shape**:

```go
// prompts/internal/tools/tools.go
package tools

import "github.com/ikigenba/agentkit"

// All returns the six built-in tools confined to sandboxRoot.
// SandboxRoot is threaded in at call time (not package-level global)
// so tests can supply a temp directory and the runner supplies the
// per-run sandbox path.
func All(sandboxRoot string) []agentkit.Tool
```

Each tool is constructed with `agentkit.RawTool(name, description, schema, fn)` where:
- `name` — unchanged from the local agentkit (bash, read, write, edit, glob, grep)
- `description` — a short prose description added for the first time (the local `Descriptor` had no description field; the published `Tool` interface requires one)
- `schema` — the existing JSON Schema from each tool's local package, copied verbatim
- `fn` — a closure over `sandboxRoot` that calls the existing tool dispatch logic (also copied from the local packages, adapted to return `(string, error)` matching `agentkit.Tool.Call`)

The sandbox confinement logic (path validation against sandboxRoot) is preserved unchanged inside each tool's `fn`.

**No new package-level state**: `tools.All(sandboxRoot)` is called once per run (in `execute`), producing a fresh slice of tool values. The tool implementations remain stateless.

**Tool descriptions** (one per tool, added here for the first time):

| tool  | description |
|-------|-------------|
| bash  | Run a shell command in the sandbox. No network access. |
| read  | Read a file from the sandbox. |
| write | Write or overwrite a file in the sandbox. |
| edit  | Apply a targeted string replacement to a file in the sandbox. |
| glob  | List files matching a glob pattern in the sandbox. |
| grep  | Search for a pattern in files in the sandbox. |

## Rejected

**Keep the local agentkit module for its tools only** — renaming the local module to something like `promptstools` and keeping a replace directive. Rejected: unnecessary coupling to a dead-end module; the tool logic is simple enough to copy.

**Use `agentkit.NewTool[In]` with typed input structs** — the tool schemas are already hand-written JSON Schema objects. `agentkit.RawTool` is the right constructor; `NewTool` would require defining typed input structs and then throwing away the hand-written schemas.

## Verification

- R-K0MO-FDTH — `tools.All(sandboxRoot)` returns exactly 6 tools with names bash, read, write, edit, glob, grep; each satisfies the `agentkit.Tool` interface.
- R-K1UK-T5K6 — each tool's `Call` rejects a path that escapes `sandboxRoot` (path traversal attempt returns an error without touching the filesystem outside the sandbox).
