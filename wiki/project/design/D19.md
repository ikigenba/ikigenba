# Decision 19 — Per-call-site configuration in production (retire the single global model)

**Decision.** Replace the single global `Config.ModelID` with **full per-site configuration**: each production LLM call site owns a `DefaultCallSite()` carrying its designed values, and every model-tuning field — model, reasoning, temperature, output budget — is overridable per site from its own manifest knobs. The closed set of **production service** call sites is **extract, compile, ask-subject, ask-synthesis**.

## Each site owns a `DefaultCallSite()`

Every stage exposes a no-secret constructor returning its designed `llm.CallSite` (D5), so the composition root never hand-builds a bare one (the drift D6 warns against):

```go
// each stage package (values expressed in the D5 Config vocabulary; every
// default carries Provider "openai", Model "gpt-5.6-luna", and the stage's
// embedded instruction preamble as System — see D6/D7/D9/D36):
func extract.DefaultCallSite() llm.CallSite      // Stage "extract",       Effort "low", MaxTokens 16384, MaxParseRetries 2
func compile.DefaultCallSite() llm.CallSite      // Stage "compile",       Effort "low", MaxTokens 16384
func ask.DefaultSubjectCallSite() llm.CallSite   // Stage "ask-subject",   Effort "low", MaxTokens 16384
func ask.DefaultSynthesisCallSite() llm.CallSite // Stage "ask-synthesis", Effort "low", MaxTokens 16384
```

No default sets `Temperature` or `Thinking`: gpt-5.6 models are reasoning models addressed with `effort`, and temperature pins are not portable to them (research §13). The knobs below can still set those fields for an operator running a model that accepts them; the *defaults* never do. The single in-code default model constant is `ModelID = "gpt-5.6-luna"` with provider `openai`.

The `Stage` label is the site's stable identity — it forms the accounting `name` `wiki.<stage>` on every prompts call (D62) and is the per-stage inspection key in prompts' `calls`/`usage` tools. The two `ask` sites keep their distinct labels **`ask-subject`** and **`ask-synthesis`** so spend and configs stay separable.

## Per-site config resolved from per-site knobs

`Config.ModelID` is removed. `Config` carries a resolved `CallSites` value; each field is that site's `DefaultCallSite()` with its env overrides applied:

```go
type CallSites struct {
    Extract      llm.CallSite
    Compile      llm.CallSite
    AskSubject   llm.CallSite
    AskSynthesis llm.CallSite
}
type Config struct {
    CallSites         CallSites
    EmbedSite         llm.EmbedSite // D34
    WorkerConcurrency int
    SearchDefault     int
    SearchCap         int
    LLM               *llm.Client   // the D5 prompts client (base URL from registry, D1)
}
```

A single resolver applies a site's four model-tuning knobs onto its default:

```go
// resolveCallSite layers <PREFIX>_MODEL / _REASONING / _TEMPERATURE / _MAX_TOKENS onto base.
// An unset knob leaves the base default; a present-but-malformed knob is a startup error (fail loudly).
func resolveCallSite(getenv func(string) string, prefix string, base llm.CallSite) (llm.CallSite, error)
```

**Manifest knobs** (`ManifestExtras`, replacing `MODEL_ID`) — four per site, defaulted to each `DefaultCallSite`'s value so an all-unset environment reproduces today's behavior exactly:

```
EXTRACT_MODEL        EXTRACT_REASONING        EXTRACT_TEMPERATURE        EXTRACT_MAX_TOKENS
COMPILE_MODEL        COMPILE_REASONING        COMPILE_TEMPERATURE        COMPILE_MAX_TOKENS
ASK_SUBJECT_MODEL    ASK_SUBJECT_REASONING    ASK_SUBJECT_TEMPERATURE    ASK_SUBJECT_MAX_TOKENS
ASK_SYNTHESIS_MODEL  ASK_SYNTHESIS_REASONING  ASK_SYNTHESIS_TEMPERATURE  ASK_SYNTHESIS_MAX_TOKENS
```

**Value parsing (fail-loud on garbage), mapping onto the D5 `Config` vocabulary verbatim — wiki invents no vocabulary:**
- `_MODEL` — a model name string; empty/unset keeps the default. (Validation against the catalog is prompts' concern — a `400` at call time, D5.)
- `_REASONING` — `off` → `Thinking:false`, or `low`/`medium`/`high` → `Effort:<value>`. Any other non-empty value is a **startup error** naming the valid set — never a silent fallback.
- `_TEMPERATURE` — a float parsed to `*float64`; literal `default` (or unset) means `nil` (provider default). A non-numeric, non-`default` value is a startup error.
- `_MAX_TOKENS` — a positive int; unset keeps the default; `0`, negative, or non-numeric is a startup error (a production JSON site must never run without an explicit ceiling — D18).

The composition root (`cmd/wiki/main.go`) builds each stage from its resolved site: `extract.New(client, cfg.CallSites.Extract)`, `compile.New(client, cfg.CallSites.Compile, …)`, `ask.New(subjectStore, pageStore, client, cfg.CallSites.AskSubject, cfg.CallSites.AskSynthesis)`. No bare `llm.CallSite{…}` remains.

**Rejected.**
- **Model-only production knob (reasoning/temp/budget frozen in code).** Per-site config is the entire point; freezing the other levers freezes an unproven guess. Fail-loud parsing keeps the freedom from corrupting a determinism-critical site. Rejected.
- **A new per-site override struct (`CallSiteConfig{Model,Reasoning,…}`) over `CallSite`.** Redundant — the resolver mutates the default in place; a parallel type is a second shape to keep in sync. Rejected.
- **Keeping `MODEL_ID` and deriving per-site models from it.** Re-creates the global; extract could never differ from compile. Rejected — that is the bug.
- **Exposing prompts' whole config vocabulary as knobs** (`thinking_budget`, retry keys, …). The four levers above are the ones the stages designed for; a site needing more edits its `DefaultCallSite`. Rejected as knob sprawl.

**Verification.**
- R-GGIG-AN7W — With `EXTRACT_MODEL` and `COMPILE_MODEL` set to **different** values, `NewConfig` yields `CallSites.Extract.Model` and `CallSites.Compile.Model` each equal to its own knob (not equal to each other) — model resolved per site, not from one shared global.
- R-GHQC-OEYL — A non-model knob set for one site reaches only that site: `ASK_SYNTHESIS_REASONING=low` (or a `_TEMPERATURE`/`_MAX_TOKENS` override) produces the configured value on `CallSites.AskSynthesis` while the other three sites retain defaults.
- R-A25B-A1V6 — With **all** per-site knobs unset, every resolved site equals its stage `DefaultCallSite()`: each of the four sites carries `Provider "openai"`, `Model "gpt-5.6-luna"`, `Effort "low"`, `MaxTokens >= 16384`, a nil `Temperature`, and a nil `Thinking` — an unconfigured environment serves every chat stage on gpt-5.6-luna with the effort knob, never a temperature or thinking pin.
- R-GK65-FYFZ — Parsing fails loudly at startup on a malformed knob: an unrecognized `_REASONING` (not `off`/`low`/`medium`/`high`) returns a startup error naming the valid set; a non-numeric `_TEMPERATURE` (other than `default`) and a `_MAX_TOKENS` that is non-positive or non-numeric each return a startup error — none silently fall back to a default.
- R-GLE1-TQ6O — `ask.DefaultSubjectCallSite()` and `ask.DefaultSynthesisCallSite()` carry the distinct stage labels `ask-subject` and `ask-synthesis` and `MaxTokens>=16384`, and the composition root constructs `ask` from them (no bare `llm.CallSite{Stage:"ask",…}` remains) — the two ask sites are independently configured and separately accounted.
