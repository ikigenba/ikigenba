# Decision 63 — The retirement: `llm_calls` and the recorder stack leave wiki; agentkit is confined to the eval workbench

**Decision.** The prompts-service conversion deleted everything the prompts-backed seam obsoletes — the bar is **no orphaned fragments**: no dead types, no unreferenced files, no vocabulary that outlived its feature. The eval workbench (D64–D67) later returned as a deliberate, bounded exception, so this decision now states the retirement's **current** scope: the call-logging stack stays gone; agentkit exists in the module but is confined to the workbench packages.

- **The `llm_calls` footprint.** The forward migration dropping the `llm_calls` table and its indexes stands (committed migrations stay frozen). The `LLMCallStore`, the `Recorder` seam, `CallRecord`, `WithJobID`/`JobID` context threading, and `recording_embedder` remain deleted — none of that vocabulary returns with the workbench, which keeps no call log. Inspection of production inference stays with prompts' `calls`/`usage` MCP tools (name `wiki.<stage>`, `group_id` per D62).
- **The MCP surface.** The `llm_calls` tool stays removed from wiki's tool list; the MCP `Instructions` string and the `guide` content do not present LLM-call inspection as a wiki capability (they may point at prompts' `calls` tool as the successor).
- **The old judge harness stays dead.** The retired `internal/eval` (dataset runner, **LLM judge**, JSONL recorder) is not what D64–D67 rebuild: the new workbench scores deterministically, keeps no call log, and never judges with a model. `extract.DefaultPromptInstructions` stays exported and addressable — it is the workbench's contract that eval runs the same prompt text production runs (D64 makes it the same file).
- **agentkit confinement.** With chat on `/complete` (D5) and embeddings on `/embed` (D34), the **service** has no use for agentkit. The module's agentkit imports are confined to the workbench: only `cmd/eval-extract` and `internal/eval` may import it. D1 owns the production-binary proof (`go list -deps ./cmd/wiki`); this decision owns the source-tree proof.
- **Config residue.** The service reads no `ANTHROPIC_API_KEY`/`OPENAI_API_KEY` (D1), carries no `agentkit.Provider`/`EmbeddingProvider` fields on its `Config`, and has no reasoning-value translation helpers; D19's per-site knobs target the D5 `Config` vocabulary.

**Rejected.**
- **Keeping `llm_calls` dormant** (table present, writes stopped) — a schema lying about a capability; the no-orphans bar exists to prevent exactly this.
- **A wiki `llm_calls` proxy tool over prompts' data** — a cross-service read path duplicating what prompts' `calls` tool already serves.
- **Extending the retirement to ban the eval tree and agentkit from the module forever** (the previous form of this decision) — that ban's real target was orphaned *production* machinery, and it hit dev tooling as collateral. The workbench is live, referenced, and bounded; the retirement keeps its point (no dead code, no shadow call log, no second inference path in the service) without it.
