# Decision 63 — The retirement: `llm_calls`, the recorder stack, the eval harness, and agentkit all leave wiki

**Decision.** The conversion deletes everything the prompts-backed seam obsoletes, completely — the explicit bar is **no orphaned fragments**: no dead types, no unreferenced files, no vocabulary that outlived its feature.

- **The `llm_calls` footprint.** A new forward migration (`bin/create-migration wiki drop_llm_calls`) drops the `llm_calls` table and its indexes (committed prior migrations stay frozen; the data is disposable per the suite migration window). The `LLMCallStore`, the `Recorder` seam, `CallRecord`, `WithJobID`/`JobID` context threading, and `recording_embedder` are deleted. Inspection moves to prompts' `calls`/`usage` MCP tools (name `wiki.<stage>`, `group_id` per D62).
- **The MCP surface.** The `llm_calls` tool is removed from wiki's tool list; the MCP `Instructions` string and the `guide` content no longer mention LLM-call inspection (they may point at prompts' `calls` tool as the successor). The web surface never exposed it; nothing changes there.
- **The eval harness.** `cmd/eval-extract` and `internal/eval` (dataset runner, judge, scorecard, JSONL recorder) are deleted, along with `testdata/eval/` gold cases. Future judging is a dev-time process outside wiki's code whose only contract is **prompt identity**: it must use the same prompt text wiki's extract stage generates, so `extract.DefaultPromptInstructions` (the exported production preamble) stays exported and addressable; the `WithPromptInstructions` option retires with its only consumer.
- **agentkit.** With chat on `/complete` (D5), embeddings on `/embed` (D34), and the harness gone, no agentkit import remains anywhere; `go.mod` drops the require (D1 owns the module-graph proof; this decision owns the source-tree proof).
- **Config residue.** `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` reads, the `agentkit.Provider`/`EmbeddingProvider` fields on `Config`, and the reasoning-value translation helpers disappear; D19's per-site knobs re-target the D5 `Config` vocabulary.

**Rejected.**
- **Keeping `llm_calls` dormant** (table present, writes stopped) — a schema lying about a capability; the no-orphans bar exists to prevent exactly this.
- **A wiki `llm_calls` proxy tool over prompts' data** — a cross-service read path duplicating what prompts' `calls` tool already serves.
- **Keeping `internal/eval` compiled-but-unused for the future harness** — the future harness is explicitly not wiki code; unused packages are the drift this decision deletes.
