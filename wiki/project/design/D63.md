# Decision 63 — The retirement: the recorder stack and every in-repo tuning executable stay gone

**Decision.** The prompts-service conversion deleted everything the prompts-backed seam obsoletes, and the tuning-machinery removal (D70) finished the job — the bar is **no orphaned fragments**: no dead types, no unreferenced files, no vocabulary that outlived its feature. This decision states the retirement's **current** scope: the call-logging stack stays gone, and no tuning executable, eval package, or provider-SDK dependency exists anywhere in the module.

- **The `llm_calls` footprint.** The forward migration dropping the `llm_calls` table and its indexes stands (committed migrations stay frozen). The `LLMCallStore`, the `Recorder` seam, `CallRecord`, `WithJobID`/`JobID` context threading, and `recording_embedder` remain deleted. Inspection of production inference stays with prompts' `calls`/`usage` MCP tools (name `wiki.<stage>`, `group_id` per D62).
- **The MCP surface.** The `llm_calls` tool stays removed from wiki's tool list; the MCP `Instructions` string and the `guide` content do not present LLM-call inspection as a wiki capability (they may point at prompts' `calls` tool as the successor).
- **The tuning machinery is gone too.** `cmd/autotune`, `cmd/eval-extract`, `cmd/eval-analysis`, `internal/eval`, and the `eval/` data tree are deleted (D70); tuning runs through the external `autotune` tool over the committed `autotune/<step>/` folders (D71). `extract.DefaultPromptInstructions` stays exported — it is now the package-local embedded production prompt (D6), the artifact an adopted tuning winner is manually copied into.
- **agentkit is out of the module.** With chat on `/complete` (D5), embeddings on `/embed` (D34), and the workbench gone, nothing imports agentkit; `go.mod` carries no `github.com/ikigenba/agentkit` require. D1 owns the binary-closure proof; this decision owns the module-wide proof.
- **Config residue.** The service reads no provider key (D1). `internal/llm`'s deprecated evaluation-residue compatibility fields (`CallSite.Model`, `CallSite.Temperature`, `CallSite.Reasoning`, `CallSite.MaxTokens`, `DisableReasoning`) are deleted — the D5 `CallSite{Stage, System, Config, MaxParseRetries}` shape is the whole type again.

**Rejected.**
- **Keeping `llm_calls` dormant** (table present, writes stopped) — a schema lying about a capability; the no-orphans bar exists to prevent exactly this.
- **A wiki `llm_calls` proxy tool over prompts' data** — a cross-service read path duplicating what prompts' `calls` tool already serves.
- **Keeping the in-repo workbench beside the external tool** (the previous form of this decision carved out exactly that exception) — two tuning stacks that drift; the external tool's folder contract carries everything the workbench did, and the repo keeps only data plus scorers (D71/D72).

**Verification.**
- R-KFX6-MNEW — The identifiers `Recorder`, `CallRecord`, `LLMCallStore`, `NewRecordingEmbedder`, and `WithJobID` appear nowhere in non-test source, and `extract.DefaultPromptInstructions` remains exported — the recorder stack stays retired while the production prompt stays addressable.
- R-A4L4-1LCK — `grep -rn "ikigenba/agentkit" --include='*.go' .` from `wiki/` (excluding `project/`) matches nothing, and `wiki/go.mod` contains no `agentkit` require — no code in the module imports or depends on agentkit.
