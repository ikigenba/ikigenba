# Decision 34 — The embedding call site: prompts `/embed`, one model for both sides

**Decision.** Embedding is its own configured call site (model + vector size), executed through the prompts service's loopback `POST /embed` (D1; API footprint in research §12) rather than a wiki-held provider. Accounting is prompts' `calls` table; wiki records nothing (D63).

**One model, two uses.** A single embedding model serves both sides — embedding a page and embedding a question — because the two must live in the same vector space to be comparable. So there is **one** embed config, used in two places, distinguished on prompts' record by `name`:

- **`wiki.embed-page`** — embedding a page during ingest or merge (D35); role `document`; `group_id` = the ingest job id, origin per the job (D62).
- **`wiki.embed-query`** — embedding a question at ask time (D32); role `query`; `group_id` = the ask correlation id, origin `user:<email>` (D62).

**Config (knobs `EMBED_MODEL`, `EMBED_DIMS`) — unchanged.** Resolved alongside the other call sites (D19), fail-loud parsing:

```go
type EmbedSite struct {
    Model string // default "text-embedding-3-small"
    Dims  int    // default 512
}
```

- `EMBED_MODEL` — model name string (a prompts-catalog embedding entry); unset keeps `text-embedding-3-small`.
- `EMBED_DIMS` — a positive integer; unset keeps `512`; `0`, negative, or non-numeric is a **startup error** (fail loud, never a silent fallback).

The default stays **`text-embedding-3-small` at 512 dimensions**; storing `model`/`dims` with every vector (D30) makes a later change a re-embed, not a schema break. Whether the model/dims combination is valid is **prompts' catalog validation** — a `400` from `/embed` at call time, not a wiki-side re-check (no second vocabulary, D5's rule).

**The client (`internal/llm`).** The embed client lives beside the chat client — same base URL, same context-deadline-only doctrine, same fail-loud errors:

```go
// Embed posts one batch to prompts /embed and returns vectors in input order.
// Request: {origin, name, group_id, model, dimensions, role, inputs}.
// role is "document" (page side) or "query" (ask side).
func (c *Client) Embed(ctx context.Context, site EmbedSite, attr Attribution, role string, inputs []string) ([][]float32, error)
```

The composition root wires the page side and the query side to this one method with their respective role and `name`; the old `recordingEmbedder` wrapper and the raw agentkit embedder are both gone (D63).

**Rejected.**
- **Separate model knobs for the page side and the query side** — they must share one vector space; two models would silently break search. One config, enforced by construction.
- **Wiki-side validation of model/dims against a local catalog** — duplicates prompts' catalog and drifts; the `400` is the validation.
- **Recording embed calls wiki-side** — retired with the whole `llm_calls` footprint (D63); the spend record is prompts' row (`request_body` = the inputs, vectors never stored).

**Verification.**
- R-Z932-H2RA — Config resolves the embed knobs: with `EMBED_MODEL`/`EMBED_DIMS` unset the site is `text-embedding-3-small` @ 512; a set `EMBED_DIMS` overrides it; a non-positive or non-numeric `EMBED_DIMS` returns a **startup error** (no silent fallback to a default).
- R-1385-QVMS — An `Embed` call posts `/embed` JSON carrying the site's model and dimensions, the given role, origin, `group_id`, name, and the inputs, and returns the response's vectors in input order (captured/served via the httptest prompts).
- R-14G2-4NDH — The composition-root wiring sends the **page** side as `name: "wiki.embed-page"` with role `document` and the **query** side as `name: "wiki.embed-query"` with role `query`, through the same single client method (a raw or mislabeled wiring fails this).
- R-15NY-IF46 — **Live check (operator-run, real running prompts service):** with the suite up, an `Embed` call with the configured model and `Dims=512` against prompts' real loopback `/embed` returns one vector of length 512 per input — proving the real endpoint accepts the configured model+dimensions and the wiring produces usable vectors, a property no httptest fake can establish.
