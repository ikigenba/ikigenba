# Decision 66 — The eval runner (`cmd/eval-extract`): production-faithful calls through agentkit, scorecards out

**Decision.** `cmd/eval-extract` is the workbench's deterministic measure step: point it at a prompt file and a gold corpus, it runs the extract call **exactly as production shapes it** for every case, scores the results (D65), and writes a scorecard. It is a dev tool — built locally, never shipped, and the only composition root besides the service's (it, not `internal/eval`, wires agentkit providers and reads provider keys, per D1/D63).

```
eval-extract run     -prompt <file> -gold <dir> -config <file> -out <scorecard.json>
                     [-split dev|holdout] [-repeat N]
                     [-model <id>] [-provider <p>] [-temperature <f>] [-max-tokens <n>]   # experiment overrides
eval-extract compare -candidate <scorecard.json> -baseline <baseline.json>
```

- **Production-faithful envelope.** `internal/extract` exports two thin seams over its production internals — `extract.Render(instructions string, h DocumentHeader, text string) string` (the exact prompt assembly `Extract` sends) and `extract.Validate(subjects []ExtractedSubject) error` (the exact response validation, D6's rules). The runner calls **these**, so what it measures is what production runs; the functions are the production path, not copies.
- **The model call.** Chat goes through agentkit's anthropic provider (or the provider named by config/flags), with the D64 `eval` pins: temperature 0, thinking off, max_tokens 16384. A response failing `extract.Validate` (or JSON parse) is re-prompted up to `max_parse_retries` times, mirroring production's D5/D6 retry shape. Embeddings for scoring go through agentkit's openai provider behind the D65 `EmbedFunc` seam, wrapped in the disk cache (rooted under `tmp/`). Keys come from the environment (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`); a missing needed key fails loudly before any call.
- **`run`.** Loads config and gold (default split `dev`), runs every case, scores, writes one scorecard (D65's deterministic marshal) to `-out`. `-repeat N` runs the whole set N times and additionally records each run's composite and `Epsilon` over them — how a baseline (and its noise floor) is produced. Flag overrides replace the config's `eval` block for the run and the **effective** call config is echoed verbatim in the scorecard, so cross-config numbers can never be mistaken for comparable.
- **`compare`.** Reads a candidate scorecard and a baseline (with epsilon), applies D65's `Accept`, prints the verdict, and exits 0 on accept / 1 on reject — the loop's keep-or-discard is a shell exit code, not model judgment.
- **Failure doctrine.** Unreadable prompt file, bad config, malformed gold, a case whose model call exhausts retries, a missing key: exit non-zero with an error naming the cause, and write no partial scorecard (write is atomic: temp file + rename).
- **The real-contract proof.** Everything above is unit-testable against fakes except the one claim that matters most: that the runner actually drives real providers. A live smoke test, build-tagged `eval_live` (excluded from the default suite; run explicitly via `go test -tags eval_live ./cmd/eval-extract/...` with real keys), performs one real chat extraction over a one-paragraph document and one real embedding call, asserting a validated non-empty extraction and non-empty vectors come back.

**Rejected.**
- **Routing eval calls through the prompts service** — the workbench's operating condition is a local checkout with no suite running; requiring prompts up would make measurement depend on the very stack it's meant to be independent of.
- **A generic standalone eval tool outside the module** — Go `internal/` visibility would force it to re-implement `Render`/`Validate`, and a reimplementation silently drifts from production, quietly invalidating every score. In-module reuse is the faithfulness guarantee.
- **A shared `evalkit` library now** — one consumer exists; extraction of a generic surface waits until a second service adopts the pattern and shows what generic actually is.
- **Skipping the live test entirely (fakes only)** — a mock accepts whatever it's handed; nothing would prove the agentkit wiring, key handling, and real response shapes actually work. Build-tagging keeps the default suite deterministic while keeping the real proof runnable.

**Verification.**
- R-KY7O-D7JB — Envelope identity: the prompt text the production `Extractor.Extract` path sends (captured via a fake D5 client) is byte-identical to `extract.Render(DefaultPromptInstructions, h, text)` for the same inputs — the exported seam *is* the production assembly, and a response rejected by production's validation is rejected by `extract.Validate`.
- R-KZFK-QZA0 — `run` end-to-end against a fake chat provider and fake `EmbedFunc`: given a prompt file, a seeded gold dir, and the pinned config, it scores every dev case, writes a scorecard whose per-case and aggregate numbers match the D65 scorer's for the same inputs, and exits 0; a response that fails validation once then succeeds is re-prompted and the case still scores.
- R-L0NH-4R0P — Flag overrides: `-model`/`-temperature` replace the config's eval values for the run and the scorecard's echoed call config records the effective (overridden) values verbatim.
- R-L1VD-IIRE — Failure exits: an unreadable prompt file, a malformed gold case, and a chat call that fails past its retries each exit non-zero with an error naming the cause, and no scorecard file is left at `-out`.
- R-L339-WAI3 — `-repeat 3` runs the corpus three times and the output records three per-run composites plus an epsilon equal to their max−min.
- R-L4B6-A28S — `compare`: a candidate strictly above baseline best + epsilon exits 0 and prints accept; equal-or-below exits 1 and prints reject.
- R-L5J2-NTZH — Live smoke (`-tags eval_live`, real `ANTHROPIC_API_KEY` + `OPENAI_API_KEY`): one real agentkit chat call returns an extraction for a one-paragraph document that passes `extract.Validate` with ≥1 subject, and one real embedding call returns a non-empty vector of the configured dimensions — the workbench demonstrably drives the real providers.
