# Decision 47 — nginx: session-gate the whole human surface (root + `/subject/` + `/static/`)

**Decision.** D39 session-gated only the **exact root** `= /srv/wiki/`; everything else under the mount fell to the **bearer** prefix `location /srv/wiki/ { auth_request /_authn; }`. That was correct when the human surface was a single name+version page, but the read surface (D42–D46) now also serves **subject pages** (`/srv/wiki/subject/…`) and references **embedded assets** (`/srv/wiki/static/…`) — both of which a browser reaches with the dashboard **session cookie**, not a bearer token. Under the current fragment they would hit the bearer gate and 401. This Decision adds the two missing **session-gated prefix locations** so the entire human surface is cookie-gated, while the agent surfaces (`/mcp`, `/health`, `/feed`) stay bearer-gated and the PRM doc stays open. The trust boundary is unchanged — nginx remains the sole gate; in-process every route is ungated (D42/D45).

This is a **config** change to `wiki/etc/nginx.conf` (the path-routed location fragment, not a vhost), mirroring the precedent in `sites/etc/nginx.conf` (which gates `= /srv/sites/mcp` with bearer and a session-gated `/srv/sites/private/` prefix beside the session-gated root). It is **structural** — its proof is a named-fragment check in a dedicated plan phase (the pattern D39 used for Phase 64), plus the in-process ungated-routing behaviors already pinned by D42 (R-WC29-XALJ) and D45 — not a Go `R-id` test, because nginx config is not exercised by `go test`.

## The fragment, after this Decision

Order of the location blocks under the mount (most specific wins regardless of source order):

| Location | Gate | Upstream | Serves |
|---|---|---|---|
| `= /srv/wiki/.well-known/oauth-protected-resource` | **open** | `…/.well-known/…` | PRM bootstrap (unchanged) |
| `= /srv/wiki/` | **session** (`/_session-authn`) | `…/` | Home + ask (D43/D44) — **unchanged from D39** |
| `/srv/wiki/subject/` | **session** (`/_session-authn`) | `…/subject/` | Subject pages (D45) — **NEW** |
| `/srv/wiki/static/` | **session** (`/_session-authn`) | `…/static/` | Embedded Carbon assets (D42/D39) — **NEW** |
| `/srv/wiki/` (prefix) | **bearer** (`/_authn`) | `…/` | `/mcp`, `/health`, `/feed` — **unchanged** |

The two new prefix locations copy the existing session-root block verbatim except for the proxy_pass URI, which **preserves** the sub-path (the mount strip keeps the segment after `/srv/wiki/`, so the Go mux sees `/subject/…` and `/static/…`, matching D42's in-process routes):

```nginx
# Subject pages — human web UI, gated by the dashboard browser SESSION (cookie).
# More specific than the bearer `/srv/wiki/` prefix, so it wins for /subject/*.
location /srv/wiki/subject/ {
    auth_request /_session-authn;
    auth_request_set $wiki_owner $upstream_http_x_owner_email;
    proxy_set_header X-Owner-Email $wiki_owner;
    proxy_pass http://127.0.0.1:3001/subject/;   # preserve the /subject/ sub-path
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_http_version 1.1;
}

# Embedded Carbon assets — same session gate (the page and its CSS/fonts are one
# coherent human surface, as D39 established for the exact-root asset reference).
location /srv/wiki/static/ {
    auth_request /_session-authn;
    proxy_pass http://127.0.0.1:3001/static/;    # preserve the /static/ sub-path
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_http_version 1.1;
}
```

The bearer prefix `location /srv/wiki/ { auth_request /_authn; … }` is **untouched** and keeps gating `/mcp`, `/health`, `/feed` for agents, including the existing 429-rate-limit `@wiki_authn_500` handling. `proxy_pass …/static/` forwards `X-Owner-Email`? No — static needs no identity; only the root and `/subject/` forward owner (the ask page records the owner on its `ask` call, D44; the subject page reads no identity).

## Why session, not bearer, for the human prefix

A browser carries the `dashboard_session` cookie, never an opaque bearer; `/_session-authn` is the dashboard-owned, loopback-only endpoint that validates that cookie — the same coarse gate `sites`' private tier uses (any logged-in dashboard user may view any app's human surface; correct for a read-only navigation surface inside the tenant boundary). The agent paths keep the bearer gate because agents carry tokens and must not be reachable by a mere session.

**Rejected.**
- **Leaving `/subject/` and `/static/` on the bearer prefix (the D39 status quo).** Browsers have no bearer → every subject page and the CSS/fonts 401. The whole point of the read surface is human navigation. Rejected.
- **Flipping the catch-all `/srv/wiki/` prefix to `/_session-authn` and carving the agent paths out as exact bearer locations** (the inverse of the chosen shape). Workable, but it would re-gate `/mcp`/`/health`/`/feed` and risks missing one; adding two narrow session prefixes beside the untouched bearer prefix is the smaller, safer diff and leaves the agent surface provably unchanged. Rejected.
- **One session prefix `/srv/wiki/` covering everything and bearer-carving only `= /srv/wiki/mcp`.** `/health` and `/feed` would then be session-gated, changing agent/monitoring behavior. The additive carve-outs leave them exactly as they are. Rejected.
- **Serving `/static/` open (no gate).** The suite exposes nothing un-gated except the PRM doc; assets ride the same session gate as the pages they style (D39's established stance). Rejected.

**Verification.** Structural — **no `R-id`**. Proven by:
- a **named-fragment check** plan phase (the D39 Phase 64 pattern): assert `wiki/etc/nginx.conf` contains the two new `location /srv/wiki/subject/` and `location /srv/wiki/static/` blocks, each with `auth_request /_session-authn` and a `proxy_pass` that preserves its sub-path, and that the bearer `location /srv/wiki/` prefix and the PRM/exact-root blocks are intact;
- the in-process ungated-routing behaviors it relies on, already pinned by **D42 R-WC29-XALJ** (the `/subject/` route is mounted and collision-free) and **D45** (subject page served with no identity header), and **D39 R-LAND-UNGT** (root ungated in-process).
