# Decision 48 — The markdown → sanitized-HTML rendering seam (`internal/markdown`)

**Decision.** The read surface (D44 ask answers, D45 subject prose) renders
LLM-authored prose as **formatted, sanitized HTML** instead of escaped raw text.
A small pure package `internal/markdown` owns that transform behind one function;
the web templates reach it through a registered template func.

```go
// Package markdown renders LLM-authored wiki prose to safe, styled HTML for the
// read surface. It adds no domain logic and mutates nothing.
package markdown

// Render parses prose as CommonMark + GFM (tables) and returns sanitized HTML
// safe to embed directly in a template. Raw/dangerous HTML is stripped and only
// an allowlist of formatting elements and safe-scheme links survives, so the
// result can never carry active markup (no <script>, no javascript: URL).
// Plain prose with no markdown round-trips to a <p> paragraph.
func Render(prose string) template.HTML
```

**Pipeline (two vetted libraries, in order):**

1. **`goldmark`** parses prose and renders HTML. Configured with the **GFM
   extension** (`extension.GFM` — tables, strikethrough, autolink, task lists)
   for tables, and **without** `goldmark.WithUnsafe()`, so goldmark's default
   behavior — *escaping* raw HTML blocks and inline HTML — already neutralizes an
   embedded `<script>`.
2. **`bluemonday`** (`UGCPolicy()`) sanitizes goldmark's HTML output as
   defense-in-depth. It enforces the element allowlist (`h1`–`h6`, `p`, `strong`,
   `em`, `del`, `ul`/`ol`/`li`, `code`/`pre`, `blockquote`, GFM
   `table`/`thead`/`tbody`/`tr`/`th`/`td`, `a`) and a **URL-scheme allowlist**
   (http/https/mailto) — the piece goldmark alone does *not* provide, so a
   `[x](javascript:…)` link loses its dangerous href.

Both are pure-Go, no cgo (consistent with the `modernc.org/sqlite` constraint),
proxy-fetched like `github.com/ikigenba/agentkit` (no committed `replace`).

**Wiring.** The renderer is exposed to the web layer as a `template.FuncMap`
entry `md`, so `homeTemplates`/`subjectTemplates` (D42) are built with
`.Funcs(template.FuncMap{"md": markdown.Render})` and the templates call
`{{ md .Subject.Body }}` (D45) and `{{ md .Answer.Text }}` (D44). Because `md`
returns `template.HTML`, `html/template` emits it without re-escaping. The
`SubjectView` / `ask.Answer` struct shapes are **unchanged** — the prose stays a
plain `string` field; only the *rendering* at the template boundary changes. The
renderer and sanitizer policy are built once as package-level singletons (both
are concurrency-safe for reuse).

**Boundary.** This package is pure (string → `template.HTML`), no IO, no DB, no
identity. It owns *rendering and safety* only; *which* prose is rendered and the
footer/link sets stay with D44/D45; the CSS that styles the emitted elements is
D49.

**Rejected.**
- **Hand-rolled markdown renderer + sanitizer.** Markdown is a large grammar and
  HTML sanitization is security-critical; rolling either invites both bugs and
  XSS holes. Use vetted libraries — this is exactly where discipline is the wrong
  tool. Rejected.
- **`goldmark` alone (no sanitizer).** goldmark's default-escape stops raw-HTML
  injection, but it does **not** filter link URL schemes — `[x](javascript:…)`
  would render an executable href. bluemonday closes that gap and gives an
  explicit allowlist. Rejected.
- **`goldmark` with `WithUnsafe()` then trust bluemonday.** Enabling raw-HTML
  passthrough widens the attack surface and leans the entire safety guarantee on
  one library; keeping goldmark in its safe default is defense-in-depth. Rejected.
- **`blackfriday` / `gomarkdown`.** Less actively maintained and weaker GFM-table
  support than goldmark. Rejected.
- **Returning `string` and converting in the handler.** Forces every call site to
  remember the `template.HTML` cast (an easy place to drop the safety contract or,
  worse, mark unsanitized text safe). Returning `template.HTML` from the one
  audited function keeps the safe boundary in a single place. Rejected.

**Verification.** Unit tests call `markdown.Render` directly against the **real**
goldmark + bluemonday (no mock, no network — the real libraries are the substrate
that can falsify each claim). Each asserts on the returned HTML string.

- R-SS0J-U7PG — **Headings render as heading elements.** `Render("## Acme")`
  output contains an `<h2` element wrapping `Acme` and does **not** contain the
  literal `## Acme`.
- R-ST8G-7ZG5 — **Emphasis renders.** `Render("**bold** and *italic*")` contains
  `<strong>bold</strong>` and `<em>italic</em>`, with no literal `**`/`*`.
- R-SUGC-LR6U — **Lists render.** `Render` of a `-` bulleted block yields
  `<ul>…<li>` and of a `1.` block yields `<ol>…<li>`, not literal `-`/`1.` lines.
- R-SVO8-ZIXJ — **Inline and fenced code render.** Inline `` `x` `` yields a
  `<code>` element; a triple-backtick fenced block yields `<pre>` wrapping
  `<code>` — not literal backticks.
- R-SWW5-DAO8 — **Blockquotes render.** `Render("> quoted")` yields a
  `<blockquote>` containing `quoted`, not a literal `>`.
- R-SY41-R2EX — **GFM tables render.** A pipe table
  (`| A | B |\n| - | - |\n| 1 | 2 |`) yields a `<table>` with `<th>`/`<td>` cells
  (`A`,`B`,`1`,`2`) and contains no literal `|` row text — proving the GFM table
  extension is enabled (a base-CommonMark renderer would leave the pipes).
- R-SZBY-4U5M — **Safe inline links survive.** `Render("[Acme](https://x.test)")`
  yields `<a` with `href="https://x.test"` wrapping `Acme`.
- R-T0JU-ILWB — **Embedded raw HTML is stripped.** `Render` of prose containing
  `<script>alert(1)</script>` returns output with **no** `<script` substring and
  no executable element — the raw tag does not survive (neither rendered nor as
  the literal active tag).
- R-T1RQ-WDN0 — **`javascript:` link schemes are neutralized.** `Render("[x](javascript:alert(1))")`
  returns output whose any emitted `<a>` carries **no** `href` beginning
  `javascript:` — the dangerous scheme is removed by the URL allowlist (a wrong
  impl using goldmark alone would emit `href="javascript:alert(1)"` and fail this).
- R-T2ZN-A5DP — **Plain prose round-trips to a paragraph.** `Render("Just a sentence.")`
  yields `<p>Just a sentence.</p>` (text intact, wrapped in a paragraph), proving
  unmarked prose renders cleanly rather than being dropped or doubled.
