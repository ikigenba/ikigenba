# Decision 46 — The orphan index: subjects with zero inbound mentions (read-time, single-pass)

**Decision.** The home page (D43) lists the **orphans** — subjects nothing links to — so every subject is reachable even when no page's prose names it. A subject `X` is an **orphan** iff it has **zero inbound mentions**: no *other* subject's page body names `X` by its `norm_name` or by any D25 alias key that resolves to `X` (the exact D12 matching). This is precisely "`X` appears in no page's `MentionedBy`."

**Computed read-time, not stored — but in one corpus pass, not N² scans.** This honors D12's standing decision (links are projected at read time; a materialized `page_links` table is a deferred optimization — no migration, no write-path change, always correct across drift). But D12's per-subject `PageWithLinks` scans every other page for **one** subject; doing that for every subject to build the orphan list would be O(subjects × pages). Instead the orphan computation **inverts** it into a single pass:

```go
// Orphans returns every subject with zero inbound mentions across the current
// corpus, ordered deterministically by public path (D11). Read-time; no stored
// links table, no migration.
//
// One pass, O(total corpus size):
//   1. Load all subjects and all D25 aliases (AliasStore.ListAll) — same data D12
//      uses — and build keyIndex: norm_name ∪ alias keys  →  CANONICAL subject id.
//   2. referenced := {}. For each page body P (owned by subject S_P):
//        for each subject id m in Mentions(P.body, keyIndex) with m != S_P:
//            referenced.add(m)           // self-mention never rescues (D12 self-exclude)
//   3. return [ s for s in subjects if s.id not in referenced ]   // sorted by Path
//
// Matching is the exact D12 rule (whole, hyphen/edge-bounded run over Normalize(body));
// a match via an alias key counts toward the CANONICAL subject, so a page naming X
// only by a folded short name makes X non-orphan.
func (s *Service) Orphans(ctx context.Context) ([]Subject, error)
```

The single pass scans each page body **once** against a shared key index (built from the same subjects+aliases D12 loads), so cost is linear in the corpus (subjects + total page bytes), not quadratic. For a single-box wiki (hundreds–low-thousands of subjects, each page ≤ 12,000 chars, D3) that is a few MB scanned per home view — milliseconds, no caching needed. The web adapter maps `[]Subject` → `[]web.Ref{Href:"subject/"+Path(s), Name:s.Name}`.

**Orphan-ness is defined on inbound mentions only**, independent of whether the orphan itself has a compiled page: `X` is referenced iff some *other* body names it. The design target is the common case where every subject has a page (compile is part of integration, D4/D7); a page-less subject simply contributes no body to the scan and is evaluated for inbound like any other.

**Self-mention does not rescue.** A page naming its own subject is excluded (`m != S_P`, mirroring D12's self-exclusion), so a subject whose only mention is in its own prose is still an orphan — correct, because no *navigation* reaches it.

**Rejected.**
- **A materialized `orphans` / `page_links` table maintained on the ingest/merge write path.** Faster reads, but adds a migration, a write-path step, and a second source of truth that drifts on the new-subject and merge-alias axes — exactly what D12 rejected for links. Read-time projection is always correct; at single-box scale the single pass is fast enough. Materialization stays a deferred optimization (revisit only if the corpus pass ever shows up in a real home-render profile). Rejected for this release.
- **Reusing `PageWithLinks` per subject to test each for empty `MentionedBy`.** O(subjects × pages) — re-scans the whole corpus once per subject. The inverted single pass produces the same set in one corpus scan. Rejected.
- **Naive `Mentions(body, allSubjectKeys)` per body with per-key boundary search (O(keys × body) each).** Correct but O(subjects × corpus); the shared key index makes each body O(its own size). (The matcher contract is unchanged — this is how the corpus pass consults it.) Rejected as the implementation strategy.
- **In-memory memoization of the orphan set (invalidated by a corpus generation counter).** A real option if profiling ever demands it, but it adds invalidation state for no measured need at this scale. Deferred. Rejected for now.
- **Listing page-less subjects specially, or excluding orphans that lack a page.** Over-specifies an edge case the integration pipeline does not normally produce; orphan-ness is inbound-only and uniform. Rejected.

**Verification.** Against a **real temp SQLite** (migrated, composing `SubjectStore`/`AliasStore`/`PageStore` as the service does) — orphan detection is a property of real subjects, aliases, and stored page bodies, so it is proven on the real engine with the real D12 matcher, not a mock.

- R-QSR2-AFAD — **Zero-inbound is the membership rule.** With subjects `A`, `B`, `C` where `A`'s page body names `B` (so `B` is referenced) and **no** page names `C`, `Orphans` returns `C` and **not** `B` — a subject mentioned by another page is excluded, a subject mentioned by no page is included. *(real temp SQLite)*
- R-QTYY-O712 — **Self-mention does not rescue.** A subject `S` whose **only** occurrence (by its norm_name) anywhere is inside `S`'s **own** page body — named by no other page — is returned by `Orphans` (still an orphan); adding a second subject whose page names `S` then removes `S` from the result. *(real temp SQLite)*
- R-QV6V-1YRR — **Alias mentions count as inbound (canonical).** With an alias `vasari → W` installed and another subject `F` whose page body names only the bare "Vasari" (W's norm_name absent from F's body), `W` is **not** in `Orphans` — the alias mention is inbound to the canonical `W`; remove the alias row and, with no page naming `W`'s canonical name, `W` becomes an orphan. *(real temp SQLite; composes with D12 R-1XX5-QDCY / R-1Z52-453N)*
- R-QWER-FQIG — **Deterministic order.** `Orphans` returns its subjects ordered by public `type/slug` path (D11), stable across calls on identical DB state. *(real temp SQLite)*
