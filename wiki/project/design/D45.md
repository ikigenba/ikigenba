# Decision 45 — The subject page (`GET /subject/{type}/{slug}`)

**Decision.** Following any link lands here. The route (D42) rejoins `{type}/{slug}` into the public path (D11) and calls the injected `PageFinder.PageByPath(ctx, path)`, which resolves the subject **alias-aware** (D29 `Resolver.ResolveByPath` — a folded-away path forwards to the survivor) and projects its links read-time (D12 `Service.PageWithLinks`). The page renders the compiled prose with a footer of **outbound** + **inbound** subject links. The web adds no domain logic — it is an HTML rendering of D12/D29 data.

## The `PageFinder` seam returns structured links, not a markdown footer

The MCP page surface returns `{Path, Title, Body}` where `Body` already has D12's **markdown** footer baked in (`RenderFooter`). The web needs the prose and the link sets **separately**, to render HTML `<a>` links. So the web seam returns a structured view; the `cmd/wiki/main.go` adapter builds it from `PageWithLinks` **without** calling `RenderFooter`:

```go
// SubjectView is the web projection of a subject's page (D12 LinkedPage).
type SubjectView struct {
    Title    string // page title (the subject's display name)
    Body     string // the COMPILED PROSE only — no footer (RenderFooter is MCP/markdown)
    Outbound []Ref  // D12 Mentions:    subjects this page's prose names (by name/alias)
    Inbound  []Ref  // D12 MentionedBy: subjects whose page names this one
}
// PageFinder.PageByPath resolves path alias-aware (D29) and projects links (D12).
// A genuinely-unknown path (D29 ErrSubjectNotFound) maps to web.ErrNotFound.
var ErrNotFound = errors.New("web: subject not found")
```

Adapter: `Resolver.ResolveByPath(path)` → on hit, `svc.PageWithLinks(subject.ID)` → `SubjectView{Title: page.Title, Body: page.Body (stored prose), Outbound: refs(page.Mentions), Inbound: refs(page.MentionedBy)}`, mapping each `wiki.Ref{Path,Name}` → web `Ref{Href:"subject/"+Path, Name}`. On `ErrSubjectNotFound` → `ErrNotFound`. Because the served fields derive from the **resolved** subject, an alias-forwarded path renders **identically** to the survivor's own path (D29 `R-AL5R-PL1P`).

## Layout (`subject.tmpl`)

- **"Ask another question"** at top → `.` (the base = reset home D43). Present on every subject page, including not-found.
- **Title** (H1, display type) — the subject's name.
- **Compiled prose body** — `SubjectView.Body` rendered as **sanitized markdown HTML** via the D48 `md` template func (`{{ md .Subject.Body }}`), so the compiled page reads like an article (headings, emphasis, lists, GFM tables, and any inline links the prose contains) rather than showing literal `##`/`**`. The injection guard is D48's sanitizer — raw HTML is stripped and `javascript:` link schemes neutralized — not template escaping. The not-found body ("No page exists for this subject.") is fixed plain text and `md` round-trips it to a paragraph.
- **Footer** — `RenderFooter`'s HTML analogue, built from the structured Refs:
  - **Mentions** (outbound) before **Mentioned by** (inbound); each is a list of `<a href="{{.Href}}">{{.Name}}</a>`.
  - A section is omitted when its slice is empty; the **whole footer** (and its rule) is omitted when both are empty — mirroring D12's footer discipline. De-duplicated and path-ordered by the adapter (D12).

## Not-found

A path resolving to neither a live subject nor an alias (D29 `ErrSubjectNotFound` → web `ErrNotFound`) renders a **styled Carbon 404 page** — same layout shell, the "ask another question"/home affordance intact — with HTTP status **404**, never a raw `http.Error` text or a 500. The human who followed a stale link gets an on-brand page back to home.

**Rejected.**
- **Reusing the MCP `{Path,Title,Body}` (markdown-footer) shape.** The web must render HTML links and place outbound/inbound as styled sections; a pre-baked markdown footer string can't be turned back into structured links without re-parsing. The seam returns structured Refs + prose. Rejected.
- **Emitting the body as plain HTML-escaped text** (the prior approach). Showed the compiled prose's literal `##`/`**`/`|` instead of formatting — the product now requires the page to read like an article. The injection concern it addressed is handled instead by D48's sanitizer (raw HTML stripped, `javascript:` neutralized), which is stronger than escaping and still emits formatted output. Rejected.
- **A bare `http.NotFound` (plain text) on a miss.** Breaks the on-brand human surface; a person following a dead link should get a styled page home, not `404 page not found`. Rejected.
- **Making the web seam alias-*unaware* (plain `GetByPath`).** Would 404 a folded-away path that D29 forwards for `page`/`claims`, making the web inconsistent with MCP. The seam uses `ResolveByPath`. Rejected.
