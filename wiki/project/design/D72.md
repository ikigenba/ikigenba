# Decision 72 — The four scorers: deterministic where mechanical, sol-judged where prose

**Decision.** Each tune folder ships one `score` executable satisfying the external tool's scorer contract (research §13: argv = case dir, stdin = model output, stdout = `{"score": <0..1>, "feedback": "..."}`, non-zero exit aborts the run). All four are python3 scripts, self-contained in their folder, with their scoring constants (weights, thresholds) declared at the top of the script — the scorer's business, not autotune's. Credentials: scorers that call out read `OPENAI_API_KEY` from the environment (the repo `.envrc`, D1/D70); embeddings go through the external `embed` CLI (`EMBED_BIN` override; disk-cached, so warm-cache scoring is offline and free).

- **extract** — the deterministic subject/claim/field scorer, taken **verbatim** from the autotune repo's example (research §13): claim alignment by cosine over `embed` (`text-embedding-3-small` @ 1536, threshold 0.80, margin 0.03), weights 0.35 subject / 0.50 claim / 0.15 field. `gold.json`: the gold subjects/claims (the example's shape).
- **analysis** — the deterministic three-list scorer **ported from the retired `internal/eval` Go implementation to python3**, preserving its math: the `sub_queries`, `keywords`, and `aliases` list scores and their composite, with embedding-assisted alignment through the same `embed` pins. `gold.json`: the gold analysis lists (the retired workbench's gold shape).
- **compile** — **hybrid**: deterministic contract gates first, judge second, combined by top-of-script weights.
  - *Deterministic gates* (no model consulted): output parses to the `{title, body}` contract with both non-empty; body ≤ 12,000 characters (the product cap); every inline `[job-id]` citation in the body refers to a claim id present in the case's `input.txt`; no invented citation. A gate violation floors that component's score deterministically — identical output, identical penalty, every run.
  - *Judge* (one call, `gpt-5.6-sol`, the folder's committed `judge-prompt.txt` rubric, effort pinned): scores coverage of the case's gold key facts, absence of facts beyond the supplied claims, lead discipline, and organization, returning rubric-anchored numeric subscores. The same rubric prompt scores dev and holdout — never a per-split variant.
  - `gold.json`: the key facts the page must convey plus per-case notes the judge receives.
- **synthesis** — same hybrid pattern: deterministic gates (parses to the `{found, text, citations}` contract; every citation names a page supplied in `input.txt`; a `found:true` answer carries ≥1 citation; an expected-empty case answers `found:false`), then a sol judge for groundedness (no facts beyond the supplied pages) and completeness against the gold key points. `gold.json`: expected key points, expected `found` value, and the allowed citation set.
- **Self-tests, offline.** Each folder ships `fixtures/`: canned model outputs with hand-computed expected results. The `autotune` test package (D71) shells each scorer over its fixtures and asserts the output — the deterministic paths (extract and analysis fully; compile/synthesis gate paths with the judge **skipped via a `SCORE_SKIP_JUDGE=1` env hook** the scripts honor for exactly this purpose) run in the ordinary green suite with no network (extract/analysis fixtures pre-warm the `embed` cache or use fixture vectors). A wrong constant or a broken port fails `go test ./...`, not the first paid run.
- **The judge is a real external contract.** A judge call that "works" against a stub proves nothing — the two judge paths each get a **live** id, exercised only when the operator supplies the environment (the `WIKI_TUNE_LIVE=1` + `OPENAI_API_KEY` gate, skipping otherwise — the R-15NY-IF46 pattern), asserting an actual sol call returns rubric-shaped subscores for a fixture case.
- **Improver/judge same-model caveat, on the record:** the improver and the judge are both sol, so the improver can drift toward the judge's tastes. The guards are architectural: the deterministic gates are un-gameable, holdout is judged with the identical rubric, and adoption stays a manual operator decision.

**Rejected.**
- **Pure LLM-judge scores for compile/synthesis** — noisy, and the improver optimizes the judge instead of the contract; the mechanical majority of each contract (caps, citations, shape) is checkable for free.
- **Pure deterministic scoring for compile/synthesis** — prose quality (coverage, organization, groundedness nuance) has no closed-form metric; pretending otherwise scores formatting, not quality.
- **A different judge model than sol** — the operator pinned "evaluate everything with gpt-5.6-sol"; a second model is a second contract to maintain.
- **Scorers calling the OpenAI API directly for embeddings** — the `embed` CLI exists precisely to be the machine's single embedding credential/cache boundary; bypassing it re-pays for every re-run and forks the pins.
- **Rewriting the extract scorer** — the example's port is verified against the workbench's numbers; verbatim reuse keeps one source of truth.

**Verification.**
- R-AAOL-YG21 — The extract `score`, run over a fixture case with a canned model output on stdin, exits 0 and emits `{"score": s}` with `s` equal to the fixture's hand-computed expected value; a malformed candidate output yields the fixture's expected floor score, not a crash.
- R-ABWI-C7SQ — The analysis `score`, run over a fixture case, reproduces the retired Go scorer's math: the three list scores and composite equal the fixture's expected values computed under that math.
- R-AD4E-PZJF — The compile `score` with the judge skipped applies its deterministic gates: an over-cap body, a citation to a claim id absent from `input.txt`, and a malformed `{title, body}` each score the fixture's expected floored value, while a clean fixture output passes the gates — identical inputs, identical scores, across two runs.
- R-AECB-3RA4 — The synthesis `score` with the judge skipped applies its deterministic gates: a citation to an unsupplied page, a `found:true` answer with zero citations, and a `found:true` answer on an expected-empty case each score the fixture's expected floored value; a clean fixture output passes — identical inputs, identical scores, across two runs.
- R-AFK7-HJ0T — *(live; runs only with `WIKI_TUNE_LIVE=1` and `OPENAI_API_KEY` set, else skips)* The compile scorer's judge path completes a real `gpt-5.6-sol` call for a fixture case and returns a total score in `[0,1]` composed from actual rubric subscores — the judge contract works against the real provider, not a stub.
- R-AGS3-VARI — *(live; same gate)* The synthesis scorer's judge path completes a real `gpt-5.6-sol` call for a fixture case and returns a total score in `[0,1]` composed from actual rubric subscores.
