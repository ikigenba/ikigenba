# Decision 1 — Dependency on the prompts service (unified inference; no agentkit)

**Decision.** wiki performs **no direct provider inference**. Its inference dependency is the suite's **prompts service**, reached over the loopback plumbing tier: `POST /complete` for chat-shaped one-shots and `POST /embed` for embeddings (research §12 documents the exact API footprint). The `github.com/ikigenba/agentkit` module dependency is **removed entirely**: no `require`, no types in any signature, no provider keys.

- **Address resolution:** the composition root resolves prompts' base URL with `registry.BaseURL("prompts")` (the in-repo `registry` replace-sibling wiki already depends on) and injects it as a plain string into the D5 client. No env var, no hardcoded port, no second source of truth.
- **Keys leave wiki:** `ANTHROPIC_API_KEY` and `OPENAI_API_KEY` are no longer read anywhere in wiki; provider credentials live with prompts alone. wiki's config keeps only its own knobs (per-site generation settings D19, embed knobs D34).
- **Prod build:** `bin/ship` (`GOWORK=off`) builds with no agentkit resolution at all; the module graph is the appkit/eventplane/registry replace-siblings plus the pure-Go sqlite/markdown dependencies.
- **Failure doctrine:** prompts down or erroring ⇒ wiki's call fails loudly to its caller (job error, ask error). No wiki-side fallback provider, no queueing, no transport-level retry (D5).

**Rejected.**
- **Keeping agentkit beside the prompts client** (dual back ends behind D5) — leaves a second inference path and a keyed provider in wiki, defeating the suite decision that prompts is the sole inference service and the sole spend ledger; also leaves exactly the orphaned code the conversion is required to remove (D63).
- **A wiki env var for prompts' base URL** — the registry library is the suite's authoritative service→port table; an env override would be a second, drift-prone source. Tests inject a `httptest` URL through the constructor instead.
- **Calling prompts through MCP** — `/mcp` is the product surface for agents; sibling daemons use the loopback plumbing endpoints precisely so inference consumers need no MCP handshake on a hot path (prompts' own design).

**Verification.**
- R-0UOV-2HFX — A production-shaped build of wiki (`GOWORK=off`, no `go.work` in scope) compiles with `github.com/ikigenba/agentkit` absent from the module graph (`go mod graph | grep agentkit` is empty).
- R-0VWR-G96M — `wiki/go.mod` contains **no** `require` and **no** `replace` for `github.com/ikigenba/agentkit`; its only `replace` directives are the in-repo `appkit`, `eventplane`, and `registry` siblings.
