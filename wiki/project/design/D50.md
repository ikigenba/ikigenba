# Decision 50 — Eliminate the web-font FOUT (font-display + self-served fonts + preload)

**Problem.** The rendered read surface (home, ask, subject pages) flashes: it
paints with the fallback system font, then visibly **swaps** to the web fonts a
moment later. Two causes compound:

1. Every `@font-face` block in the embedded `internal/web/static/tokens.css`
   declares `font-display: swap`, which by design paints fallback first and swaps
   to the web font whenever it finishes loading — i.e. *after* first paint.
2. The `@font-face` `src` is **origin-absolute** — `url('/static/fonts/X.woff2')`.
   CSS `url()` resolves against the document origin, not the `<base href>`, so
   under the `/srv/wiki/` mount it resolves to `https://<host>/static/fonts/…`
   (the apex/dashboard), **not** `/srv/wiki/static/fonts/…`. wiki silently borrows
   the dashboard's fonts and its own embedded `static/fonts/*.woff2` (served by
   `StaticHandler` at `GET /static/`) go unused. Nothing preloads the fonts, so
   the fetch only starts after the stylesheet is parsed, guaranteeing the swap
   lands after first paint.

**Decision.** Three coordinated changes; no Go logic, no migration, no new
dependency. The change set is confined to `internal/web/static/tokens.css` and
`internal/web/layout.tmpl` (the single shared `<head>`).

1. **`font-display: swap` → `font-display: optional`** on all four `@font-face`
   blocks. `optional` gives the browser a short (~100 ms) block window and then
   commits to whatever is available **with no swap after first paint** — this is
   what removes the jar.

2. **Self-serve the fonts: `src: url('/static/fonts/X.woff2')` →
   `url('fonts/X.woff2')`** on all four blocks. Relative to the stylesheet at
   `/srv/wiki/static/tokens.css`, this resolves to `/srv/wiki/static/fonts/X.woff2`
   — the service's own embedded fonts via `StaticHandler` — so wiki is
   self-contained and the URL aligns with the preload target (below).

3. **Preload the above-the-fold fonts** in `layout.tmpl`'s `<head>` (the one
   template shared by every page via `homeTemplates`/`subjectTemplates`). Add,
   alongside the existing `<link rel="stylesheet" href="static/tokens.css">` and
   under the existing `<base href="{{.Mount}}">`:

   ```html
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/space-grotesk.woff2">
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/ibm-plex-sans.woff2">
   ```

   These are the **display** (`Space Grotesk`, headings) and **body**
   (`IBM Plex Sans`) families painted immediately. `crossorigin` is **mandatory**
   — fonts are always fetched in CORS mode, so a preload without it double-fetches
   and the preload is wasted. The `href` is base-relative (resolves to
   `/srv/wiki/static/fonts/…`), **byte-identical to the new `@font-face` `src`
   target**, so the preloaded bytes satisfy the `@font-face` request. With the
   font in cache inside the `optional` window, first paint uses the real font.

The mono family (`IBM Plex Mono`, used in the footer label and `code`/`pre`) is
**intentionally not preloaded**: it is minor/below the fold, and `optional`
guarantees no jar regardless — at worst its first uncached paint uses the fallback
and the real font is cached for the next view. Preloading it would compete for
bandwidth with the two fonts that matter at first paint.

**Boundary.** D50 owns only the font-loading behavior of the read surface
(`tokens.css` `@font-face` rules + the `<head>` preload). It does not touch the
design-token *values* (the `--font-*`/`--color-*`/`--space-*` variables are
unchanged), the markdown CSS (D49), or any handler/route — `StaticHandler` (Phase
63/72) already serves `static/fonts/*.woff2`.

**Rejected.**
- **`font-display: block`.** FOIT — text is invisible during the block period,
  then pops in. Trades a font swap for a text flash; still a flash. Rejected.
- **`font-display: fallback`.** Still permits a swap inside its (longer) swap
  window on a slow load. `optional` is the only value that forbids a post-paint
  swap outright. Rejected.
- **Preload only, keep `swap`.** If the network is slow enough that the font
  misses first paint, `swap` still swaps. Preload reduces the odds; `optional`
  removes the failure mode. Rejected as insufficient alone.
- **Keep the origin-absolute `src`.** Leaves wiki dependent on the apex serving
  fonts and makes the preload target (`/srv/wiki/static/fonts/…`) differ from the
  `@font-face` request (`/static/fonts/…`), so the preload would be wasted (double
  fetch). Rejected.

**Verification** (httptest against the `NewHandler` read surface and the
`StaticHandler` static route; the embedded `tokens.css` and the rendered `<head>`
are both reachable from responses).

- R-KFVF-EMEO — **No `@font-face` declares `font-display: swap`; all use
  `optional`.** Reading the embedded `internal/web/static/tokens.css`, every
  `@font-face` block contains `font-display: optional` and the file contains **no**
  `font-display: swap` occurrence — a block left as `swap` (the jar-causing value)
  fails this. *(asset read / served `GET /static/tokens.css`)*
- R-KH3B-SE5D — **Font `src` is service-relative, not origin-absolute.** The
  embedded `tokens.css` contains **no** `url('/static/fonts/` (origin-absolute,
  apex-resolving) occurrence, and each `@font-face` `src` uses the mount-relative
  `url('fonts/…woff2')` form — proving the service serves its own fonts and the
  URL matches the preload target. *(asset read)*
- R-KIB8-65W2 — **The rendered `<head>` preloads the display and body fonts with
  `crossorigin`.** `GET /` (home) and `GET /subject/{type}/{slug}` (with a stub
  `PageFinder`) each return a `<head>` containing a `<link rel="preload" as="font"
  type="font/woff2" crossorigin …>` for both `space-grotesk.woff2` and
  `ibm-plex-sans.woff2`, each `href` resolving to the same `static/fonts/…` path as
  the `@font-face` `src` — a missing `crossorigin`, a missing family, or a
  mismatched path fails this. *(httptest, stub `PageFinder`)*
- R-KJJ4-JXMR — **The service actually serves its own fonts at the new path.**
  `GET /static/fonts/space-grotesk.woff2` and `GET /static/fonts/ibm-plex-sans.woff2`
  through the real `StaticHandler` each return `200` with a woff2 content type
  (`font/woff2` or `application/octet-stream`) and a non-empty body — exercising
  the real embedded-FS route the preload/`src` now point at, not merely asserting a
  CSS string. A 404 here (font not embedded/served at this path) fails this.
  *(httptest against the real `StaticHandler`)*
