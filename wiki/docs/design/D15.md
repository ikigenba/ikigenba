# Decision 15 — Cursor pagination: the contract + the list seams

**Decision.** Every listing that can grow without bound — jobs, subjects, a subject's claims, the LLM-call footprint — returns a **bounded page plus an opaque cursor** for the next, with any filter applied **before** paging (product: "Page large results"). Pagination is **keyset (seek)**, not offset: each list orders by a stable, unique key and the cursor carries the last row's key, so the next page asks for rows strictly *after* it. This is O(log n) per page and immune to skip/duplicate under concurrent inserts — `LIMIT/OFFSET` is O(n) and shifts rows when the set changes underneath the pager.

## Order keys and filters

Each collection orders by `(sort_column, id)` with the ULID `id` as the unique tiebreak; the filter narrows the set in the same `WHERE` before the keyset predicate:

| collection | order key | filters |
|---|---|---|
| jobs | `received_at, id` | `status` (one of the five), time range (`received_at` since/until) |
| subjects | `name, id` | `type`, name-substring (`norm_name LIKE`) — existing filters, now paged |
| claims (by subject) | `id` | `subject` (required) |
| llm_calls | `started_at, id` | `job_id`, `stage`, time range |

## The `internal/page` package (pure; no DB)

Owns the request shape and the cursor codec, table-tested directly:

```go
package page

const (
    DefaultLimit = 50
    MaxLimit     = 200
)

// Params is a cursor-pagination request.
type Params struct {
    Limit  int    // clamped to [1, MaxLimit]; 0 → DefaultLimit
    Cursor string // opaque; "" = first page
}

// EncodeCursor packs ordered key components into one opaque token (base64url over a
// delimited join). DecodeCursor reverses it; ok=false on a malformed token.
func EncodeCursor(parts ...string) string
func DecodeCursor(token string) (parts []string, ok bool)
```

A **malformed cursor is a clean error, never a silent reset to page 1** (fail loudly): a cursor we cannot decode is a client bug to surface, not to paper over by silently restarting from the top (which can loop a client forever re-reading page 1). The decode failure propagates as a typed error the MCP layer renders as a tool error (D16).

## The store seams

Stores return `(items, nextCursor, error)` — a plain tuple the MCP layer wraps as `{items, next_cursor}` (D16); no generic result struct.

```go
type JobFilter     struct { Status string; Since, Until time.Time }
type LLMCallFilter struct { JobID, Stage string; Since, Until time.Time }

func (s *JobStore)     ListJobs(ctx, f JobFilter, p page.Params)          (jobs []Job, next string, err error)
func (s *SubjectStore) List(ctx, typ, nameContains string, p page.Params) (subjects []Subject, next string, err error) // reshaped
func (s *ClaimStore)   ListBySubject(ctx, subjectID string, p page.Params) (claims []Claim, next string, err error)     // reshaped
func (s *LLMCallStore) List(ctx, f LLMCallFilter, p page.Params)          (calls []CallRecord, next string, err error)
```

## Mechanics

Each query appends the keyset predicate in the explicit portable form and over-fetches by one to detect a further page:

```sql
... WHERE <filters>
    AND (sort_col > ?1 OR (sort_col = ?1 AND id > ?2))   -- omitted entirely when cursor is ""
  ORDER BY sort_col, id
  LIMIT <clampedLimit> + 1
```

If the `+1` row comes back, drop it and set `nextCursor = EncodeCursor(lastReturned.sortCol, lastReturned.id)`; otherwise `nextCursor = ""` (last page). The single-column claims key encodes just `id`. Indexes backing the orderings: the existing `jobs_pending_received (status, received_at, id)`, `claims_subject (subject_id)`, and the D13 `llm_calls_*` indexes; this decision's migration adds `subjects (name, id)`.

**How it's tested.** Against a real temp SQLite migrated by the appkit runner:
- Insert N rows and page through at limit `k`: **every row appears exactly once, in key order, and `nextCursor` is empty only on the final page** (no skips, no duplicates across page boundaries).
- A filter narrows before paging: status (jobs), stage / job_id / time-range (llm_calls), type / name-substring (subjects), subject (claims) — only matching rows are paged.
- `Limit` clamps: `0 → DefaultLimit`, `> MaxLimit → MaxLimit`.
- Codec round-trips as a table test (`Decode(Encode(parts)) == parts`); a corrupted token decodes to `ok=false` and the store list returns a clean error rather than silently restarting.

**Rejected.**
- **`LIMIT/OFFSET` pagination** — O(n) scan to the offset and unstable: an insert/delete before the offset shifts every later page, skipping or repeating rows. Keyset is strictly better for an append-heavy footprint. Rejected.
- **A transparent (plaintext) cursor exposing the raw key** — invites clients to construct or mutate cursors and couples them to the sort columns; opacity keeps the key shape a private implementation detail we can change. Rejected.
- **Silent reset to the first page on a bad cursor** — hides a client bug and risks an infinite re-read loop. Fail loudly. Rejected.
- **A generic `Page[T]` result struct** — adds generic-field friction for no gain; a `(items, next, err)` tuple is simpler and the MCP layer shapes the wire form. Rejected.

**Verification.**
- R-17C5-VP2I — Paging a collection of N rows at limit `k` yields every row exactly once in key order across pages, with `nextCursor` non-empty until the final page and empty on it (no skipped or duplicated rows at page boundaries).
- R-18K2-9GT7 — A filter is applied before paging: a status filter (jobs), a `stage`/`job_id`/time-range filter (llm_calls), a `type`/name-substring filter (subjects), and the required `subject` (claims) each restrict the paged set to matching rows only.
- R-19RY-N8JW — `Params.Limit` is clamped: `0` yields `DefaultLimit` rows (when available) and a value above `MaxLimit` yields at most `MaxLimit`.
- R-1C7R-ES1A — `EncodeCursor`/`DecodeCursor` round-trip ordered key parts losslessly, and `DecodeCursor` reports `ok=false` for a malformed token.
- R-1DFN-SJRZ — A list call given an undecodable cursor returns a clean error and does not silently restart from the first page.
