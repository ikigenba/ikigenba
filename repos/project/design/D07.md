# Decision 7 — The MCP tool surface

**Decision.** Structured MCP over the shared `appkit/mcp` transport, the
two-file pattern (`internal/mcp/mcp.go` — `Instructions`, the domain
interface, `NewHandler(svc, rt)`; `internal/mcp/tools.go` — the tool table +
`var Events`). Bare verb names (`toolPrefix == ""`); chassis adds
`health`/`reflection` (never registered here). Every handler threads
`id.OwnerID` from `server.Identity` as the scoping key, and the create paths
(`clone`, `session_start`) also capture `id.OwnerEmail` as the row's display
snapshot; owner is never read from args. Success via
`appkitmcp.StructuredResult`, errors via
`appkitmcp.ErrorResult` with the closed vocabulary; every tool declares
`InputSchema` and `OutputSchema`.

Nine domain tools:

| tool | input | success `structuredContent` | errors |
|---|---|---|---|
| `clone` | `{name}` | `{repo}` (view below) | `validation`, `conflict`, `source_unavailable` |
| `list` | `{}` | `{items: [repo]}` | — |
| `get` | `{name}` | `{repo}` | `not_found` |
| `delete` | `{name}` | `{deleted: true}` | `not_found` |
| `session_start` | `{repo, instructions, branch?}` | `{session}` | `validation`, `not_found` |
| `session_list` | `{repo?}` | `{items: [session]}` | `validation` |
| `session_get` | `{id}` | `{session}` | `not_found` |
| `session_output` | `{id, offset?, limit?}` | `{lines, total, offset}` | `not_found`, `validation` |
| `session_cancel` | `{id}` | `{cancelled: bool, session}` | `not_found` |

Views: `repo` = `{name, clone_url, default_branch, created_at}`; `session` =
`{id, repo, issue_number?, attempt, branch, status, error?, pr_url?,
created_at, started_at?, ended_at?}` — no instructions echo in list (size),
full record in `session_get`. Neither view surfaces an owner field (results
are already the caller's own, owner-scoped), so the suite's owner-id
conversion adds nothing to expose here — its keep-`owner_email`-add-`owner_id`
rule applies only to results that already show an owner.

Semantics binding the other Decisions: `clone` → D4 `CloneRepo` (the caller's
`owner_id`/`owner_email` from `server.Identity` recorded as provisioner);
`delete` → D4 removal / D9 survival; `session_start`
→ D5 `Enqueue` of a manual session (`branch` defaults to
`ikigenba/session-<id>`; a caller-supplied branch must match `ikigenba/*` —
branch-namespace enforcement is runner code in v1); `session_cancel` → D5
`Cancel` (false + current view when already terminal); `session_output`
line-slices `output.jsonl` exactly as prompts' run output read does.
Listing/reading is owner-scoped the way the suite's other services scope:
`list`/`session_list` return the caller's records (`owner_id` match, never
`owner_email`).

**Rejected.**
- **`log`/`status`/`diff` repo-inspection verbs in v1** — they belong to the
  guidance phase of the later direction; v1's read surface is sessions, and
  the PR is the review surface.
- **A `create` verb doing double duty (init-or-clone)** — v1 is clone-only;
  the verb is named for the one thing it does. `create` returns with
  local-only repos in the release work.
- **Free-form branch names on `session_start`** — everything the bot pushes
  stays in the `ikigenba/*` namespace, one rule with no exceptions to audit.

**Verification** (substrate: the assembled `appkit/mcp` handler over
`httptest` with real SQLite and fixture remotes; identity injected as the
transport does).

- R-FN1M-P1FW — `clone`/`list`/`get`/`delete` round-trip: clone returns the
  repo view; a duplicate clone returns code `conflict`; `get` on a missing
  name returns `not_found`; after `delete`, `get` is `not_found` and the
  state dir is gone; `list` shows only the calling owner's repos (scoped by
  `owner_id`).
- R-FO9J-2T6L — `session_start` returns a `queued`/`running` session view
  with branch `ikigenba/session-<id>` by default, rejects a branch outside
  `ikigenba/*` with `validation`, and rejects an unknown repo with
  `not_found`; `session_cancel` on a running session returns
  `cancelled: true` and the session ends `cancelled`; on a terminal session
  it returns `cancelled: false` without altering the row.
- R-FPHF-GKXA — `session_list` filters by repo and owner (`owner_id`);
  `session_get` returns the full record including `error`/`pr_url` when set;
  `session_output` returns the requested line window and correct `total`
  from a real transcript file, and `not_found` for an unknown id.
- R-FQPB-UCNZ — every tool declares both schemas; a malformed-args call
  returns code `validation` **without any side effect** (no row, no state
  dir, no enqueue — asserted against the DB); every error path returns a
  code from the closed vocabulary.
- R-FRX8-84EO — the tool table registers exactly the nine names above (no
  `health`/`reflection` collisions), and through the assembled handler
  `tools/list` exposes 9 + the 2 chassis tools.
- R-IEYB-SNAO — the create paths capture identity from the transport, not
  arguments: a `clone` and a `session_start` driven through the assembled
  handler with injected identity `X-Owner-Id` = X / `X-Owner-Email` = E create
  a row whose stored `owner_id` is X and `owner_email` is E, **even when the
  tool arguments carry a different or bogus owner value** (owner is not an
  accepted argument and is ignored); a subsequent `get`/`session_get` by the
  same X returns the record while a caller presenting a different `owner_id`
  gets `not_found`. (Substrate: the assembled `appkit/mcp` handler over real
  SQLite with identity injected as the transport does; the stored row read
  back from the DB.)
