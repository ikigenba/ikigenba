# Decision 4 — Version production: `bump` / `ship` emit `v`-prefixed SemVer + the release bundle

**Decision.** The version *producers* are brought in line with the `v`-everywhere,
full-SemVer-2.0 contract (D03), and `ship` now produces the **versioned bundle**
(D02), not a bare binary.

- **`bin/bump`** reads `<svc>/VERSION`, increments the requested field
  (`major|minor|patch`) **carrying the `v`**, and writes the `v`-prefixed result
  back (still a path-limited commit of only that file, still on `main`). It
  **rejects** a `VERSION` file that is not valid `v`-prefixed SemVer rather than
  silently coping (the old "bare X.Y.Z" contract is gone). The pre-release
  identifier, when one is set, is preserved/advanced per SemVer.
- **`bin/ship`** builds `main` static for `linux/amd64` (`GOWORK=off`), stamps the
  binary with the **full** version (the `VERSION` core plus a `+<SHA>`
  build-metadata suffix from the built commit, via `-ldflags`), then **assembles a
  bundle** and `scp`s the bundle (not a bare binary) to the box `/tmp`.

**The release bundle.** The unit of delivery is one `tar.gz`, **named with the
full version**, carrying every shipped tier together:

```
<svc>-v<full>.tar.gz
  <svc>          the static binary
  nginx.conf     the service's nginx location fragment
  manifest.env   the service's authored, portable manifest (D11; shipped verbatim)
  share/...      shipped read-only resources (absent if the service ships none)
```

**Bundle naming schema.** The filename is exactly:

```
<svc>-<version>.tar.gz
```

where `<svc>` is the canonical service name and `<version>` is the **full
v-prefixed SemVer 2.0 identity** (D03) — `vMAJOR.MINOR.PATCH[-prerelease]+<sha>`,
the same string the binary stamps and `libexec/<svc>-<version>` carries. Examples:

```
crm-v0.7.1+a1b2c3d.tar.gz
wiki-v0.7.1-rc.2+a1b2c3d.tar.gz
```

The `+<sha>` build metadata is always present in a shipped bundle (`ship` always
stamps the built commit). The same filename is the box `/tmp` staging name and the
`opsctl stage --artifact <path>` argument (D10). `<svc>` and `<version>` are passed
to `stage` explicitly, so the name is **constructed and validated**, never parsed
back apart.

`stage` (D10) unpacks it onto the box as `libexec/<svc>-v<full>`,
`etc/<v>/nginx.conf`, `etc/<v>/manifest.env`, and `share/<v>/…` (D02). The service
**authors its shipped tiers in source mirroring the install tree** —
`<svc>/etc/nginx.conf`, `<svc>/etc/manifest.env` (D11), and an optional
`<svc>/share/…` — and `ship` packages exactly those plus the built binary. Source
layout == install layout; a service with no `share/` ships no `share/` entry. The
`manifest.env` is **authored and portable** (no box paths; D11) and is packaged
**verbatim** — `ship` does not generate, stamp, or rewrite it.

The version embedded in the binary, the bundle filename, the bundle's binary, and
opsctl's deploy preflight (D03 gate) all carry the same full `v`-prefixed string,
so `<svc> version`, the staged `libexec/` filename, and the stage/deploy preflight
agree by construction.

**Rejected.**
- *Ship a bare binary and apply config/resources out of band.* The gap this work
  closes: a fragment change then never rides the version and cannot roll back with
  the binary. The bundle carries all shipped tiers as one versioned unit. Rejected.
- *Fix only the consumer side (opsctl) and leave `bump`/`ship` emitting bare
  numbers.* opsctl's `v`-everywhere gate would then reject the suite's own
  artifacts — "any legal SemVer" is a constraint on **both** ends. Rejected.
- *Carry `-rel`/`+SHA` as out-of-band metadata rather than in the version string.*
  Defeats a single canonical identity the filesystem and `version` verb share.
  Rejected.
- *Author the fragment/resources somewhere other than the service source tree
  (e.g. a central opsctl dir).* Splits a service across two repos' worth of intent
  and breaks "the service owns its shipped tiers." Rejected for `<svc>/etc/` +
  `<svc>/share/`.

**Verification.**
- R-44HU-2IOD — `bump patch` on a `VERSION` of `v0.7.1` writes `v0.7.2` (keeps the
  `v`); `bump` against a bare `0.7.1` file **fails** with a non-zero exit (rejects
  the old form). Substrate: `bin/bump.test.sh` (shell, real git repo fixture).
- R-45PQ-GAF2 — `ship` stamps the full version: the built binary's `version` output
  equals `v<core>+<sha>` for the built commit (the `+<sha>` matches the source
  commit, not a placeholder), and the produced bundle is named `<svc>-v<core>+<sha>.tar.gz`
  with that same string. Substrate: `bin/ship.test.sh` (shell) asserting the
  stamped string + bundle name; the embedded-version round-trip is a build smoke
  against a real `go build`.
- R-P4CO-FY2L — `ship`'s bundle is a `tar.gz` containing the built binary as
  `<svc>`, the service's `<svc>/etc/nginx.conf` as `nginx.conf`, the service's
  `<svc>/etc/manifest.env` as `manifest.env` **byte-for-byte** (packaged verbatim,
  not generated/stamped), and the contents of `<svc>/share/` under `share/` (and
  **no** `share/` entry when the service has no `<svc>/share/`). Substrate:
  `bin/ship.test.sh` (shell) listing/extracting the produced archive against a
  fixture service tree.
