# Decision 10 — stage / deploy / rollback / prune orchestration

**Decision.** opsctl owns the on-box lifecycle as four verbs over the D02 layout
and the D07 S3 store. Off-box `ship` only delivers the bundle to `/tmp` and prints
the two box commands (`stage`, then `deploy`); every state change is on-box.

**`opsctl stage <svc> v<ver> --artifact /tmp/<svc>-v<full>.tar.gz`** — make a
version's tiers present, change nothing live:

1. **Version gate** (D03) — refuse a non-`v`/partial/invalid version before any work.
2. **SHA-collision guard** — if `libexec/<svc>-v<full>` already exists, refuse unless
   the artifact's bytes are identical to what is staged; a *differing* re-stage of an
   already-present version fails loudly unless `--force` (a forgotten `bump`/rebuild
   must not silently overwrite a deployed binary).
3. **Unpack** the bundle into the versioned tiers (D02): `libexec/<svc>-v<full>`,
   `etc/<v>/nginx.conf`, `etc/<v>/manifest.env`, `share/<v>/…` — placed **root-owned**
   (D01).
4. **Delete the `/tmp` artifact** once unpacked.

`stage` never swaps a symlink, migrates, or restarts — a staged version is inert
until `deploy`.

**`opsctl deploy <svc> v<ver>`** — promote a staged version, in this fixed order:

1. **Back up first, UNCONDITIONALLY** — an `opsctl backup <svc>` S3 snapshot (D07)
   on **every** deploy, before any mutation. (This replaces the old "back up only if
   the schema advances" conditional: the rollback target must always exist, so the
   backup is never skipped.)
2. **Migrate** (`<bin> migrate`, forward-only; D11 verb), then `chown` the `state/`
   tree back to `<svc>:<svc>` (the root-run migrate may create root-owned DB files).
3. **Atomic swap** all three symlinks together to `<new>` (D02): `bin/run`,
   `etc/current`, `share/current`.
4. **nginx reload** — the stable system symlink already resolves through
   `etc/current` (D01), so a reload picks up the new fragment; no system file is
   rewritten.
5. **Restart** the unit and confirm `is-active`.
6. **Prune** old versions (D02).

**There is no manifest-regeneration step.** The old deploy step that ran
`<bin> manifest` and stamped box paths is **gone** — the manifest is the authored,
shipped `etc/current/manifest.env` (D11), and the box paths are composed from
`IKIGENBA_ROOT` (D11), so deploy neither writes nor generates a manifest.

**`opsctl rollback <svc> [-N]`** — recover by going back to a previous backup. `-N`
selects a backup **by recency** over the service's S3 snapshots (D07), newest-first:
`-0` = most recent, `-1` = the next older, …; **`rollback <svc>` with no flag means
`-0`**.

1. List the service's snapshots newest-first; resolve the `(N+1)`th.
2. Read the **producing version embedded in that snapshot's key** (D07).
3. The binary for that version **must still be retained** in `libexec/` — if it was
   pruned, **fail loudly** (do not silently roll to a different version).
4. **Restore** that snapshot's `state/` (D07 restore: stop · replace `state/` · reset
   non-state) and **swap the three symlinks** to that recorded version.
5. **nginx reload · restart · is-active.**

A **delayed rollback** — where deploys have since advanced the schema past what the
old binary carries — **fails loudly** via the forward-only downgrade guard (the old
binary refuses to serve a DB whose applied schema is ahead of its embedded schema),
rather than booting against an incompatible DB.

Nightly snapshots (D09) **count** in the `-N` index: it is a *backup* reference, not
a *deploy* reference.

**Testability seam.** All four verbs run **in-gate, unprivileged**: a real
filesystem under a temp `OPSCTL_ROOT`/`SysRoot` for the symlink swaps and unpack;
the `System` seam (stubbed) for systemctl stop/start/restart/is-active and nginx
reload; the `AppRunner` seam (stubbed) for `<bin> migrate`/`schema`; the
`ObjectStore` `fakeStore` (D07) for backup/restore. That `aws` moves real bytes and
that nginx actually reloads through the system symlink are **on-box** checks
outside the green gate.

**Rejected.**
- *Back up only when the schema advances (the status quo).* Then a non-schema deploy
  has no rollback target and a rollback would restore a stale or missing snapshot.
  Every deploy backs up. Rejected.
- *`-N` selects by deploy history / a release ledger.* A second source of truth to
  keep in sync with S3; the snapshots already form the recovery index, and their
  keys carry the version. `-N` reads S3 directly. Rejected.
- *Roll to the newest retained binary when a snapshot's version was pruned.* Silently
  changes which code runs against a restored DB. Fail loudly instead. Rejected.
- *Let `stage` overwrite a present version silently.* Hides a forgotten `bump`. The
  SHA-collision guard refuses unless bytes match or `--force`. Rejected.

**Verification.**
- R-84VR-7U2K — `stage` (a) refuses an invalid/partial version (D03 gate, non-zero
  exit, nothing unpacked); (b) with `libexec/<svc>-v<full>` already present and a
  **differing** artifact, refuses unless `--force`; (c) on success unpacks all four
  tiers (`libexec/<svc>-v<full>`, `etc/<v>/nginx.conf`, `etc/<v>/manifest.env`,
  `share/<v>/`) **and deletes the `/tmp` artifact**. Substrate: opsctl unit test,
  real `tar`/filesystem under a temp `OPSCTL_ROOT`.
- R-863N-LLT9 — `deploy` takes an S3 backup **unconditionally and before any
  mutation**: with a stubbed `System`/`AppRunner` and a `fakeStore`, a deploy whose
  schema does **not** advance still issues exactly one backup `Put`, and it is
  ordered before the migrate/swap calls. Substrate: opsctl unit test (seam call
  order + fake store).
- R-87BJ-ZDJY — `deploy` performs its steps in order — backup → migrate → atomic
  swap of all three symlinks → nginx reload → restart/is-active → prune — and
  performs **no** manifest-regeneration step (it never runs a `manifest` verb and
  never writes `etc/current/manifest.env`). Substrate: opsctl unit test asserting the
  recorded seam-call sequence and the absence of any manifest write.
- R-88JG-D5AN — `rollback <svc>` with **no** `-N` resolves to `-0`, the most recent
  snapshot (identical target to an explicit `-0`). Substrate: opsctl unit test (fake
  store seeded with several snapshots; assert the resolved key).
- R-89RC-QX1C — `rollback <svc> -N` resolves the `(N+1)`th-newest snapshot, restores
  its `state/`, and swaps **all three** symlinks to the **version embedded in that
  snapshot's key** (D07) — not to the current or newest version. Substrate: opsctl
  unit test, fake store + real filesystem (assert restored bytes + the three symlink
  targets).
- R-8AZ9-4OS1 — `rollback` to a snapshot whose embedded version is **no longer
  retained** in `libexec/` **fails loudly** (non-zero exit, no symlink swapped, live
  release untouched) rather than rolling to a different version. Substrate: opsctl
  unit test (fake store referencing a pruned version).
- R-8C75-IGIQ — a **delayed** rollback where the live DB's applied schema is ahead of
  the rolled-to binary's embedded schema **fails loudly** via the forward-only
  downgrade guard (the binary refuses to serve), surfaced as a failed rollback rather
  than a boot against an incompatible DB. Substrate: opsctl unit test with a stubbed
  `AppRunner`/`System` reporting the downgrade-guard failure (applied > embedded).
