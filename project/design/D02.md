# Decision 2 — Versioned release bundle: `libexec/` binary + `etc/<v>`/`share/<v>` + symlink swap

**Decision.** A release is a **versioned bundle** (the `tar.gz` D04 produces), not a
bare binary. Its three shipped tiers are placed on the box **by version** and the
live version is selected entirely by **symlinks** — the same atomic-`rename(2)`
cutover `bin/run` already uses, applied uniformly to every tier:

```
/opt/<svc>/
  bin/run            → ../libexec/<svc>-v<full>       (the binary; D02 swap point)
  libexec/
    <svc>-v<full>                                     (binary — a bare FILE; FHS /usr/libexec)
  etc/
    current          → <v>                            (active config symlink)
    <v>/nginx.conf                                    (per-version shipped fragment, retained)
    <v>/manifest.env                                  (per-version shipped authored manifest, retained; D11)
  share/
    current          → <v>                            (active resources symlink)
    <v>/...                                           (per-version shipped resources, retained)
```

- **`libexec/` holds only binary files**, one per version, carrying the full
  v-prefixed SemVer identity (D03). No directories, no bundle contents — strictly
  FHS `/usr/libexec`.
- **`etc/<v>/` and `share/<v>/`** are the per-version retained homes for the
  bundle's other shipped tiers. `stage` (D10 orchestration) unpacks the `tar.gz`
  so that, for version `v`, `libexec/<svc>-v<full>`, `etc/<v>/nginx.conf`,
  `etc/<v>/manifest.env`, and `share/<v>/…` all exist. The two `etc/<v>/` files —
  the nginx fragment and the **authored, portable** `manifest.env` (D11) — are
  shipped verbatim; nothing on the box generates or stamps them.
- **Three active-selection symlinks**, all swapped by `atomicSwap` (`rename(2)`
  over the symlink — atomic, no dangling window): `bin/run → libexec/<svc>-v<full>`,
  `etc/current → <v>`, `share/current → <v>`. A deploy repoints all three to the
  new version; a rollback repoints all three back to the target version. Because
  they key off the **same** `<v>`, the binary and its matching config (`etc/current/
  nginx.conf`, `etc/current/manifest.env`) and resources (`share/current/…`) move
  as one unit — config and resources cannot desync from the binary.
- **System nginx path.** `/etc/nginx/conf.d/locations/<svc>.conf` is a **stable**
  symlink → `/opt/<svc>/etc/current/nginx.conf`, created once by `setup` (D01). It
  follows the per-service `etc/current` swap automatically, so deploy/rollback only
  need an nginx **reload** (the reload + ordering is D10), never to rewrite a system
  file.
- **`etc/current` is also where the launcher sources the manifest.** The systemd
  unit's launcher reads `/opt/<svc>/etc/current/manifest.env` (D11), so a swap
  repoints the served config and the launcher's env source together.
- **`share/current`** is the stable read path the service uses
  (`/opt/<svc>/share/current/…`).

This **replaces** the old two-level `bin/run → current → releases/<v>/<svc>`
indirection: `releases/` (dir-per-version) and the intermediate `current` symlink
are gone. The external contract is unchanged — the systemd unit still execs
`/usr/local/bin/ikigenba-launch <svc>`, which execs the stable path
`/opt/<svc>/bin/run`; only what the symlinks *point at* moves.

**Prune** keeps the newest N versions by D03 precedence and deletes each older
version as a **set** — its `libexec/<svc>-v*` file **and** its `etc/<v>/` (both the
nginx fragment and the manifest) **and** its `share/<v>/` — never deleting the live
or the rollback target's set.

opsctl `Layout` grows per-version + active accessors (illustrative):

```go
func (l Layout) LibexecBinary(v string) string   // /opt/<svc>/libexec/<svc>-<v>   (full v-prefixed SemVer)
func (l Layout) RunLink() string                 // /opt/<svc>/bin/run
func (l Layout) EtcVersionDir(v string) string   // /opt/<svc>/etc/<v>
func (l Layout) NginxConfFile(v string) string   // /opt/<svc>/etc/<v>/nginx.conf
func (l Layout) ManifestFile(v string) string    // /opt/<svc>/etc/<v>/manifest.env
func (l Layout) EtcCurrentLink() string          // /opt/<svc>/etc/current          (→ <v>)
func (l Layout) ActiveNginxConf() string         // /opt/<svc>/etc/current/nginx.conf
func (l Layout) ActiveManifest() string          // /opt/<svc>/etc/current/manifest.env
func (l Layout) ShareVersionDir(v string) string // /opt/<svc>/share/<v>
func (l Layout) ShareCurrentLink() string        // /opt/<svc>/share/current        (→ <v>)
```

**Rejected.**
- *Keep `releases/<v>/` dir + `current` flip.* The old dir-per-release existed to
  swap a multi-file release atomically as a set; we now swap a *set of versioned
  paths* by repointing three symlinks, which is the same atomic primitive without a
  parent indirection dir. Rejected.
- *Per-file `etc/` symlinks (`etc/nginx.conf → <v>/nginx.conf`, plus a separate
  `manifest.env`).* As `etc/<v>/` grew a second shipped file (the manifest), N
  per-file symlinks would have to be swapped in lockstep and could partially apply.
  One `etc/current → <v>` symlink swaps the whole config tier atomically and mirrors
  `share/current`. Rejected for the single directory symlink.
- *One bundle dir per version under `libexec/<svc>-v<full>/`.* Violates FHS —
  `libexec` is for binaries, not config/resources. The whole point of mirroring FHS
  is that the tier fixes meaning/ownership; collapsing the bundle into `libexec`
  throws that away. Rejected — `libexec` holds only the binary file.
- *Select the active `share/`/`etc/` via a `manifest.env` stamp.* Introduces a
  second selection mechanism alongside the symlink swap; one consistent mechanism
  (symlinks, like `bin/run`) is simpler and atomic. Rejected.
- *`bin/run` as a wrapper script.* Encodes the live version in a script body or a
  pointer file (re-introducing indirection) and duplicates `ikigenba-launch`.
  Rejected for a plain symlink.

**Verification.**
- R-3TIQ-ML04 — after deploy of `v<new>`, `bin/run` resolves to
  `libexec/<svc>-v<new>` and that file is the just-placed binary; legacy
  `releases/` and `current` do not exist. Substrate: real filesystem (temp
  `OPSCTL_ROOT`).
- R-1BF5-X7QS — after `stage` of a `v<full>` bundle, all per-version paths exist
  for that version: `libexec/<svc>-v<full>` (the binary), `etc/<v>/nginx.conf`,
  `etc/<v>/manifest.env`, and `share/<v>/` (with the bundle's resources). Substrate:
  real filesystem + real `tar` (unpack a real archive into a temp `OPSCTL_ROOT`).
- R-1A79-JG03 — after a deploy of `v<new>`, **all three** active symlinks resolve
  to the **same** version `<new>`: `bin/run → libexec/<svc>-v<new>`,
  `etc/current → <new>` (so `etc/current/nginx.conf` and `etc/current/manifest.env`
  are `<new>`'s), `share/current → <new>` (binary, config, and resources are never
  left on mismatched versions). Substrate: real filesystem.
- R-3UQN-0CQT — rollback from `v<new>` repoints all three symlinks
  (`bin/run`, `etc/current`, `share/current`) back to `<old>`, and `<old>`'s set is
  all still present (rollback never deleted it): `libexec/<svc>-v<old>`,
  `etc/<old>/{nginx.conf,manifest.env}`, and `share/<old>/`. Substrate: real
  filesystem.
- R-3VYJ-E4HI — each active symlink resolves to an existing target at **every**
  point of a deploy swap (assert before, and immediately after, each atomic
  repoint — there is no observable state where `bin/run`, `etc/current`, or
  `share/current` dangles). Substrate: real filesystem.
- R-1CN2-AZHH — prune keeps the newest N by D03 precedence and deletes each older
  version as a set — its `libexec/<svc>-v*` **and** `etc/<v>/` (nginx fragment +
  manifest) **and** `share/<v>/` — and never deletes the live or rollback target's
  set (no orphaned `etc/<v>`/`share/<v>` left behind, and no live tier removed).
  Substrate: real filesystem (temp `OPSCTL_ROOT` seeded with N+ versions).
