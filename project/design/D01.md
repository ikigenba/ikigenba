# Decision 1 — The `/opt/<svc>/` install tree

**Decision.** Every service is a self-contained subtree under `/opt/<svc>/`
holding *all* of its on-box parts, mirroring the Linux FHS so each tier's location
fixes its meaning, ownership, and lifecycle. The canonical paths, owned by
opsctl's `Layout` type (rooted at `OPSCTL_ROOT`, default `/opt`):

```
/opt/<svc>/                              0755 <svc>:<svc>
  bin/run            → ../libexec/<svc>-v<full>   (D02: the atomic swap point)
  libexec/                               root-owned   (FHS /usr/libexec — binaries only)
    <svc>-v<full>                                     (D02: versioned binary FILES)
  etc/                                   root-owned   (FHS /etc — config)
    current          → <v>                            (D02: active config symlink)
    <v>/nginx.conf                                    (D02: per-version shipped fragment, retained)
    <v>/manifest.env                                  (D11: per-version shipped authored manifest, retained)
  share/                                 root-owned   (FHS /usr/share — shipped read-only resources)
    current          → <v>                            (D02: active resources symlink)
    <v>/...                                           (D02: per-version shipped resources, retained)
  state/                                 0711 <svc>:<svc>   (D05: backed up + restored; TRAVERSE-only)
    <svc>.db (+ -wal/-shm)               0640 <svc>:<svc>   (private — never nginx-readable)
    www/                                 0750 <svc>:web     (served content; nginx reads via the `web` group)
      public/                            0750 <svc>:web     (served directly, no auth)
      private/                           0750 <svc>:web     (served behind the introspection auth gate)
  cache/                                 0750 <svc>:<svc>   (D05: NOT backed up; reset on restore)
    <svc>.db.generation                                    (event-plane epoch sidecar)
```

**There is no `backups/` tier.** Backups are off-box only (D07); nothing retained
on the box is a backup of record, so the old `/var/backups`-style local tier is
gone. The flat `data/` dir is also gone (split into `state/` + `cache/` by D05),
as are `releases/` + `current` (replaced by `libexec/` + the symlinks, D02). The
apex TLS cert (D07) remains the only artifact handled outside this tree.

**Shipped tiers are root-owned.** `libexec/`, `etc/`, and `share/` carry the
shipped artifact and are only ever *read* by the running service, so they are
**root-owned**, with the per-version subdirs (`etc/<v>/`, `share/<v>/`) and binary
files placed root-owned at stage time (D02). Dirs `0755`, files `0644` — readable
by the `<svc>` user, writable only by root (deploy/stage run as root). Local
(`state/`, `cache/`) data is app-owned, exactly as D05 requires.

**The nginx fragment is shipped, not generated.** Each service authors its own
nginx location fragment in its source tree; `ship` packages it into the bundle as
`etc/<v>/nginx.conf` (D02/D04). `setup` no longer synthesizes it. `setup` instead
creates a **stable system symlink** `/etc/nginx/conf.d/locations/<svc>.conf →
/opt/<svc>/etc/current/nginx.conf`, set once and never rewritten; because
`etc/current` is itself the per-service active-version symlink (D02), a deploy
or rollback that repoints it changes what the system path resolves to, so deploy
needs only an nginx **reload** (D10), never a system-file rewrite.

**Served content & the `web` group.** (Unchanged from the settled model.) Any
service may have `state/www/{public,private}` — the universal version of `sites`'
`served/{public,private}`. nginx must read it without ever reading the DB:

- `state/` is **`0711` traverse-only** — nginx passes *through* it to reach `www/`
  but cannot list it; the DB at **`0640 <svc>:<svc>`** is unreadable by any
  non-owner. **The DB's privacy rests on `0711` + `0640`, not on the group.**
- A shared **`web` group** (members: `<svc>` and `www-data`) owns `state/www/**`
  at `0750 <svc>:web`, so nginx reads served content but the rest of the box does
  not. `setup` creates the group and applies ownership.
- **Default-private, then selectively open `www/`:** `chown -R <svc>:<svc> state/`
  → `chmod 0711 state/` → re-open `state/www/` to `<svc>:web`. A skipped re-open
  yields **403 on `www/`** (loud, fail-safe), never a leaked DB.
- **public vs private is an nginx concern, not a filesystem one:** both are
  `0750 <svc>:web` on disk; the shipped fragment serves `public/` directly and
  gates `private/` behind the same `/srv/<svc>/` introspection the proxy routes
  use.

`Layout` accessors (illustrative; per-version accessors are D02):

```go
func (l Layout) StateDir() string        // /opt/<svc>/state
func (l Layout) DBPath() string          // /opt/<svc>/state/<svc>.db
func (l Layout) WWWDir() string          // /opt/<svc>/state/www
func (l Layout) WWWPublicDir() string    // /opt/<svc>/state/www/public
func (l Layout) WWWPrivateDir() string   // /opt/<svc>/state/www/private
func (l Layout) CacheDir() string        // /opt/<svc>/cache
func (l Layout) GenerationPath() string  // /opt/<svc>/cache/<svc>.db.generation
func (l Layout) EtcDir() string          // /opt/<svc>/etc
func (l Layout) ShareDir() string        // /opt/<svc>/share
// REMOVED: BackupsDir, ReleasesDir, ReleaseDir, ReleaseBinary, CurrentLink,
//          CurrentBinary, DataDir, WWWServedDir
```

This is primarily a **structural** decision; its behavior is proven by the
Decisions it enables (D02 swap/placement, D05 backup boundary). The behavior it
owns is that `setup` materializes the tree with the right ownership/permissions so
a service can run, the backup boundary holds (no on-box backup tier exists), and
nginx can serve `state/www/` **without** the DB ever being readable by `www-data`.
`setup` applies ownership through an `Owner` seam (real impl `os.Chown`; production
`setup` runs as root) so the ownership plan is asserted **in-gate without
privilege**, while mode bits are set and asserted on a real temp tree.

**Rejected.**
- *Keep the flat `data/` dir.* Conflates backed-up state with disposable runtime
  data — the exact boundary the effort makes explicit (D05). Rejected.
- *Keep a local `backups/` tier.* A backup on the same disk as the data is not a
  backup (D07); nothing on the box is a backup of record. Rejected.
- *Generate the nginx fragment in `setup`.* Then a fragment change cannot ride the
  versioned bundle and cannot roll back with the binary — the gap this work closes.
  Rejected for shipping it in the bundle.
- *Mirror the full FHS (`var/lib`, `var/cache`, `run`, …) literally.* The `/var`
  semantics are worth borrowing (D05) but dragging in unused path depth is
  cargo-culting; `state/`/`cache/` carry the meaning without it. Rejected.

**Verification.**
- R-3SAU-8T9F — `setup` against a temp `OPSCTL_ROOT` creates `state/`, `cache/`,
  `libexec/`, `bin/`, `etc/`, `share/` under `/opt/<svc>/` with the right **mode
  bits** — `state/` `0711` (traverse-only — not `0755`, not group/other readable),
  DB `0640`, `state/www/{public,private}` `0750`, `cache/` present and writable,
  `etc/` and `share/` present and root-owned — **and** it creates **no** `backups/`
  directory, **and** it requests ownership `<svc>:<svc>` for `state/`+DB and
  `<svc>:web` for `state/www/**` via the `Owner` seam (asserted on the recorded
  calls). Substrate: real filesystem (temp dir) for the mode bits + stubbed `Owner`
  for the ownership plan — runs unprivileged.
- R-LHY1-6IS8 — `setup` creates the stable system symlink
  `/etc/nginx/conf.d/locations/<svc>.conf` → `/opt/<svc>/etc/current/nginx.conf` (a
  symlink, not a copied file), so that swapping `etc/current` (D02) changes what
  the system path resolves to without rewriting it. Substrate: real filesystem
  under a temp `SysRoot` (symlink target asserted), unprivileged.
- R-VCF3-PLWD — the service's **shipped** nginx fragment (`etc/<v>/nginx.conf`, as
  packaged in the bundle) serves `state/www/public` directly (no `auth_request`)
  and serves `state/www/private` **behind** the `/srv/<svc>/` introspection
  `auth_request` — the private location carries the introspection gate, the public
  one does not. Substrate: structural assertion on the shipped fragment
  (byte/field), matching the suite's existing `nginx_test.go` convention. (The
  through-running-nginx request check is an on-box/manual step, outside the green
  gate.)

One earlier behavior — the access-control model holding by construction (a
`web`-group process can read `state/www/{public,private}` and cannot read
`state/<svc>.db` or list `state/`) — lost its id-tagged test when the
served-tree ownership logic was replaced (the `web`-group model rework). It
remains a normative statement of the model, carried without an id; the mode
bits and ownership plan are asserted piecewise by R-3SAU-8T9F, and the live
`www-data`-gid read is exercised on-box, outside the green gate.
