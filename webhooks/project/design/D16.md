# Decision 16 — Structured MCP adoption (structuredContent, output schemas, closed error vocabulary)

*Cites the suite contract `docs/structured-mcp-design.md` (single verb surface;
`structuredContent` + mirrored text; declared `outputSchema`; the shared
closed-vocabulary error codes). This Decision states webhooks's conformance to
that contract; it does not restate it.*

## Decision.

webhooks's four domain tools (`create`, `list`, `delete`, `rotate`, D6) adopt
the appkit `2025-06-18` result contract. The refactor **formalizes the JSON the
tools already emit** — it never reshapes a success payload. Three changes, all
inside `internal/mcp/tools.go`:

1. **Success results become `StructuredResult`.** Every `appkitmcp.JSONResult(v)`
   call site (the four domain verbs) becomes `appkitmcp.StructuredResult(v)`.
   The helper emits `structuredContent: v` **plus** a text content block holding
   the identical JSON — machines read `structuredContent`, agents read the text.
   `StructuredResult` now returns `(map[string]any, error)` (marshal failure is
   surfaced), so each handler propagates that error honestly rather than
   discarding it:

   ```go
   // internal/mcp — success path, per verb.
   func (h *toolHandlers) toolCreate(ctx context.Context, raw json.RawMessage, id server.Identity) (map[string]any, error) {
       // ... h.svc.Create(...) → out := h.webhookView(wh); out["secret"] = secret
       return appkitmcp.StructuredResult(out) // returns (result, err); err propagated, not swallowed
   }
   ```

   The emitted value per verb is **byte-for-byte what D6 already specifies**:
   `create` → `{name, trigger_url, created_at, last_triggered_at, secret}`;
   `list` → `{items: [{name, trigger_url, created_at, last_triggered_at}]}`
   (the secret-free `webhookView`); `delete` → `{deleted: true}`;
   `rotate` → `{name, trigger_url, secret}`.

2. **Each domain tool declares an `outputSchema`.** A hand-authored
   `map[string]any` JSON-schema literal (`type`/`properties`/`required`) is set
   on each `appkitmcp.Tool`, mirroring the emitted JSON verbatim, and surfaced by
   `tools/list`. Following the `appkit/mcp` house style (`objectSchema`), the
   tool table adds a tiny private helper beside the existing `obj`/`typ`
   descriptor helpers:

   ```go
   // internal/mcp — output-schema literals (mirror the emitted JSON exactly).
   func outObj(props map[string]any, required ...string) map[string]any {
       o := map[string]any{"type": "object", "properties": props}
       if len(required) > 0 {
           o["required"] = required
       }
       return o
   }

   var createOutputSchema = outObj(map[string]any{
       "name":              typ("string"),
       "trigger_url":       typ("string"),
       "created_at":        typ("string"),
       "last_triggered_at": map[string]any{"type": []string{"string", "null"}},
       "secret":            typ("string"),
   }, "name", "trigger_url", "created_at", "last_triggered_at", "secret")
   // list → {items: [webhookView]}; delete → {deleted: bool}; rotate →
   // {name, trigger_url, secret}. Each Tool sets OutputSchema: <literal>.
   ```

   webhooks exposes **no prose tool** — it has no `guide`/`describe` and no
   raw-content read; all four service-owned tools are structured and every one
   declares an `outputSchema`. (The chassis `health`/`reflection` declare their
   own schemas centrally — webhooks does nothing for them.) There is therefore
   **no prose exception** in webhooks's surface.

3. **Errors carry the closed-vocabulary code.** The old hand-built
   `errorEnvelope`/`toolErr` pair (which marshalled `{error:{code,message,field}}`
   into a string and passed it to the deleted `ErrorResult(string)`) is replaced
   by the re-signed `appkitmcp.ErrorResult(code appkitmcp.ErrorCode, msg string)`,
   which sets `isError: true` and `structuredContent: {code, message}`. The
   domain-sentinel → code mapping (owned by D6) is:

   | sentinel | appkit `ErrorCode` |
   |---|---|
   | `webhooks.ErrNameTaken` (name collision) | `appkitmcp.ErrConflict` (`"conflict"`) |
   | `webhooks.ErrInvalidName` | `appkitmcp.ErrValidation` (`"validation"`) |
   | `webhooks.ErrNotFound` (unknown or not-owned) | `appkitmcp.ErrNotFound` (`"not_found"`) |
   | any other error | `appkitmcp.ErrInternal` (`"internal"`, opaque message) |

   The name-collision code **changes value**: the retired hand-built envelope
   used `"duplicate"`, which is **not** in the suite's closed vocabulary; the
   correct closed-vocabulary code for a uniqueness collision is `conflict`.
   webhooks's tool errors draw only from `{validation, not_found, conflict,
   internal}`; `too_large` and `source_unavailable` do not arise on this surface
   (the 1 MiB payload cap is a `413` on the **public HTTP ingress** (D4), not an
   MCP tool result).

   ```go
   func toolErr(err error) map[string]any {
       switch {
       case errors.Is(err, webhooks.ErrNameTaken):
           return appkitmcp.ErrorResult(appkitmcp.ErrConflict, err.Error())
       case errors.Is(err, webhooks.ErrInvalidName):
           return appkitmcp.ErrorResult(appkitmcp.ErrValidation, err.Error())
       case errors.Is(err, webhooks.ErrNotFound):
           return appkitmcp.ErrorResult(appkitmcp.ErrNotFound, err.Error())
       default:
           return appkitmcp.ErrorResult(appkitmcp.ErrInternal, "internal error")
       }
   }
   ```

The identity/guard model is unchanged. The gated `POST /mcp` route keeps
`rt.RequireIdentity` (D1); the **public** `POST /in/<name>` ingress (D4) is not a
loopback guard — it rejects `X-Owner-Id`/`X-Owner-Email`/`X-Client-Id` because their presence
means a misrouted internal request and deliberately tolerates
`X-Forwarded-Proto` — so it does **not** adopt appkit's `LoopbackOnly`
(`X-Forwarded-Proto`-keyed) guard and is untouched here. The chassis `/feed`
mount is already wrapped by appkit. webhooks therefore has **no loopback guard
site to swap**.

## Rejected.

- **Keeping the `{error:{code,message,field}}` envelope (and `field:"name"`).**
  The suite's structured-error shape is `{code, message}` from
  `appkitmcp.ErrorResult`; a webhooks-specific wrapper with an extra `field` key
  would fork the shared vocabulary a machine caller branches on. The validation
  message text still names the offending argument, so the self-correction signal
  survives without a bespoke field. Dropped.
- **Retaining `duplicate` as the collision code.** It is outside the closed
  vocabulary; a machine caller cannot branch on a code the contract does not
  define. Mapped to `conflict`.
- **Runtime schema validation of results.** The suite contract explicitly leaves
  response↔schema conformance to the service's own tests, not the transport;
  webhooks proves it in the test suite (below), not at request time.
- **Declaring an `outputSchema` for a "prose" tool.** webhooks has none; every
  tool it owns returns structured data.

## Verification.

(All via the real assembled `appkit/mcp` handler — `mcp.NewHandler(svc, rt)` over
a real-SQLite `Service` — driven over `httptest` with JSON-RPC `tools/list` /
`tools/call` and the authenticated identity supplied as `X-Owner-Id` (plus
`X-Owner-Email` where a snapshot is asserted), the substrate D6/D12 already use. The service's own HTTP/MCP layer is the real
substrate per the suite contract; no live third-party call is involved.)

- R-DRUS-R3AP — `tools/list` returns a **non-nil `outputSchema`** for every
  webhooks-declared domain tool (`create`, `list`, `delete`, `rotate`); table
  driven over the four names, each asserted present. (Falsifiable: a tool left
  with a nil `OutputSchema` — the prose-opt-out — is omitted from `tools/list`
  and fails the assertion. webhooks declares no prose tool, so none may be
  missing.)
- R-DT2P-4V1E — each domain tool's success result is a `StructuredResult`: it
  carries a `structuredContent` object **and** a mirrored `text` content block
  whose parsed JSON equals `structuredContent`; no domain success returns a
  bare text-only result (`structuredContent` absent). Table driven over the four
  verbs. (Falsifiable: a residual `TextResult`/`JSONResult`-only success — no
  `structuredContent`, or a text block that does not round-trip to the same
  object — fails.)
- R-DUAL-IMS3 — each domain tool's success `structuredContent` **conforms to
  that tool's declared `outputSchema`**: `create` has exactly
  `{name, trigger_url, created_at, last_triggered_at, secret}` with `secret`
  present; `list` is `{items: [...]}` where each item has
  `{name, trigger_url, created_at, last_triggered_at}` and **no** `secret`/
  `secret_hash`; `delete` is `{deleted: true}`; `rotate` is
  `{name, trigger_url, secret}`. The schema mirrors the emitted JSON verbatim.
  (Falsifiable: a schema that names a key the payload omits, or omits a key the
  payload emits — e.g. dropping `secret` from `create` — fails.)
- R-DVIH-WEIS — a `create` for a name already in use returns `isError: true`
  with `structuredContent.code == "conflict"` — **not** the retired legacy
  `"duplicate"` and not any other code. (Falsifiable against a two-`create`
  same-name setup on real SQLite; pins the changed collision code value.)
- R-DWQE-A69H — every domain-error result sets `isError: true` and carries
  `structuredContent.code` drawn from appkit's closed `ErrorCode` set
  (`validation`, `not_found`, `conflict`, `too_large`, `source_unavailable`,
  `internal`): an invalid-name `create` → `validation`; a `delete`/`rotate` of a
  name the caller does not own → `not_found`; an injected unexpected store
  failure → `internal`. Table driven over the sentinels. (Falsifiable: a code
  outside the closed set, or a missing `structuredContent` on an error, fails.)
- R-DXYA-NY06 — **structural:** no `JSONResult` identifier remains in webhooks's
  non-test Go source —
  `grep -rn 'JSONResult' internal cmd --include='*.go' | grep -v _test.go`
  (run from `webhooks/`) returns empty. (Falsifiable: any un-migrated call site
  fails; the whole service no longer compiles against a JSONResult-free appkit
  until every site is converted.)
