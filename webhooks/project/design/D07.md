# Decision 7 — nginx location fragment (tiers)

## Decision.

`etc/nginx.conf` is the service's path-routed **location fragment** (not a vhost;
no `server{}`, `listen`, or cert). The loopback port is a literal in the fragment
(webhooks's fixed registry port), so the fragment ships verbatim — nothing
substitutes it at install (`opsctl setup` on the box, `nginx/run` in dev). It is
dropped at
`/etc/nginx/conf.d/locations/webhooks.conf` and included by the dashboard apex
block. The live tiers under one `/srv/webhooks/` mount, plus a feed shield and a
catch-all:

```nginx
# 1. unauth PRM bootstrap (exact — wins over the /in/, /static/ and / prefixes)
location = /srv/webhooks/.well-known/oauth-protected-resource {
    proxy_pass http://127.0.0.1:3006/.well-known/oauth-protected-resource;
    proxy_set_header Host $host; proxy_set_header X-Forwarded-Proto $scheme;
}

# 2. gated MCP (exact) — auth_request, identity hygiene, 429 passthrough
location = /srv/webhooks/mcp {
    auth_request /_authn;
    auth_request_set $wh_owner  $upstream_http_x_owner_email;
    auth_request_set $wh_client $upstream_http_x_client_id;
    auth_request_set $authn_status $upstream_status;
    auth_request_set $authn_retry  $upstream_http_retry_after;
    error_page 500 = @webhooks_authn_500;
    proxy_set_header X-Owner-Email $wh_owner;   # REPLACES any inbound — identity hygiene
    proxy_set_header X-Client-Id  $wh_client;
    proxy_pass http://127.0.0.1:3006/mcp;
    proxy_set_header Host $host; proxy_set_header X-Forwarded-Proto $scheme; proxy_http_version 1.1;
}

# 3. shield the internal feed from the public mount
location = /srv/webhooks/feed { return 404; }

# 4. human LANDING page (exact) — gated by the dashboard browser SESSION cookie,
#    NOT the bearer gate: a browser holds no token, and a name+version page needs
#    only a coarse "is this a logged-in box user" check. Exact `=` so it never
#    shadows /mcp, the PRM well-known, /feed, or the /in/ prefix.
location = /srv/webhooks/ {
    auth_request /_session-authn;
    auth_request_set $wh_session_owner $upstream_http_x_owner_email;
    proxy_set_header X-Owner-Email $wh_session_owner;
    proxy_pass http://127.0.0.1:3006/;      # trailing slash -> upstream sees "/"
    proxy_set_header Host $host; proxy_set_header X-Forwarded-Proto $scheme; proxy_http_version 1.1;
}

# 5. landing STATIC assets (prefix) — same coarse dashboard-session gate as the
#    landing page; proxied to the service's embedded static handler. Longer than
#    the catch-all prefix, so it wins for /static/… and 404s nothing.
location /srv/webhooks/static/ {
    auth_request /_session-authn;
    proxy_pass http://127.0.0.1:3006/static/;
    proxy_set_header Host $host; proxy_set_header X-Forwarded-Proto $scheme; proxy_http_version 1.1;
}

# 6. PUBLIC ingress — NO auth_request, NO identity headers set; proxied to loopback
location /srv/webhooks/in/ {
    client_max_body_size 2m;                    # coarse outer guard; the handler enforces the real 1 MiB cap (D4)
    proxy_pass http://127.0.0.1:3006/in/;   # trailing slash strips the mount → service sees /in/<name>
    proxy_set_header Host $host; proxy_set_header X-Forwarded-Proto $scheme;
}

# 7. catch-all — unknown subpaths 404 here, never fall through to the dashboard
location /srv/webhooks/ { return 404; }

# 8. faithful 429 re-emit when auth_request collapsed a rate-limit to 500
location @webhooks_authn_500 {
    if ($authn_status = 429) { add_header Retry-After $authn_retry always; return 429; }
    return 500;
}
```

- **Landing/static are session-gated, not bearer-gated.** They use
  `auth_request /_session-authn` (the dashboard-owned, loopback-only cookie
  validator that `sites` already uses for its private tier) — never `/_authn`. A
  browser cannot present a bearer token, and a name+version page warrants only a
  coarse "logged-in box user" check, never per-resource authorization. A failed
  session check yields `401`.
- **Ingress (tier 6) sets no identity headers** (so D4's defense-in-depth never
  404s a real call) but **does** set `X-Forwarded-Proto` — legitimate, and D4
  deliberately ignores it.
- **Precedence:** exact-`=` (tiers 1–4) always beat prefixes, so the landing root
  `= /srv/webhooks/` and the PRM/`/mcp`/`/feed` exacts never shadow one another;
  among prefixes longest-match wins, so `/srv/webhooks/static/` and
  `/srv/webhooks/in/` both beat the catch-all `/srv/webhooks/`. No tier shadows
  another.
- **`client_max_body_size 2m`** sits above the handler's authoritative 1 MiB cap,
  so the handler's `413` (D4) is the binding limit rather than nginx's default
  `1m` silently pre-empting it.
- **Catch-all `return 404`** (not a gated proxy): webhooks proxies only its known
  surfaces (PRM, `/mcp`, `/in/`, the landing root, and `/static/`); anything else
  404s without reaching the service or the dashboard.

Dev wiring: `webhooks` is already in the `nginx/run` fragment loop (D8) so the dev
front door on `:8080` mirrors prod; the two new tiers ship inside the same
verbatim fragment and need no harness change.

## Rejected.

- **A gated `/srv/webhooks/` catch-all proxy (crm-style) to serve the landing.**
  crm proxies its whole mount through one gated prefix; webhooks exposes only its
  known surfaces. Serving the landing via the catch-all would hand the service
  every unknown subpath as well. The landing is instead an **exact** `=`
  location and static a **dedicated** prefix, both session-gated, while the
  catch-all stays `return 404`.
- **Bearer-gating the landing (`/_authn`).** A browser holds no bearer token, so
  the page would be unreachable to the very humans it is for. The coarse
  dashboard-session gate is the correct trust check for a name+version page.
- **Omitting the `/feed` 404 shield.** Would expose the internal, unauthenticated
  SSE feed through the public mount — the feed is loopback-only by design.
- **Relying on nginx's default `1m` body limit.** Leaves the cap implicit in the
  proxy and couples it to an nginx default; the explicit `2m` + handler `1 MiB`
  keeps the authoritative cap in the service.

## Verification.

The ingress/MCP/PRM/feed ids exercise **real nginx** via the dev front door on
`:8080` with the suite up (harness in D8) — these hinge on real nginx routing and
cannot be proven against a stub. The **landing/static** ids are proven by a
**content-assertion test** over the committed fragment file (mirroring crm's
`nginx_test.go`): the session gate cannot be driven live through `:8080` because
the e2e harness holds no dashboard session cookie, so the fragment's exact-match,
session-gate, and upstream target are asserted against the fragment text — the
substrate that can falsify a wrong location.

- R-OD12-3CVG — Through `:8080`, `POST /srv/webhooks/in/<name>` carrying a valid
  bearer secret and NO OAuth token returns `202`. (Proves the ingress tier is
  unauthenticated, proxied to the service, and injects no identity header — a wrong
  fragment that gated it would 401, and one that set `X-Owner-Email` would make the
  handler 404.)
- R-OE8Y-H4M5 — Through `:8080`, a request to `/srv/webhooks/mcp` with no bearer
  token is rejected with `401`. (Proves the MCP tier is gated via `auth_request`.)
- R-OFGU-UWCU — Through `:8080`, `GET
  /srv/webhooks/.well-known/oauth-protected-resource` returns `200` without any
  token. (Proves the PRM bootstrap tier is open.)
- R-OGOR-8O3J — Through `:8080`, `GET /srv/webhooks/feed` returns `404`. (Proves
  the internal feed is shielded from the public mount.)
- R-TTUW-5O3V — The committed fragment defines the landing as an **exact-match**
  `location = /srv/webhooks/` that is **session-gated** (`auth_request
  /_session-authn`, not `/_authn`), forwards the session owner
  (`proxy_set_header X-Owner-Email $wh_session_owner`), and proxies to the
  upstream **root** `http://127.0.0.1:3006/` with a trailing slash. (A wrong
  fragment that used the bearer gate, a prefix instead of `=`, or the wrong
  upstream path fails the content assertion.)
- R-TV2S-JFUK — The committed fragment defines `location /srv/webhooks/static/`
  as **session-gated** (`auth_request /_session-authn`) and proxies to the
  upstream `http://127.0.0.1:3006/static/`. (Proves landing assets are gated by
  the same coarse session as the page and reach the embedded static handler.)
- R-TWAO-X7L9 — Adding the landing/static tiers leaves every prior tier intact in
  the committed fragment: the catch-all `location /srv/webhooks/` still `return
  404`, `/mcp` is still bearer-gated (`auth_request /_authn`), and
  `/srv/webhooks/feed` still `return 404`. (Proves the additive tiers shadow none
  of the existing surfaces — a fragment that turned the catch-all into a proxy or
  dropped the feed shield fails it.)
