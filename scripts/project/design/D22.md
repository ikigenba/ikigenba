# Decision 22 — `suite.mcp(service, tool, args)`: the generic MCP verb client

**Problem.** The structured MCP contract makes every domain verb
machine-callable — `structuredContent` under a declared `outputSchema`, typed
error codes — precisely so a deterministic caller needs **one generic client,
not per-service libraries** (`docs/structured-mcp-design.md`, "No per-service
client libraries"). A run must be able to call any suite service's tools with
identity asserted correctly and errors surfaced as the typed exceptions D21
pins.

**Decision.** One function in `suite.py`:

```python
def mcp(service, tool, args=None):
    """Call a suite service's MCP tool; returns its structured result."""
```

- **Addressing.** `service` is looked up in the `SUITE_SERVICES` map (D21). An
  unknown name raises `ValueError` **locally, before any HTTP** — a programming
  error, not a `ToolError`. `args=None` is sent as `{}`.
- **The wire call.** One `POST <base>/mcp` with body
  `{"jsonrpc": "2.0", "id": 1, "method": "tools/call",
  "params": {"name": <tool>, "arguments": <args>}}` — the appkit MCP endpoint
  is stateless for `tools/call` (no `initialize` handshake; verified against
  the running suite). Headers:
  - `X-Owner-Email: $SUITE_OWNER_EMAIL` — required: appkit's
    `requireIdentityHeaders` answers 401 to an empty owner on `/mcp`;
  - `X-Client-Id: scripts:$SUITE_SCRIPT_ID` — the caller-asserted machine
    identity (the suite convention `svc:<id>`, exactly prompts' `suite.go`
    shape); services stamp it into emitted events' `origin`;
  - `Content-Type: application/json`. **No `X-Forwarded-Proto`** — this is a
    loopback caller, not a proxied request.
- **Result mapping** (in order):
  1. transport failure — connection refused, timeout, torn body →
     `ToolError("source_unavailable", …)`;
  2. non-200 HTTP status → `ToolError("internal", "<service> /mcp returned
     <status>")`;
  3. a JSON-RPC `error` object → `ToolError("validation", message)` for
     `-32602` (malformed arguments), else `ToolError("internal", message)`;
  4. `result.isError` true → `ToolError(code, message)` taken from the
     result's `structuredContent` `{code, message}` (the closed-vocabulary
     error the service declared); a legacy error result without
     `structuredContent` maps to `ToolError("internal", <concatenated text>)`;
  5. success **with** `structuredContent` → return that object (a dict) —
     verbatim, never reshaped;
  6. success **without** `structuredContent` (a prose tool: `describe`,
     `run_output`, wiki's `page`, …) → return the concatenated text of the
     `content` blocks (a str). The mild return-type polymorphism mirrors
     exactly what the tool declares: `outputSchema` present → dict, absent →
     prose. The caller knows which tool it is calling.

**Rejected.**
- **Per-service wrapper namespaces (`suite.crm.save(…)`)** — a second,
  drift-prone description of an API that is already self-describing; the road
  `docs/structured-mcp-design.md` explicitly closed.
- **Raising on prose (schema-less) results** — walls off real tools for
  purity; the text *is* those tools' contract.
- **Wrapping prose results (`{"text": …}`)** — invents a shape the wire does
  not have; verbatim over invented.
- **Performing the MCP `initialize` handshake** — the appkit endpoint
  dispatches `tools/call` statelessly; a handshake would add a round trip and
  state for nothing.
- **Retries on `source_unavailable`** — D21's no-retry posture; the script
  owns its retry policy.

**Verification** (real substrate: probe scripts under real `python3` — the D21
harness — against a **real local HTTP server**, `httptest` on loopback playing
a suite service's `/mcp`: it records method, path, headers, and body, and
returns canned JSON-RPC responses; its URL is injected via `SUITE_SERVICES`).

- R-I0GA-YTQ5 — the happy path: `suite.mcp("svc", "get", {"id": "x"})` issues
  exactly one `POST /mcp` whose recorded body is a JSON-RPC `tools/call` for
  tool `get` with arguments `{"id": "x"}` and whose recorded headers carry
  `X-Owner-Email` equal to `$SUITE_OWNER_EMAIL`, `X-Client-Id` exactly
  `scripts:$SUITE_SCRIPT_ID`, and **no** `X-Forwarded-Proto` — and returns the
  response's `structuredContent` object verbatim (deep-equal, dict). An
  omitted `args` is sent as `{}`.
- R-I1O7-CLGU — a success result **without** `structuredContent` (two text
  content blocks) returns their concatenated text as a `str` — proving the
  prose fallback, not an exception and not a wrapper dict.
- R-I2W3-QD7J — an `isError` result with
  `structuredContent {code: "not_found", message: m}` raises `ToolError` with
  `.code == "not_found"` and `.message == m` (probe catches and prints both) —
  a machine branches on `.code`, never on message text.
- R-I5BW-HWOX — `suite.mcp("nonesuch", "get")` raises `ValueError` and the
  server records **zero** requests — the unknown-service rejection is local.
- R-I6JS-VOFM — with the service entry pointing at a closed loopback port,
  `suite.mcp` raises `ToolError` with `.code == "source_unavailable"` —
  transport failure is typed, distinguishable from absence and from a bug.
