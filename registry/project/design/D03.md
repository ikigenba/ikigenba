# Decision 3 — The resolution API: name → port, name → base URL, loud on unknown

## Decision

Callers never touch the `Services` slice directly; they resolve by name through a
small total API. The loopback host is the one constant `127.0.0.1`, quarantined
here so no caller ever writes it again.

```go
const loopbackHost = "127.0.0.1"

// Port returns the loopback port for a known service name; ok is false for an
// unknown name (and port is then 0). This is the checked form for callers that
// want to handle a miss.
func Port(name string) (port int, ok bool)

// MustPort returns the port for name, or panics if name is unknown. The strict
// form for composition-root wiring where an unknown name is a programming error.
func MustPort(name string) int

// BaseURL returns "http://127.0.0.1:<port>" for a known name, and panics on an
// unknown name (it is MustPort composed with the loopback host). This is what
// collapses the hardcoded peer maps: registry.BaseURL("crm") replaces a literal
// "http://127.0.0.1:3100".
func BaseURL(name string) string
```

Lookup is O(1): a `map[string]int` (or `map[string]Service`) built once from
`Services` in an `init` (or a package-level composite literal derived from the
slice). The slice stays the source of truth; the map is a derived index.

The unknown-name policy is deliberately **two-tier**, mirroring the suite's "fail
loudly" principle: `Port` gives callers a checked miss (`ok == false`, never a
zero masquerading as a real port), while `MustPort`/`BaseURL` panic — because a
name that isn't in the compile-time table is a bug in the caller, not a runtime
condition to recover from.

`BaseURL` returns only the scheme+host+port origin, with **no path**. Appending
`/feed`, `/health`, etc. is the caller's concern — `registry` owns the address, not
the routes. A consumer writes `registry.BaseURL("crm") + "/feed"`.

## Rejected

- **A single `Port(name) int` that returns `0` on miss.** A zero port is a valid
  int that flows downstream into a broken URL — a silent wrong address, the exact
  failure mode this project removes. The checked `(int, bool)` plus the panicking
  `MustPort` make a miss impossible to ignore.
- **A `FeedURL(name)` / `HealthURL(name)` route helper set in `registry`.** Routes
  like `/feed` are an `appkit`/eventplane convention, not addressing. Baking them
  in couples the address table to route names and invites one helper per path.
  `registry` stops at the origin; callers append their path.
- **Returning `*url.URL` from `BaseURL`.** Every current call site wants a string
  to concatenate a path onto; handing back a `*url.URL` forces `.String()` at each
  site for no benefit and pulls `net/url` into the type surface. A string origin is
  what the peer maps hold today.
- **`localhost` as the host.** The existing peer-map and `/feed` literals all use
  `127.0.0.1`; matching them avoids a DNS round-trip and a behavior change.

## Verification

- R-B642-6EO8 — **`Port` returns the exact registered port and `ok == true` for a
  known name** (e.g. `Port("crm")` → `3100, true`; `Port("dashboard")` →
  `3000, true`). Falsifiable: a wrong port or `ok == false` for a known name fails
  it.
- R-B7BY-K6EX — **`Port` returns `ok == false` (and `port == 0`) for an unknown
  name** and does **not** panic. Falsifiable: a lookup that panics, or returns
  `ok == true`, or a nonzero port, fails it. Pins the checked-miss contract.
- R-B8JU-XY5M — **`MustPort` panics on an unknown name** (fail loudly) and returns
  the correct port for a known one. Falsifiable: a `MustPort("nope")` that returns
  instead of panicking fails it; asserted via `recover`.
- R-B9RR-BPWB — **`BaseURL` composes exactly `http://127.0.0.1:<port>`** for a
  known name (e.g. `BaseURL("crm")` == `"http://127.0.0.1:3100"`) with no trailing
  path or slash. Falsifiable: a different host, a wrong port, or an appended path
  fails it.
