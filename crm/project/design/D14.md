# Decision 14 — Delete the chassis shims: `internal/ids` and the `internal/db` wrappers

**Problem.** Two crm packages exist only to forward to appkit. `internal/ids`
is a byte-level copy of the ULID helper appkit already exports
(`appkit/logging.NewULID` — same 48-bit-time + 80-bit-random, Crockford
base32). `internal/db` wraps `appkit/db.Open`/`LoadMigrations`/`Migrate` in
`Open`/`Migrate` functions whose only value is saving an import line. Shims
like these are exactly the per-service drift surface this design round removes;
crm is the reference shape other services copy, so it must not model them.

**Decision.**

1. **`crm/internal/ids` is deleted.** The domain package's call sites
   (`contact.go`, `task.go`, `interaction.go`, …) import `appkit/logging` and
   call `logging.NewULID()`. Generated ids are unchanged in shape (26-char
   Crockford base32 ULIDs), so nothing downstream notices.
2. **`crm/internal/db` keeps only what is app-side:** the embedded migration
   set (`//go:embed migrations/*.sql`, exported as `FS` for
   `appkit.Spec.Migrations`) and the `migrations_outbox_test.go` byte-equality
   guard. The `Open` and `Migrate` wrapper functions are deleted; the domain
   and MCP test harnesses call `appkit/db.Open` and
   `appkit/db.Migrate(ctx, conn, appkit/db.LoadMigrations(db.FS, "migrations"))`
   directly.
3. **No behavior changes.** Same pragmas, same runner, same migration bytes;
   the suite proves it by staying green with no test *assertions* changed
   (only harness plumbing).

**Rejected.**
- **Keeping the shims for import brevity** — two extra lines per call site is
  not worth a per-service package that can drift (crm's `ids` doc comment
  already described session/state ids crm never had — copy rot in action).
- **Deleting `internal/db` entirely** — the migrations embed must live in a
  crm package (`go:embed` cannot cross modules) and the outbox DDL guard test
  is genuinely app-side; the package shrinks to its legitimate content instead.
- **Hoisting the outbox migration SQL into appkit** — deliberately deferred
  (committed migrations are immutable; appkit-owned migration SQL raises
  versioning questions that deserve their own design round).

**Verification.** Structural — no `R-XXXX-XXXX` ids (shim deletion; the
behaviors are already pinned by the existing domain/MCP suites). Realized by a
plan phase whose done bar is:

- `grep -rn "crm/internal/ids" crm --include=*.go` returns no matches, and
  `crm/internal/ids/` does not exist.
- `grep -rn "func Open\|func Migrate" crm/internal/db/db.go` returns no
  matches, while `db.FS` and the outbox guard test remain.
- The suite stays green (design Conventions commands) with no assertion
  changes.
