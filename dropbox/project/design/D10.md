# Decision 10 — Prove no loopback-port literal survives, and guard the deploy artifacts against registry drift

**Decision.** The adoption in D9 is only complete if it is **enforced by a
test**, not by good intentions (the suite's "enforce with systems" principle).
Two guards, both living entirely under `dropbox/` and running in the ordinary
`cd dropbox && go test ./...`:

1. **A source-scan guard** proving no bare loopback-address string literal of the
   form `127.0.0.1:30xx` remains anywhere in dropbox's Go source.
2. **A drift guard** proving the two static deploy artifacts that *must* carry
   the literal port — `etc/manifest.env` and `etc/nginx.conf` — agree with the
   authoritative `registry` value, so a future renumber fails a dropbox test
   instead of silently reaching deploy.

**Why two guards, and why this split.** After D9 there are two distinct kinds of
port literal left in the `dropbox/` tree:

- **Go-source loopback URLs** (the `events.go` example origin D9 removes) — this
  is the drift vector D9 eliminates, and guard (1) proves it is gone and stays
  gone.
- **Non-Go deploy artifacts** — `etc/nginx.conf` is an nginx fragment (nginx
  cannot call Go) and `etc/manifest.env` is generated deploy metadata; both
  legitimately contain the literal `3200`. `registry` cannot *own* their
  contents without generating them, which is out of scope. What it *can* do is
  make dropbox's own tests assert those artifacts match `registry`, converting a
  silent drift into a red test. Guard (2) does exactly that.

**Guard (1) — source-scan (`R-QLO8-2HE3`).** A test in `cmd/dropbox` (or a small
new `internal` guard package) walks every `*.go` file under the `dropbox/`
module and fails if any file's source contains a bare loopback-port address
literal (`127.0.0.1:30` followed by two digits). Two constraints make it robust:

- **It must not match itself.** The test assembles the forbidden needle from
  fragments at runtime (e.g. `"127.0.0.1:" + "30"`) rather than embedding the
  full literal, and skips its own filename, so the guard's own source can never
  trip it.
- **It scans real Go source, including comments.** After D9 the only sites that
  carried the address form — `internal/dropbox/events.go` (the Sample origin and
  the `NewOutboxProducer` doc comment) and, until phase 08, the nginx-fragment
  test's `proxy_pass` assertions — are converted to `registry` calls, so the
  scan is green; reintroducing a hardcoded loopback URL turns it red.

This guard covers the **string-literal** loopback URLs. It deliberately does
**not** try to flag the bare integer `3200` (an int literal like `3200` is
ambiguous and grepping for it is noisy). The two own-port integer literals
(`Spec.Port`, the `DROPBOX_PORT` default) are eliminated by D9's
`registry.MustPort("dropbox")` changes and proven gone by guard (2), which emits
the manifest from `registry` and matches the committed file.

**Guard (2) — deploy-artifact drift (`R-QMW4-G94S`).** Two existing tests are
re-pointed at `registry` so their expected port is *derived*, not hardcoded:

- **`cmd/dropbox/main_test.go`** — the manifest byte-equality test
  (`TestManifestLibraryByteEqualsCommittedFile`, existing id R-8IAN-FB87)
  currently emits `manifest.Emit(manifest.Fields{… Port: 3200 …})` and compares
  to the committed `etc/manifest.env`. Change the literal to
  `Port: registry.MustPort("dropbox")`. Every other field
  (`Feed:"/feed"`, the `OUTBOX_RETENTION_*` extras) is unchanged; the emitted
  `PORT=3200` still byte-matches the committed file today. If `registry`
  renumbers dropbox, `manifest.Emit` produces a different `PORT` line and the
  byte-equality assertion fails — surfacing that the committed `manifest.env` is
  stale.
- **the nginx fragment tests** (`internal/web/nginx_test.go` today; they relocate
  to `cmd/dropbox` **unchanged** when D11 dissolves `internal/web`) — the
  exact-match landing block and the `/static/` block assert their `proxy_pass`
  upstream as the literal `http://127.0.0.1:3200/` and
  `http://127.0.0.1:3200/static/`. Build the expected origin from
  `registry.BaseURL("dropbox")` (== `"http://127.0.0.1:3200"`) and assert the
  fragment contains `"proxy_pass " + registry.BaseURL("dropbox") + "/"` and the
  `"…/static/"` variant. The assertion holds today; a renumber makes the
  still-literal `etc/nginx.conf` disagree with `registry.BaseURL("dropbox")` and
  the test goes red.

Both re-pointed tests keep their **existing** behavioral assertions — the
session gate (`auth_request /_session-authn`), the exact-match vs prefix
distinction, the surviving PRM / bearer-prefix / `= /srv/dropbox/content` 404
locations, and the manifest's non-port fields; only the **port value** they
compare against stops being a hardcoded number and becomes a `registry` call.
This removes the last `127.0.0.1:3200` / `3200` literals from dropbox's Go
source while turning the static artifacts into drift-caught, not drift-prone.

**Scope guard.** These guards add tests only — no production code beyond D9, no
new package required (a single guard test file is enough), no schema change (**no
migration**), and no edit to `etc/nginx.conf` or `etc/manifest.env` themselves
(their literals stay; the tests now police them). Nothing outside `dropbox/` is
touched.

**Rejected.**
- **Regenerating `etc/nginx.conf` / `etc/manifest.env` from `registry`.** Out of
  scope and unnecessary: a drift-catching test delivers the safety without a
  codegen step or a cross-module dependency.
- **An AST/int-literal scan for the bare `3200`.** Too noisy to be falsifiable —
  `3200` carries no context that marks it as a port. Deriving the manifest and
  nginx expectations from `registry` (guard 2) proves the own-port literal is
  gone far more precisely.
- **Leaving the nginx/manifest tests as literal `3200` assertions.** They would
  keep bare `30xx` literals in dropbox's Go source (violating D9's end-state)
  and, worse, would keep *passing* after a `registry` renumber — the exact
  silent drift this whole adoption removes.

**Verification.**
- R-QLO8-2HE3 — **no bare `127.0.0.1:30xx` loopback-address string literal
  exists in dropbox's Go source.** A guard test walks the module's `*.go` files
  (skipping itself, needle assembled at runtime) and fails on any such literal.
  Falsifiable: reintroducing a hardcoded loopback origin like
  `"http://127.0.0.1:3200"` makes it red; with D9 + phase 08 applied it is green.
- R-QMW4-G94S — **the committed `etc/manifest.env` and `etc/nginx.conf` ports
  agree with `registry`.** The manifest test emits with
  `registry.MustPort("dropbox")` and byte-matches the committed `manifest.env`;
  the nginx tests assert the fragment contains `registry.BaseURL("dropbox") + "/"`
  proxy targets. Falsifiable: a `registry` value that differs from the committed
  artifacts' `3200` fails these tests (the intended drift alarm); matching
  values pass.
