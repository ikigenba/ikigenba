# Decision 13 — Delete the chassis shim (`internal/db` wrappers) and true up the doctrine doc

**Decision.** Two cleanups that land the reference shape whole, mirroring crm's
and notify's shim deletion:

1. **`internal/db` shrinks to its legitimate content.** The `Open` and `Migrate`
   wrapper functions in `dropbox/internal/db/db.go` are deleted — appkit owns DB
   opening and the migration run (`Spec.Migrations: db.FS`, already wired). What
   remains is what genuinely must live in a dropbox package: the embedded
   migration set (`migrationsFS`/`FS`; `go:embed` cannot cross modules) and the
   app-side migration guards — `migrations_load_test.go` (the embedded set loads
   through `appkit/db`'s runner) and `migrations_outbox_test.go` (asserts
   `003_outbox.sql` contains `outbox.SchemaSQL` verbatim — the producer's outbox
   DDL guard). The `context`/`database/sql`/`appkit/db` imports used only by the
   deleted wrappers go with them.
2. **Test harnesses call appkit's db package directly.** Two test files call the
   deleted `db.Migrate` today: `internal/dropbox/store_test.go` and
   `internal/mcp/tools_test.go`. Their harness helpers switch to
   `appkitdb.LoadMigrations(db.FS, "migrations")` + `appkitdb.Migrate(ctx, conn, migs)`
   (and `appkitdb.Open` where they open a handle) — the same swap notify's
   `migrateNotify` helper made. **No test assertion changes** — harness plumbing
   only.
3. **`dropbox/CLAUDE.md` states the converted truth**, per the suite doctrine
   (docs state current truth; purge what the change falsified; no archaeology).
   dropbox's doc file is `CLAUDE.md` (a regular file — the service carries no
   separate `AGENTS.md`; where the suite convention pairs them, edit the one that
   exists). The falsified claims to purge:
   - the **MCP surface** described as an in-package "JSON-RPC 2.0 transport"
     with `internal/mcp/mcp.go` "the transport" and `tools.go` holding "the four
     tool descriptors" it dispatches — replaced by: `internal/mcp` declares
     `Instructions` + the two domain tools (`list`, `get`) as an `appkit/mcp`
     tool table assembled by `NewHandler`; `health` and `reflection` are the
     chassis-standard tools (the surface is still four tools, but two are
     chassis-owned, not locally implemented);
   - the **`internal/db`** line claiming "SQLite open (WAL, FK, single-writer) +
     embedded migration runner" — replaced by: appkit owns open + the
     forward-only runner via `Spec.Migrations`; `internal/db` keeps only the
     embedded `*.sql` set and the load + `003_outbox.sql` byte-equality guards;
   - the stale **`internal/server` / `internal/logging` / `internal/ids`** layout
     entries (those packages no longer exist — the only `internal/` packages are
     `db`, `dropbox`, `mcp`, `web` before D11 and `db`, `dropbox`, `mcp` after);
   - the **`internal/web`** landing/embedded-assets framing wherever it survives
     — replaced by: the human landing page and its `static/` assets ship on disk
     under `dropbox/share/www/`, loaded through `Spec.WWW` and served by the
     chassis (`rt.WWW()` + the auto-mounted `GET /static/`);
   - the **hardcoded `port 3200`** framing — replaced by: the loopback port is
     resolved by name via `registry.MustPort("dropbox")` at the composition root
     (the value is still 3200; the doc states the mechanism, not a literal).

   Preserved untouched: the daemon/producer model, the longpoll sync engine and
   its load-bearing correctness rules, the `/feed` outbox, the loopback
   `GET /content` and `GET /list` byte routes and their private-guard contract,
   the events, the secrets story, and the no-backup decision — this conversion
   changed none of them.

**Rejected.**
- **Keeping the wrappers for import brevity** — a per-service shim that can drift,
  for two saved lines per call site; crm's and notify's shim deletion already
  made this call.
- **Deleting `internal/db` entirely** — the migrations embed must live in a
  dropbox package and the `003_outbox.sql` byte-equality guard is genuinely
  app-side; the package shrinks to its legitimate content instead.
- **Leaving `CLAUDE.md` for a later pass** — the file would assert wiring
  (JSON-RPC transport, a local migration runner, embedded web assets, a hardcoded
  port, phantom `internal/server`/`logging`/`ids` packages) that no longer exists
  in the same tree that changed it; docs move with the code.

**Verification.** Structural — no `R-XXXX-XXXX` ids (shim deletion + doc truth;
the behaviors are already pinned by the existing domain/MCP suites and the
D9–D12 ids). Realized by a plan phase whose done bar is:

- `grep -n "func Open\|func Migrate" dropbox/internal/db/db.go` returns no
  matches, while `grep -n "go:embed migrations" dropbox/internal/db/db.go` still
  matches and both `dropbox/internal/db/migrations_load_test.go` and
  `dropbox/internal/db/migrations_outbox_test.go` still exist.
- `grep -rn "db\.Migrate\b" dropbox --include=*.go` returns no matches (the
  harnesses call `appkitdb` directly).
- In `dropbox/CLAUDE.md`: `grep -n "internal/server\|internal/logging\|internal/ids" dropbox/CLAUDE.md`
  returns no matches; `grep -n "JSON-RPC 2.0 transport" dropbox/CLAUDE.md`
  returns no matches; and the converted truth is present —
  `grep -n "share/www" dropbox/CLAUDE.md` and
  `grep -n "registry.MustPort" dropbox/CLAUDE.md` each match at least once.
- The suite stays green (design Conventions commands) with no assertion changes.
