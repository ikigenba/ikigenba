# Decision 6 — The `appkit/web` package: templates + static assets over an on-disk root

**Problem.** Every service hand-rolls the same landing/static harness over its
own `//go:embed` copies, with accidental drift (double `StripPrefix`, a combined
handler, `.tmpl` vs `.html`). The mechanism — load templates, render a page,
serve `static/` with correct content types — is chassis work with no legitimate
per-service variation. Only the *content* (which pages, what data) is the
service's.

**Decision.** A new package `appkit/web` owns the mechanism over an on-disk
root. The root's conventional shape is:

```
<root>/                  (= share/www in the service source tree)
├── *.html, *.tmpl       page templates (any mix; all parsed into one set)
└── static/              assets served verbatim (tokens.css, fonts/*.woff2, …)
```

Public surface:

```go
package web // import "appkit/web"

// Site is a service's loaded web surface: the parsed template set and the
// static file tree, both rooted at one on-disk directory.
type Site struct { /* root string; tmpl *template.Template */ }

// Load parses every *.html and *.tmpl file directly under root into one
// html/template set (named by base filename, so layout/partial composition
// works across files) and returns the Site. It fails if root does not exist or
// is unreadable — a declared web root that is missing is a wiring error, not a
// blank page. A root with no template files is allowed (a static-only site).
func Load(root string) (*Site, error)

// Render executes the named template (base filename, e.g. "landing.html") to w
// with data, setting Content-Type: text/html; charset=utf-8. An unknown name
// is an error.
func (s *Site) Render(w http.ResponseWriter, name string, data any) error

// Static returns the handler for the site's static/ subtree: it strips the
// /static/ URL prefix and serves <root>/static from disk with correct content
// types and NO directory listings (a directory request is 404, mirroring the
// dashboard's noDirFS). Bytes are read from disk per request, so a dev edit is
// visible on refresh.
func (s *Site) Static() http.Handler
```

Details pinned here:

- **Templates parse once, at `Load`.** One behavior everywhere; a template edit
  in dev needs a restart (`bin/stop && bin/start`), which is accepted. Static
  assets, by contrast, are read per request (no parse step to cache).
- **`html/template`**, matching the existing per-service handlers.
- **woff2 mime**: the package's `init()` registers
  `mime.AddExtensionType(".woff2", "font/woff2")` — today re-registered in
  every service's embed shim.
- Subdirectories of root other than `static/` are not parsed and not served;
  the surface is deliberately exactly "top-level templates + one static tree"
  until a real service needs more.

**Rejected.**
- **Re-parse templates per request in dev** — two behaviors selected by
  environment; the dev win is small (a restart is cheap) and divergent dev/prod
  behavior is exactly the class of bug this suite avoids.
- **Serving the whole root (not just `static/`)** — would serve the raw
  template sources as pages; the split keeps "rendered by Go" and "served
  verbatim" disjoint.
- **An overlay of roots (`state/www` over `share/current/www`)** — no service
  needs operator-mutable Go-served pages today; dynamic page *content* comes
  from handlers rendering data, not from mutable template files. A single root
  keeps semantics obvious; overlay can be a future extension of `Load` if a
  real case appears.
- **Directory listings** — never wanted on a service mount; 404 mirrors the
  dashboard's `noDirFS` precedent.

**Verification** (all against a `t.TempDir()` root written by the test).
- R-M0CJ-U84T — `Load(root)` over a root containing `landing.html` (with a
  `{{.Service}}`/`{{.Version}}` body) succeeds, and
  `Render(w, "landing.html", data)` writes a `200`-style body containing the
  data's service and version values with
  `Content-Type: text/html; charset=utf-8`.
- R-M1KG-7ZVI — `Load` of a nonexistent root returns a non-nil error naming the
  path (it does not return an empty usable Site).
- R-M2SC-LRM7 — `Static()` serving `GET /static/tokens.css` over a root whose
  `static/tokens.css` exists returns `200` with a `text/css` content type and
  the file's bytes.
- R-M408-ZJCW — `Static()` serving `GET /static/fonts/x.woff2` returns `200`
  with content type `font/woff2`.
- R-M585-DB3L — `Static()` serving a directory path (`GET /static/` or
  `GET /static/fonts/`) returns `404` — no autoindex listing is emitted.
