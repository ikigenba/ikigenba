# Decision 7 — Chassis integration: `Spec.WWW`, the auto-mounted static route, `Router.WWW()`

**Problem.** With D5 resolving the root and D6 owning the mechanism, the chassis
must decide how a service opts in, what the chassis mounts on its behalf, and
what stays in the service's hands. The drift audit is unambiguous: serving
`/static/` has zero legitimate per-service variation (and four accidental
variants in the wild), while pages are exactly where services legitimately
differ.

**Decision.** Static is chassis-mounted; pages stay in `Spec.Handlers`.

```go
type Spec struct {
    // …existing fields…
    // WWW declares the service serves web pages/assets from the on-disk
    // www root (config WWWPath). At serve, appkit loads the root with
    // web.Load — a missing root is a fatal startup error — mounts
    // GET /static/ over the site's static tree, and exposes the loaded
    // site via Router.WWW() for the service's page handlers.
    // False (the default) changes nothing.
    WWW bool
}
```

- **`runServe`**: when `spec.WWW` is true, `site, err := web.Load(cfg.WWWPath)`
  runs right after config resolution; a load error fails the verb (fail loudly
  at startup — the service never comes up unstyled). The site is passed into
  `server.Options` (a new `WWW *web.Site` field).
- **`server.New`**: when `Options.WWW != nil` (and not apex), the standard
  route table gains `mux.Handle("GET /static/", site.Static())` — ungated
  in-process like the landing page (nginx owns the session gate, per the
  services' existing nginx fragments). The `Router` gains an accessor:

  ```go
  // WWW returns the loaded web site (nil when Spec.WWW is unset), so a
  // service's Handlers hook renders its pages through the chassis-loaded
  // templates: rt.WWW().Render(w, "landing.html", data).
  func (rt *Router) WWW() *web.Site
  ```

- **Pages stay service-side.** The service mounts `GET /{$}` (and any richer
  routes) in `Handlers`, rendering via `rt.WWW()`. The chassis never guesses at
  a landing route: which pages exist and what data they carry is domain.
- **Non-serve verbs** (`version`/`manifest`/`migrate`/`schema`) never touch the
  www root; a missing root must not break `migrate` on a box mid-provision.
- **No manifest change.** The root is derived from the layout (`IKIGENBA_ROOT`)
  or overridden by env; there is no new manifest key.
- **Dev wiring (boundary-crossing collaborator).** A converted service launched
  by the repo-root `bin/start` runs with the repo root as cwd, so the dev
  default `./share/www` (D5) does not resolve; its launch function exports
  `<APP>_WWW_PATH="$repo/<app>/share/www"` beside the existing
  `<APP>_DB_PATH` export. Like D2–D4 before it, this is a one-line repo-root
  script change verified by the live `bin/start` smoke, not by the appkit Go
  suite; the plan phase calls it out.

**Rejected.**
- **Auto-detecting the root by directory existence (no `Spec.WWW`)** — a deploy
  that forgot to ship `share/www` would silently degrade to no static route;
  the explicit declaration plus fail-loudly startup is the "unknown state is
  unacceptable" principle applied.
- **Chassis-mounting the landing route too** — pages need per-service data and
  will grow per-service routes (webhooks' active-hooks table, wiki's multi-page
  UI); a chassis-owned landing handler would be un-owned at the first real
  page. Uniformity where variation is illegitimate; a seam where it's the
  point.
- **Keeping the static mount service-side (library-only)** — leaves the exact
  three lines where today's four-way drift lives; the audit shows nobody varies
  it on purpose.
- **Loading the site lazily at first request** — moves the missing-root failure
  from startup (visible in the deploy) to first human visit (invisible until
  someone looks).

**Verification** (at the `server.New`/`httptest` seam, with a Site loaded from a
`t.TempDir()` root; the `runServe` wiring is the same values threaded through).
- R-M7NY-4UKZ — a non-apex server built with `Options.WWW` set serves
  `GET /static/tokens.css` with `200`, `text/css`, and the root's bytes — with
  **no** service-side `Handlers` registration for `/static/`.
- R-M8VU-IMBO — a `Spec{WWW: true}` serve against a nonexistent www root fails
  at startup with an error naming the path (asserted at the seam that loads the
  site — `web.Load` failing the serve path — not by booting a real box); it
  does not start and serve template-less pages.
- R-MA3Q-WE2D — inside a `Handlers` hook registered on a `WWW`-enabled server,
  `rt.WWW()` returns a non-nil Site whose `Render` serves a page route
  (`GET /{$}` returning the rendered template) through the real mux.
- R-MBBN-A5T2 — a server built **without** `Options.WWW` behaves exactly as
  before: `GET /static/anything` is `404` (no route), and `rt.WWW()` returns
  nil — proving the feature is strictly additive for unconverted services.
