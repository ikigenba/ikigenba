# Decision 8 — The `appkit/mcp` JSON-RPC transport over a declared tool table

**Problem.** Eleven services carry a copy-pasted ~150–190-line MCP transport —
`ServeHTTP`, the `initialize` / `notifications/initialized` / `tools/list` /
`tools/call` dispatch, `writeJSONRPCError`, the error envelope — differing only
in server name, instructions string, and tool table. A protocol fix must be
made eleven times.

**Decision.** A new package `appkit/mcp` owns the transport. A service declares
its tools; the chassis speaks JSON-RPC 2.0 over plain HTTP POST (no
SSE/streaming), preserving the exact wire semantics the per-service copies
share today:

```go
package mcp // import "appkit/mcp"

// Tool is one declared MCP tool: its wire descriptor plus its handler.
type Tool struct {
    Name        string
    Description string
    InputSchema map[string]any // JSON Schema object, e.g. the services' obj(...) shape
    // Handler receives the raw tools/call arguments and the nginx-injected
    // caller identity, and returns the MCP result object (the services'
    // toolResult* shape) or an error (rendered as a JSON-RPC error).
    Handler func(ctx context.Context, args json.RawMessage, id server.Identity) (map[string]any, error)
}

// Options assembles a service's MCP surface. Service/Version/Health/Events/
// Publishes/Subscriptions are threaded from the appkit Router accessors at the
// composition root; Instructions and Tools are the service's own.
type Options struct {
    Service       string
    Version       string
    Instructions  string
    Tools         []Tool
    Health        func(ctx context.Context) (map[string]any, error)
    Events        outbox.Registry
    Publishes     func() outbox.Registry
    Subscriptions func() []consumer.Subscription
}

// New validates the tool table (duplicate names and the reserved standard-tool
// names "health"/"reflection" are wiring errors) and returns the http.Handler
// for POST /mcp. The caller mounts it gated: rt.Handle("POST /mcp",
// rt.RequireIdentity(h)).
func New(opts Options) (*Handler, error)

// Result helpers, replacing every service's private copies:
func TextResult(text string) map[string]any             // one text content item
func JSONResult(v any) (map[string]any, error)          // marshaled JSON as text content
func ErrorResult(msg string) map[string]any             // isError:true envelope
```

Wire semantics (unchanged from the per-service copies, now stated once):

- **`initialize`** → result with the protocol version, `capabilities.tools`,
  `serverInfo{name: Service, version: Version}`, and `instructions`.
- **`notifications/initialized`** → acknowledged with no result body.
- **`tools/list`** → the standard tools (D9) plus every declared Tool, each as
  `{name, description, inputSchema}`.
- **`tools/call`** → dispatch by name to the Tool's `Handler` with the raw
  `arguments` and the identity read from `X-Owner-Email` / `X-Client-Id`
  (always present behind `RequireIdentity`); the returned map is the JSON-RPC
  `result`. A handler error and an unknown tool name render as the same
  JSON-RPC error shape the services emit today.
- **Malformed body** → `-32700` parse error; **unknown method** → `-32601`.
- The transport carries **no token logic** — nginx is the trust boundary,
  exactly as every per-service copy documents.

Mounting stays in `Spec.Handlers` (one line), parallel to D7's pages-stay-in-
Handlers: the tool table needs the domain Service built over `rt.DB()`, which
only exists inside the hook.

**Rejected.**
- **A `Spec.Tools` field with chassis auto-mounting** — the tool handlers close
  over the domain Service, which is constructed inside `Handlers`; a Spec-level
  table would need a second constructor hook for no gain over the one-line
  mount.
- **Adopting an external MCP SDK** — the suite's transport is deliberately
  minimal (plain POST, no streaming); a dependency would import far more
  surface than the ~180 lines it replaces.
- **Keeping the transport per-service** — is the status quo the drift audit
  indicts.
- **`any`-typed handler results** — the services' `map[string]any` result shape
  is the working convention; preserving it makes conversion mechanical.

**Verification** (through the real `ServeHTTP` seam with a test tool table).
- R-MCJJ-NXJR — `initialize` returns `serverInfo.name`/`.version` equal to
  `Options.Service`/`.Version` and `instructions` equal to
  `Options.Instructions`.
- R-MDRG-1PAG — `tools/list` includes every declared Tool with its exact name,
  description, and inputSchema.
- R-MEZC-FH15 — `tools/call` for a declared tool invokes its Handler with the
  request's raw `arguments` bytes and returns the handler's map as the JSON-RPC
  `result`.
- R-MG78-T8RU — the identity passed to the Handler carries the request's
  `X-Owner-Email` and `X-Client-Id` header values.
- R-MHF5-70IJ — an unknown method returns JSON-RPC error `-32601`, and a
  `tools/call` naming an undeclared tool returns the error shape (not a `200`
  success result).
- R-MIN1-KS98 — a malformed (non-JSON) body returns JSON-RPC error `-32700`.
- R-MJUX-YJZX — `New` returns a non-nil error for a table with two tools of the
  same name, and for a tool named `health` or `reflection` (reserved by D9);
  it does not silently shadow.
